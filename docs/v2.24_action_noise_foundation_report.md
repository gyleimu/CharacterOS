# CharacterOS V2.24 Action Noise Foundation Report

## Version

```text
CharacterOS V2.24 Action Noise Foundation
```

## Goal

V2.24 implements the first Embodiment layer.

The goal is to separate:

```text
what the character decides to do
```

from:

```text
what the character actually manages to do
```

This follows the Architecture Bible principle:

```text
Decision -> Motor System -> Noise -> Final Action
```

## New Core Module

```text
src/core/embodiment/actionNoise.ts
```

The new module introduces:

```text
ActionNoiseContext
EmbodiedAction
applyActionNoise()
```

Inputs:

```text
intendedAction
MetaState
PsychologicalBoundary
fatigue
pain
fear
excitement
skill
deterministic randomness
```

Outputs:

```text
intendedAction
finalAction
tone
noiseLevel
selfControlAvailable
misfireRisk
reasons
```

## Design Rule

Action Noise does not change the psychological decision.

It only models execution distortion.

This means:

```text
BehaviorDecision.mostLikelyAction stays intact.
EmbodiedAction.finalAction may become colder, shakier, avoidant or impulsive.
```

This keeps the decision layer auditable while adding human imperfection.

## Integration

`DerivedCharacterState` now includes:

```text
embodiedAction
```

The current flow is:

```text
BeliefState
-> NeedDeficiency
-> DesireState
-> BehaviorBias
-> BehaviorDecision
-> EmbodiedAction
```

The Dashboard `DecisionPanel` now shows:

```text
原始意图
实际行动
动作噪声
误差风险
表现状态
reasons
```

## Serialization Fix

`SerializedCharacterPhysicsState` now includes:

```text
metaState
```

This prevents dynamic meta parameters such as self-control, resilience and emotional sensitivity from being lost during persistence and API roundtrips.

## Tests

New tests:

```text
tests/core/actionNoise.test.ts
```

Coverage:

```text
stable boundary + high self-control -> final action stays close to intention
high stress + low self-control -> final action becomes distorted
same intention + different body state -> different final execution
```

Updated tests:

```text
tests/core/derivedCharacterState.test.ts
tests/core/serialization.test.ts
```

## Current Boundary

V2.24 does not implement:

```text
procedural memory
habit strengthening
autonomous background scheduler
motor skill learning
random runtime nondeterminism
```

The randomness input is deterministic on purpose so replay, benchmark and tests remain auditable.

## Meaning

CharacterOS now has the first version of:

```text
Mind -> Decision -> Embodied Imperfection -> Final Action
```

This is a small but important step away from pure reasoning and toward a character who behaves like a living, limited, embodied person.

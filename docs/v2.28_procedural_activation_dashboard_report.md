# CharacterOS V2.28 Procedural Activation Dashboard Report

## Version

```text
CharacterOS V2.28 Procedural Activation Dashboard
```

## Goal

V2.28 makes Procedural Memory visible in the current character state.

The goal is to show that some behaviors are not reflective decisions.

They are automatic routines triggered by cues.

## Default Character State

The default Lin Fan state now includes procedural routines:

```text
routine_check_message
-> 反复查看手机消息。

routine_late_night_withdrawal
-> 在深夜变得沉默，减少表达真实需求。
```

## Derived State Integration

`DerivedCharacterState` now includes:

```text
proceduralActivations
```

The system extracts cue tags from recent memories using:

```text
memory.content
memory.beliefEffect
memory.emotion
memory.clusterId
```

Then it activates matching procedural routines.

## Dashboard Integration

`DerivedStatePanel` now includes:

```text
自动行为
```

It shows:

```text
action
activationScore
automaticity
cueMatch
reasons
```

## Design Rule

Procedural activation is not a BehaviorDecision.

It represents the low-level habit layer:

```text
cue -> routine -> automatic action
```

This allows the Dashboard to show both:

```text
reflective behavior tendency
automatic behavior activation
```

## Meaning

CharacterOS can now display:

```text
what the character wants to do
what the character decides to do
what the body may distort
what habit may automatically trigger
```

This deepens the internal model without adding multi-character simulation or story generation.

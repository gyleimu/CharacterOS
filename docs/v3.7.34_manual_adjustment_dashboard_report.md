# CharacterOS V3.7.34 Manual Adjustment Dashboard Report

## Summary

This update connects manual parameter adjustment apply and rollback to the Dashboard.

The controls are explicit user actions.

They do not run automatically during Tick.

## Updated Dashboard

```text
src/components/character/PhysicsDashboard.tsx
src/app/styles.css
```

New actions:

```text
应用人工参数补丁
回滚到快照
```

The apply button is enabled only when:

```text
parameterAdjustmentPatch.status === ready
parameterAdjustmentSnapshot.status === captured
```

The rollback button is enabled only when:

```text
parameterAdjustmentSnapshot.status === captured
```

## Updated Core Safety

```text
src/core/parameters/parameterAdjustmentApply.ts
```

Patch application now checks:

```text
patch.status === ready
snapshot.status === captured
snapshot values match patch from-values
current state still matches snapshot values
all target paths can be safely written
```

This prevents repeated apply clicks or stale patch application after the character has changed.

## API Usage

The Dashboard calls:

```text
POST /api/characters/[characterId]/physics/adjustment/apply
POST /api/characters/[characterId]/physics/adjustment/rollback
```

and updates local state from the response.

## Architecture Meaning

CharacterOS now has an end-to-end manual calibration path:

```text
observe
↓
accumulate
↓
draft
↓
preview
↓
audit
↓
patch plan
↓
snapshot
↓
manual apply
↓
manual rollback
```

The important boundary still holds:

Automatic systems can observe and propose.

Only explicit human action can persist parameter changes.

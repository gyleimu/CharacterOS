### Known Limitation: Path-Based Classification

The static scanner classifies findings by file path, not by function name. This means:

- **Risk**: If someone adds a builder function with `Date.now()` default inside an operational module (e.g., `src/core/editor/`), the static scan would classify it as "allowed" because the file is in the `editor` operational module.
- **Mitigation**: Replay tests provide defense-in-depth for specific builder functions. Any new builder that uses `Date.now()` defaults would likely fail replay tests.
- **Recommendation**: V13.5 should consider function-name-aware classification or AST-based scanning.

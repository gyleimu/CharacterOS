# V10.12: Differentiated Decision Pipeline Integration — Report

**Date:** 2026-06-24
**Phase:** V10.12 — Mainline Integration of Differentiation Engine
**Status:** ✓ Complete

---

## Objective

Integrate the V10.11 Character Differentiation Engine (`buildDifferentiatedDecision`) from benchmark-only usage into the CharacterOS mainline decision pipeline. `DifferentiatedDecision` is now a first-class output of `deriveCharacterState` and is exposed through the Decision API alongside the legacy `BehaviorDecision`.

No new benchmark cases. No parameter tuning for scores. This is pipeline integration only.

---

## Why V10.12 Is Mainline Integration, Not Benchmark Expansion

V10.11 proved the differentiation engine works (benchmark V2.1: Matrix 0.858, Focus 0.676, Core reason 0.597, Same-result 0.75, Continuity 0.68, Growth 5/5). But it was only callable from `outputs/run-benchmark-v2-1.ts` with hardcoded benchmark personas and environments.

V10.12 makes it a **mainline feature**:

- `deriveCharacterState(state)` now produces a `differentiatedDecision` from the character's own identity, memories, beliefs, and coordinate — not from benchmark fixtures
- The Decision API returns the full differentiated chain alongside legacy `BehaviorDecision`
- Legacy decision is preserved unchanged
- No benchmark cases added; no parameters tuned for scores

## Integration Points

### 1. New Adapter (`differentiationAdapter.ts`)

Bridges the gap between `CharacterPhysicsState` and the `PersonaSeed`/`EnvironmentSeed` that `buildDifferentiatedDecision` requires.

| Function | Purpose |
|---|---|
| `buildPersonaSeedFromState(state)` | Constructs PersonaSeed from identity, memories, coordinate traits, beliefs, metaState |
| `buildEnvironmentSeedFromState(state)` | Constructs EnvironmentSeed from boundary phase, stressLoad, recent memories |
| `buildDifferentiatedDecisionForState(state)` | Full pipeline: seed construction → schema activation → needs → desires → strategies → action surface |

All are pure, deterministic, no LLM, no state mutation.

**Fallback strategy for sparse state:**
- No identity → name="匿名角色" fallback, group="个体"
- No memories → traits-only coreExperience
- No beliefs → generic default belief
- No explicit environment → "安静日常" derived from boundary/meta

### 2. Modified `DerivedCharacterState`

```
+ differentiatedDecision?: DifferentiatedDecision
```

Added to the interface. Populated in `deriveCharacterState()` by calling the adapter. Legacy `decision: BehaviorDecision` is unchanged.

On adapter failure, `differentiatedDecision` is silently `undefined` — the differentiation engine is additive, not a dependency.

### 3. Modified Decision API Route

`GET /api/characters/[characterId]/decision` now returns:

```
+ differentiatedDecision    (full DifferentiatedDecision)
+ schemas                   (ActivatedSchema[])
+ needs                     (NeedProfile[])
+ desires                   (DesireProfile[])
+ selectedStrategy          (BehaviorStrategy)
+ actionSurface             (ActionSurface)
+ warnings                  (V10.12 notice)
```

All legacy fields preserved: `decision`, `derived`, `narrative`.

### 4. Updated Contract (`appContracts/characterPhysics.ts`)

Added `DecisionResponse` interface with differentiated decision fields. Re-exported differentiation types for downstream consumers.

## Known Limitations

- `DifferentiatedDecision` depends on `CharacterPhysicsState` having identity, memories, and beliefs — for a fresh/default state, the fallback may produce generic schemas (e.g., always safety_seeking dominant)
- `EnvironmentSeed` is constructed from internal state, not from an actual external event — the "environment" is the character's perception of their situation, not an objective world event
- No persistence of differentiated decisions (they are recomputed on every `deriveCharacterState` call)
- Benchmark persona/environment seeds are richer than auto-constructed seeds — the adapter produces reasonable but simpler seeds

## Objective

Build a controlled, deterministic, dry-run-first harness for observing how a single character changes over simulated time — without multi-character systems, world simulation, autonomous loops, or self-action execution.

This is NOT a daemon. It is a **projection tool**: "if this character lives quietly for N hours, in steps of M hours, what happens?"

---

## Why Single-Character Harness Before Multi-Character / World Sim

| This is | This is NOT |
|---|---|
| One character, observed over time | Multi-character interaction |
| Projection tool | World simulation |
| Explicit, callable function | Autonomous infinite loop |
| Dry-run first | Self-action execution |
| Deterministic seed chain | LLM narrative generation |

Before simulating a world, we must confirm that a single character's internal life is coherent over time — that fatigue cycles, sleep patterns, boredom waves, and decision shifts make sense across multiple consecutive ticks.

## Architecture

### Request

```typescript
LongitudinalSimulationRequest {
  characterId, totalHours, stepHours, seed,
  observed?, includeDecision?, includeExplanation?,
  commitPolicy?, lifeOptions?
}
```

- `totalHours`: total simulated time (validated > 0)
- `stepHours`: hours per tick (validated > 0)
- Max steps capped at 720 (prevents unbounded loops)
- `commitPolicy.enabled` defaults to `false` (dry-run only)

### Per-Step Execution

```
for each step:
  1. Snap compact state summary before
  2. runLifeTickDryRun(state, lifeReq, lifeOptions)
  3. buildLifeDecisionContextFromDryRun(dryRun)
  4. [optional] buildDifferentiatedDecisionForState(state, {lifeContext})
  5. [optional] explainDifferentiatedDecision(...)
  6. [optional] commitLifeTickProjection(state, dryRun, commitOpts) on CLONE
  7. Snap compact state summary after
```

Seed per step: `${baseSeed}:${stepIndex}:${absoluteHour}` — deterministic chain.

### Step Result

```typescript
LongitudinalSimulationStep {
  index, elapsedHours, absoluteHour,
  lifeDecisionContext,          // always present
  differentiatedDecision?,      // when includeDecision=true
  differentiatedExplanation?,   // when includeExplanation=true
  commitResult?,                // when commitPolicy.enabled=true
  stateSummaryBefore,           // compact
  stateSummaryAfter,            // compact
  warnings, reasons
}
```

### Aggregate Metrics

```typescript
AggregateMetrics {
  totalSteps, committedSteps, generatedMemoryCount,
  averageFatigue, averageBoredom,
  sleepPhaseCounts, strategyCounts, actionDirectionCounts,
  lifeInfluenceCount,
  personalityDeltaSummary,  // trust/fear/control/openness/conscientiousness delta
  beliefCountBefore/After, memoryCountBefore/After
}
```

### Compact State Summary

```typescript
CompactStateSummary {
  memoryCount, beliefCount,
  trust, fear, control, openness, conscientiousness,
  boundaryStress, boundaryIntegrity,
  metaResilience, metaSelfControl,
  latestMemorySnippet?, topBeliefSnippet?
}
```

No full state dumps. Compact, comparable across steps.

## Safety Boundaries

| Boundary | Enforcement |
|---|---|
| Dry-run default | `commitPolicy.enabled` defaults to `false` |
| No mutation on input state | Dry-run reads only; commit clones before writing |
| Self-action never executed | `commitSelfActionCandidates` is type-level `false` |
| Max steps cap | 720 |
| Memory cap | `maxGeneratedMemoriesPerStep` and `maxTotalGeneratedMemories` |
| No API calls | Pure core function |
| No LLM | Template-based decision/explanation only |
| Deterministic | Seeded chain per step |

## Known Limitations

- No persistent life state across simulation runs — each call starts fresh from the same CharacterPhysicsState
- Personality delta in dry-run is zero (no state mutation) — only visible when commit is enabled
- `stimulationLevel` and `socialContactLevel` are fixed per simulation run, not per step
- No API yet — core harness only; V10.18 can add API if needed

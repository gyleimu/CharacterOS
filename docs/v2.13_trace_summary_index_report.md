# CharacterOS V2.13 Trace Summary Index 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.13 Trace Summary Index
```

这一阶段继续推进 V2 Personality Galaxy replay 基础设施。V2.12 已经为单个 replay artifact 生成 summary，V2.13 将所有 summary 组织成统一 index，方便未来 Dashboard、Benchmark seed 和实验对照读取。

## 本阶段新增

### 1. TraceReplaySummaryIndex

更新：

```text
src/core/trace/traceReplaySummary.ts
```

新增：

```text
buildTraceReplaySummaryIndex(params)
```

输出：

```text
title
scenarioCount
summaries
directionCounts
```

其中 `directionCounts` 用于快速观察当前 replay 库中的心理轨道分布。

### 2. Demo 输出 Index

更新：

```text
src/core/demo/traceReplayDemo.ts
```

`npm run demo:trace` 现在会额外写出：

```text
outputs/trace_replay_summary_index.json
```

当前 index 统计结果：

```text
scenarioCount: 5
directionCounts:
  defensive_drift: 3
  recovery_drift: 2
```

### 3. Summary Index 测试

更新：

```text
tests/core/traceReplay.test.ts
```

新增验证：

```text
index 覆盖所有 traceReplayScenarios
index.summaries 数量等于 scenarioCount
directionCounts 包含 defensive_drift
directionCounts 包含 recovery_drift
index 包含 support_recovery_accumulation
```

## 当前意义

现在 replay 数据链路变成：

```text
TraceReplayScenario
-> TraceReplayArtifact
-> TraceReplaySummary
-> TraceReplaySummaryIndex
```

这让 CharacterOS V2 的实验轨道第一次具备“场景库总览”能力。

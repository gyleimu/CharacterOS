# CharacterOS V10.56 — Demo Strategy Basis Report

**Date:** 2026-06-28
**Status:** Scenario Explanation Polish Complete
**Scope:** Offline read-only static demo.

## Summary

V10.56 improves the scenario comparison table in the static CharacterOS demo.

V10.55 showed the primary schema for each scenario, but the primary schema can reflect the character's dominant personality background rather than the specific reason a strategy was selected. V10.56 adds `strategySchemas`, derived from `selectedStrategy.basedOnSchemas`, and displays it as `策略依据`.

## Why

For example, Lin Fan may still show `依恋威胁图式` as a strong background schema in an opportunity scenario, while the chosen strategy is actually driven by `机会收益图式`.

The demo now separates:

- `primarySchema`: the strongest activated schema.
- `strategySchemas`: the schema basis for the selected strategy.
- `primaryNeed`: the strongest need after schema activation.

## Updated UI

The scenario comparison table now uses:

| Column | Meaning |
|---|---|
| 场景 | Fixed probe title |
| 策略 | Selected strategy |
| 策略依据 | Schema basis for selected strategy |
| 主需求 | Strongest need |

## Test Guard

## Final Statement

V10.56 makes the scenario panel more honest: it shows both the character's strongest background activation and the specific schema basis behind each selected strategy.

## 1. Service Boundary vs HTTP Route

V11.8 implements the **service boundary**, not HTTP routes.

| Service Boundary | HTTP Route |
|-----------------|------------|
| `services/explorerService.ts` | `src/app/api/.../route.ts` |
| In-memory state (single-machine) | Network-exposed endpoint |
| Pure TypeScript | Next.js handler |
| Available now | Deferred to V11.9 |

HTTP routes may be added later; the service layer is already testable and functional.

## 2. Service Methods

| Method | Read/Write | Confirmation |
|--------|-----------|-------------|
| `getExplorerManifest(id)` | Read-only | No |
| `getCharacterStateSurface(id)` | Read-only | No |
| `previewEvent(id, draft)` | Read-only | No |
| `applyEvent(id, draft, preview, confirm, reason, actor)` | **Write** | **Yes** |
| `getExplainabilityTimeline(id)` | Read-only | No |
| `createTimeMachineSnapshot(id, label)` | Write (snapshot) | No |
| `getTimeMachineTimeline(id)` | Read-only | No |
| `restoreTimeMachineView(id, snapId)` | Read-only | No |
| `getMindGalaxyEmbed(id)` | Read-only | No |

8 of 9 methods are read-only. Only `applyEvent` requires confirmation.

## 3. Read/Write Boundary

```
Read-only (default):  manifest, state, preview, explain, timeline, restore, galaxy
Write (explicit):     applyEvent (confirmation + audit entry)
```

- `writeRequiresConfirmation: false` on all read-only responses
- `writeRequiresConfirmation: true` on apply response
- In-memory state store (single-character, single-machine)

## 4. Structured Errors

```typescript
ExplorerServiceError {
  code: string,       // e.g. "CONFIRMATION_REQUIRED"
  message: string,    // Human-readable
  severity: "info" | "warn" | "error",
  recoverable: boolean
}
```

- Missing confirmation → `CONFIRMATION_REQUIRED`, error, recoverable
- Apply blocked → `APPLY_BLOCKED`, warn, recoverable
- Snapshot not found → `SNAPSHOT_NOT_FOUND`, error, recoverable

## 5. How Raw State Exposure Is Prevented

- Service methods never return `CharacterPhysicsState`
- All data passes through Explorer DTO builders
- `CharacterStateSurface` uses qualitative bands only
- JSON serialization never includes internal IDs or engine internals
- Safety note on every response

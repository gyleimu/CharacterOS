## Full Pipeline Overview

```
LifeTickRequest
    │
    ▼
[1. Scheduler]        → LifeTickPlan (validate, timeScale, seed)
    │
    ▼
[2. Energy/Fatigue]   → projectedEnergyFatigue (read from CharacterPhysicsState)
    │
    ▼
[3. Sleep/Wake]       → projectedSleepWake (reads energy/fatigue projection)
    │
    ▼
[4. Dream]            → dreamFragments (reads sleep/wake + memory/belief sources)
    │
    ▼
[5. Boredom/Inspiration] → projectedBoredom + inspirationSeeds (reads energy + sleep)
    │
    ▼
[6. Random Thought]   → randomThought (reads dream + inspiration + boredom)
    │
    ▼
[7. Self-Action Candidates] → scored candidates (reads all prior projections)
    │
    ▼
[8. Summary]          → LifeTickTrace (8 phase traces, all reasons/warnings)
    │
    ▼
[9. Persistence]      → LifeTickCommitResult (explicit opt-in, clone, audit trail)
```

## Philosophy Compliance

### ✅ Single-character first
All 11 modules operate on a single `CharacterPhysicsState` reference. No multi-character semantics.

### ✅ No chatbot behavior
No dialogue generation. Random thoughts are monologue fragments, not conversations.

### ✅ No LLM
Zero LLM calls in the entire V10 pipeline. Dream descriptions, thought phrases, and inspiration triggers are template-based.

### ✅ No world simulation
No external events, no environment model. `stimulationLevel` and `socialContactLevel` are explicit inputs, not simulated world state.

### ✅ No autonomous external action
Self-action candidates are scored and ranked but never executed. `evaluated: false`, `executed: false` enforced at the type level.

### ✅ Continuous internal life only
All projections are internal — energy, sleep, boredom, thoughts, dreams, candidates. Nothing leaves the character.

### ✅ Explicit control
Every tick is explicitly requested. Every commit requires explicit per-domain opt-in. No daemon. No background loop.

## Determinism Audit

### Finding: FULLY DETERMINISTIC ✅

- **SeededRandom**: Single LCG implementation, no `Math.random()`. Verified via `grep` — only comment references to `Math.random` exist in the life module.
- **All subsystem ticks**: Pure functions of state + context. No hidden entropy sources.
- **Tie-breaking**: Uses seeded RNG only for equivalent-weight tone/kind selection.
- **ID generation**: Hash-based, deterministic from inputs.
- **`Date.now()`**: Used only in `lifeTickPersistence.ts` for memory seed timestamps (non-semantic metadata) and in `lifeTickRunner.ts` for `createdAt` trace field (non-semantic). These do not affect computational determinism.

### Verified:
```
same character + same request → identical dry-run result (all 7 projections match)
same dry-run + same options → identical commit result (changes + skipped match)
```

## Immutability Audit

### Finding: FULLY IMMUTABLE ✅

- **Dry-run**: `CharacterPhysicsState` is read-only. All subsystem ticks operate on projected state clones. Verified: `frozenStress` and `frozenMemories` unchanged after `runLifeTickDryRun`.
- **Subsystem functions**: Every `tick*`, `compute*`, and `apply*` function returns new objects. No mutation of inputs. Verified per-subsystem.
- **Persistence**: `commitLifeTickProjection` clones state before any modification. Original state untouched. Verified.
- **Memory preservation**: Existing memories are never overwritten. New memory seeds are appended only. Verified.

## Persistence Boundary Audit

### What IS written (when enabled)

| Source | Destination | Requires |
|---|---|---|
| Dream fragment | `memories[]` as seed | `allowDreamMemorySeed: true` |
| Inspiration seed | `memories[]` as seed | `allowInspirationSeed: true` |
| Random thought | `memories[]` as seed | `allowRandomThoughtMemorySeed: true` |
| Self-action candidate | `memories[]` as trace seed | `allowSelfActionCandidateMemorySeed: true` |

### What is SKIPPED (requires schema migration)

| Domain | Reason |
|---|---|
| Energy/fatigue state | No `lifeState` field on `CharacterPhysicsState` |
| Sleep/wake state | No `lifeState` field on `CharacterPhysicsState` |
| Boredom expansion state | No `lifeState` field on `CharacterPhysicsState` |

**This is intentional, not a missing feature.** V10.9 frames schema gaps as safety decisions — the system knows what it cannot safely write and reports why.

### What NEVER happens

- Self-action execution
- Memory overwrite
- Input state mutation
- Bulk/uncontrolled writes

## Known Limitations

| Limitation | Impact | Mitigation |
|---|---|---|
| No `lifeState` field on `CharacterPhysicsState` | Energy/sleep/boredom projections cannot persist | Documented; schema migration deferred |
| `stimulationLevel`/`socialContactLevel` are fixed defaults | No world-model driven variation | Explicit inputs, can be provided by caller |
| Each tick starts from defaults | No persistent life state across ticks | Caller can feed prior projected state back |
| Memory seeds are low-importance (~0.1–0.2) | Negligible impact on memory pool | Intentional — life seeds shouldn't dominate |
| No `attention_drift`/`quiet_drift` independent modeling | Phase mapping is approximate | Deferred to future drift subsystem |

# CharacterOS V3.7.51 Blueprint Reset Lifecycle API Report

## Goal

V3.7.50 created a formal Character Blueprint for Lin Fan.

This update connects that blueprint to the character lifecycle API.

The goal is to make the single-character foundation operational without adding frontend UI.

## Service Change

Updated:

```text
src/services/characterPhysicsService.ts
```

`resetCharacter` now accepts:

```ts
{
  seedInitialExperiences?: boolean
}
```

Default behavior remains unchanged:

```text
reset -> Lin Fan blueprint state
memories: []
clusters: []
```

Explicit seeded reset:

```text
reset with seedInitialExperiences
-> Lin Fan blueprint
-> origin experiences processed through CharacterPhysicsEngine
-> MemoryNode
-> ImpactCluster
-> BeliefState
```

## API Change

Updated:

```text
src/app/api/characters/[characterId]/physics/route.ts
```

Default reset:

```http
DELETE /api/characters/[characterId]/physics
```

Seeded reset:

```http
DELETE /api/characters/[characterId]/physics?seedInitialExperiences=true
```

This gives the operator two clean starting modes:

```text
blank character seed
experience-seeded character seed
```

## Compatibility

Existing behavior is preserved.

The default reset still creates an empty Memory Galaxy, so existing tests and API users do not suddenly receive preloaded memories.

Origin experiences are only applied when explicitly requested.

## Architecture Meaning

CharacterOS now has a clearer single-character lifecycle:

```text
CharacterBlueprint
-> blank reset
or
-> experience-seeded reset
-> event processing
-> continuous tick
-> manual adjustment
-> export/import
```

This is a small but important foundation step.

The project can now start a character either from a clean psychological seed or from a seed already shaped by origin experiences, without relying on frontend tools.

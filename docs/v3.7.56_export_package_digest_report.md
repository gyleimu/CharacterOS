# CharacterOS V3.7.56 Export Package Digest Report

## Goal

State integrity snapshots tell us whether a character state was coherent at export time.

This update adds a package-level digest so CharacterOS can detect when a package's content changed after export.

This is not a security signature.

It is an audit checksum for package consistency.

## Added

```text
src/core/export/characterExportPackageDigest.ts
```

It provides:

```text
computeCharacterExportPackageDigest
compareCharacterExportPackageDigest
```

The digest uses:

```text
algorithm: sha256
canonicalization: characteros-json-v1
excludedTopLevelFields: ["packageDigest"]
```

`packageDigest` is excluded from its own digest to avoid self-reference.

## Import Change

Updated:

```text
src/core/export/characterImportPlan.ts
```

`CharacterImportPlan` now includes:

```text
packageDigestComparison
```

Comparison status:

```text
missing
matched
mismatch
```

Behavior:

```text
missing  -> old package compatibility
matched  -> package content matches embedded digest
mismatch -> high-risk needs_review
```

Confirmed import cannot bypass a digest mismatch because apply authorization depends on the import plan.

CharacterOS export/import now has three audit layers:

```text

```

This makes package movement safer before the project grows into richer editing, migration, benchmark and long-running simulation workflows.

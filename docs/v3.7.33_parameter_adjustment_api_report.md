# CharacterOS V3.7.33 Parameter Adjustment API Report

## Summary

This update exposes explicit parameter adjustment apply and rollback APIs.

The APIs are manual-only.

They do not run during Tick.
They do not auto-apply Dashboard suggestions.
They require the caller to provide a patch and/or snapshot.

## Updated Contracts

```text
src/appContracts/characterPhysics.ts
```

New request and response types:

```text
ApplyParameterAdjustmentRequest
ApplyParameterAdjustmentResponse
RollbackParameterAdjustmentRequest
RollbackParameterAdjustmentResponse
```

## Updated Service

```text
src/services/characterPhysicsService.ts
src/services/characterPhysicsDto.ts
```

New service methods:

```text
applyParameterAdjustment()
rollbackParameterAdjustment()
```

State is persisted only when the core apply trace returns:

```text
status === applied
```

Blocked results do not update the repository.

## New API Routes

```text
POST /api/characters/[characterId]/physics/adjustment/apply
POST /api/characters/[characterId]/physics/adjustment/rollback
```

Apply requires:

```text
patch
snapshot
```

Rollback requires:

```text
snapshot
```

## Safety Rule

The API does not generate patches by itself.

The caller must explicitly send the reviewed patch and snapshot.

This preserves the manual calibration boundary:

```text
system suggests
human reviews
human sends explicit apply request
```

## Architecture Meaning

CharacterOS now has a manual parameter adjustment API.

This is the first point where calibration can actually persist, but only through an explicit request.

The engine still does not modify personality automatically.

The boundary remains clear:

```text
automatic systems observe and propose
manual systems confirm and apply
```

## 1. Problem Statement

V10.71 暴露两个真实 WARN：

1. **Galaxy force 不自然饱和**：集群 mass 线性增长 → 后期事件力反而更大（force 0.141→0.350→capped）。不符合"重复经历逐渐适应 / 边际影响下降"的心理物理直觉。

2. **Trust 修复极慢**：5 次 positive support 后 trust 变动 < 0.001。支持事件修复 boundary stress 但几乎无法修复 trust/openness。

## 2. Root Cause Analysis

### 2.1 Galaxy Force Growth

`calculateClusterForce` 中 `rawMagnitude ∝ mass`。重复事件使 cluster mass 线性增长（5.37→24.42），导致 force 加速增长直至 hit maxMagnitude cap（0.35）。

```
Step 1: mass=5.37,  force=0.141
Step 2: mass=8.83,  force=0.230 (+63%)
Step 3: mass=13.16, force=0.340 (+48%)
Step 4: mass=18.36, force=0.350 (capped)
Step 5: mass=24.42, force=0.350 (capped)
```

后期事件的边际影响力反而更大，违背 diminishing returns 原则。

### 2.2 Trust Repair Weakness

Two compounding factors:
1. Support trust delta (+0.06) 比 abandonment (-0.09) 小 50%
2. Seeded abandonment cluster 产生持久 counter-force
3. Learning rate (0.02-0.03) 使 galaxy force 的 personality 影响非常缓慢

即便 5 次 support 后 support cluster mass (14.2) 远超 abandonment (2.77)，net force 仍接近零。

## 3. Fix 1: Galaxy Force Sqrt Saturation

**File:** `src/core/galaxy/potentialField.ts`

### 3.1 Mechanism

引入 sqrt mass saturation：

```typescript
const MASS_SATURATION_REF = 5;
export function saturatedMass(mass: number): number {
  return Math.sqrt(Math.max(0, mass) * MASS_SATURATION_REF);
}
```

Force 使用 `saturatedMass(mass)` 替代 raw `mass`：

```
rawMagnitude = G * saturatedMass(mass) * stability * densityFactor / distance²
```

### 3.2 Effect

| Step | Mass | Old Force | New Force | Marginal Δ |
|------|------|-----------|-----------|------------|
| 1 | 5.37 | 0.141 | 0.136 | — |
| 2 | 8.83 | 0.230 | 0.174 | +0.038 |
| 3 | 13.16 | 0.340 | 0.211 | +0.037 |
| 4 | 18.36 | 0.350 | 0.247 | +0.036 |
| 5 | 24.42 | 0.350 | 0.282 | +0.035 |

- Force 不再 hit cap
- 边际增量从 +0.089→0.010 变为 +0.038→0.035（略微递减）
- Later events contribute diminishing marginal force ✓

### 4.2 Boundary Safety → Trust Repair Nudge

**Files:** `src/core/boundary/psychologicalBoundary.ts`, `src/core/physics/physicsEngine.ts`

当 boundary stress 降低（正向事件），产生一次性 trust/fear/openness repair nudge：

```typescript
repairNudge = {
  trust: safetySignal * max(0, 0.85 - trust) * 0.012,   // 低信任→更多修复空间
  fear:  -safetySignal * fear * 0.008,                    // 高恐惧→更多恐惧降低
  openness: safetySignal * max(0, 0.7 - openness) * 0.006,
}
```

Nudge 直接应用到 coordinate（一次性 offset，不 persist 为 velocity），避免跨事件污染。

### 4.3 Trust Repair Effect

| Baseline | Trust Before | After 5 Support | Δ |
|----------|-------------|-----------------|---|
| Low-trust (Lin Fan) | 0.265 | 0.297 | **+0.032** |
| High-trust (secure) | 0.78 | 0.785 | **+0.005** |

- Low-trust baseline 获得更多修复（更多 deficit room）✓
- High-trust baseline 接近饱和，增长有限 ✓
- 单次 support 不翻转人格（step-1 trust Δ ≈ 0.006）✓
- 5 次 support 产生可见趋势 ✓

## 1. Problem Statement

V10.69 Reality Audit 发现 `audit_counterfactual_positive_event` 产生 WARN：

```
boundaryDelta over-responded: actual 0.4127 > expected 0.1443
```

正向支持事件（support）导致边界压力变化过大，校准模型期望 support 事件对 boundary 产生很低的影响（factor 0.14），但实际物理层产生的 boundaryDelta 接近 abandonment 事件的量级。

## 2. Root Cause Analysis

### 2.1 问题链路追踪

```
Event Input (support, "王雪主动解释...")
  → parseExperienceEvent() → ParsedExperienceEvent (category="support", intensity=0.55, importance=0.78)
  → CharacterPhysicsEngine.processEvent()
    → calculateEventImpactScore() → ImpactScore { value: 0.6725 }
    → applyEventBoundaryImpact()
      → applyBoundaryImpact()  ← ROOT CAUSE
        incomingStress = impactScore.value * (0.55 + vulnerability + relationshipPressure) * (1 - resilience * 0.42)
                        = 0.6725 * (0.55 + 0.57 + 0.2025) * (1 - 0.36 * 0.42)
                        ≈ 0.917  ← 高正向压力
    → diffBoundary() → boundaryDelta.stressLoad = +0.917
    → runImpactCalibrationAudit()
      → measureActualDeltas() → boundaryDelta = 0.917 * 0.45 = 0.4127
      → allocateChannelImpact() → expectedMax for support = 0.1443
      → findOverResponses() → 0.4127 > 0.1443 + 0.05 → WARN
```

### 2.2 根因

**`applyBoundaryImpact()` in `psychologicalBoundary.ts:68-72`**

该函数对所有事件类别使用相同的公式计算 `incomingStress`，总是累加正向压力：

```typescript
// 旧代码：所有事件统一处理
const incomingStress = round4(
    params.impactScore.value *
    (0.55 + vulnerability + relationshipPressure) *
    (1 - params.boundary.resilience * 0.42)
);
const stressLoad = round4(params.boundary.stressLoad + incomingStress); // 总是增加
```

对于 support/success 事件，impactScore.value 可能很高（0.67+），导致 incomingStress 接近 abandonment 事件水平（~0.9），这违背了以下设计原则：
- 正向支持事件应降低边界压力（作为"安全信号"）
- 正向支持事件不应在单次事件中造成大幅 boundary 摆动
- support 应逐步降低防御、增加开放性

**校准模型本身是正确的** — `allocateChannelImpact()` 已正确地为 support 事件分配了较低的 boundaryDelta 预期（factor 0.14 vs 0.58），问题在于物理层未区分事件效价。

## 3. Fix: Core Physics Repair

### 3.1 修复文件

**`src/core/boundary/psychologicalBoundary.ts` — `applyBoundaryImpact()`**

### 3.2 修复逻辑

引入事件类别感知的 boundary 影响计算：

- **正向事件 (support / success)：**
  - 计算 `safetySignal`：基于 impactScore 和当前压力比率
  - `incomingStress` 为**负值**（减轻压力）
  - `stressLoad` 降低（而非增加）
  - 不产生 `overflowAmount`（不会溢出）
  - `cracks` 可能愈合（减少而非增加）
  - `integrity` 缓慢提升（安全证据累积）

- **负向/中性事件 (abandonment / betrayal / general)：**
  - 保持原有行为不变
  - `incomingStress` 仍为正值
  - `stressLoad` 增加、可能溢出、产生裂纹

### 3.3 关键代码变更

```typescript
if (isPositiveSupport) {
    // V10.70: support/success 作为"安全证据"降低边界压力
    const strainRatio = params.boundary.stressLoad / Math.max(0.01, params.boundary.capacity);
    const safetySignal = round4(
        params.impactScore.value *
        Math.min(strainRatio, 1.8) *
        0.28
    );
    incomingStress = round4(-safetySignal * (0.7 + params.boundary.resilience * 0.4));
    stressLoad = round4(Math.max(0, params.boundary.stressLoad + incomingStress));
    overflowAmount = 0;
    cracks = round4(Math.max(0, params.boundary.cracks - safetySignal * 0.15));
    integrity = clamp01(params.boundary.integrity + safetySignal * 0.07);
} else {
    // 负向/中性事件：保持原有逻辑
    // ... (unchanged)
}
```

## 2. D10 关键设计

```text
Phase 3 reward recovery 是一个 intermediate step。
Phase 4 homeostasis 通过 regulatedRewardState 覆盖 Phase 3 的结果。

V5.5 的 reward_recovery subProcess trace 记录了：
  - recoverRewardBaseline() 的输入/输出
  - NOT 最终的 tick reward state

最终 state.rewardState = homeostasisResult.trace.regulatedRewardState
（D10-3 关系在 V4.12 已验证，V5.5 保持不变）

V5.4-V5.5 共享相同 D10 语义：
  - boundary_recovery: Phase 3 recoverBoundary → Phase 4 regulatedBoundary
  - reward_recovery:   Phase 3 recoverRewardBaseline → Phase 4 regulatedRewardState

测试验证：
  ✓ Phase 3 subProcess metrics 反映 recoverRewardBaseline intermediate
  ✓ D10: final rewardState === regulatedRewardState（覆盖发生）
  ✓ D10: subProcess 不干扰 overwrite semantics
```

## 3. 这是 Instrumentation，不是 Delegation

```text
V5.5 与前三步 subprocess 遵循相同原则：

  - recoverRewardBaseline() 调用仍在 runContinuousTick Phase 3 中 inline 执行
  - buildRewardRecoverySubProcessTrace() 是纯 trace builder，只读不写
  - 不改变任何 mutation 的位置、顺序或输入/输出
  - homeostasis overwrite semantics (D10) 完全不变
```

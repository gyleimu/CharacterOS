## 2. 这是 Instrumentation，不是 Delegation

```text
V5.3 procedural decay subprocess 与 V5.2 memory decay subprocess 遵循相同原则：

  - decayProceduralRoutine() 调用仍在 runContinuousTick Phase 3 中 inline 执行
  - buildProceduralDecaySubProcessTrace() 是纯 trace builder，只读不写
  - 不改变任何 mutation 的位置、顺序或输入/输出
```

| 属性 | Delegation (V4) | Instrumentation (V5.2-V5.3) |
|------|-----------------|----------------------------|
| mutation 位置 | 移到 adapter | 保持在 runContinuousTick |
| 调用方式 | 通过 adapter | 直接调用 V3 函数 |
| trace 来源 | adapter 内部 | 纯 builder 函数 |
| 行为影响 | 必须保持等价 | 零影响 |

## 5. V5.4 前瞻：Boundary Recovery

```text
V5.4 和 V5.5 需要注意 D10 overlay 语义：

Phase 4 homeostasis 会覆盖 Phase 3 的 boundary 和 reward 结果：
  - state.boundary = homeostasisResult.trace.regulatedBoundary
  - state.rewardState = homeostasisResult.trace.regulatedRewardState

这意味着 boundary_recovery 和 reward_recovery 的 subProcess trace
需要记录 recover 函数的 pre/post，而不是 Phase 3 最终写入值。
Phase 3 最终写入的 boundary/reward 值会被 Phase 4 覆盖。

memory_decay 和 procedural_decay 不受此影响，
因为 Phase 4 不覆盖 memories 或 proceduralRoutines。
```

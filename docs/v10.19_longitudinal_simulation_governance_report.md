## 3. Findings and Fixes

### Finding 1: Step Cap Warning Never Fired (MEDIUM)
**File**: `src/core/life/longitudinalSimulation.ts:194`
**Issue**: `totalSteps` was computed as `Math.min(rawSteps, MAX_STEPS)`, but the warning condition `if (totalSteps > MAX_STEPS)` could never be true because `totalSteps` was already capped.
**Fix**: Split into `rawSteps` and `totalSteps`. Warning now fires when `rawSteps > MAX_STEPS`.
**Impact**: Previously silent cap — now correctly warns when steps are truncated.

### Finding 2: Unknown Character Auto-Created Instead of 404 (HIGH)
**File**: `src/app/api/characters/[characterId]/life/simulate/route.ts:16`
**Issue**: `characterPhysicsService.getState()` auto-creates a default LinFan character when the characterId doesn't exist. This meant unknown characters always returned 200 with a simulation on the default blueprint, never 404.
**Fix**:
1. Added `hasCharacter(characterId: string): boolean` to `CharacterPhysicsService` interface and `InMemoryCharacterPhysicsService` implementation.
2. Added `hasCharacter()` check in route before `getState()`, returning 404 for unknown characters.
**Impact**: API now correctly returns 404 for unknown characters. Governance boundary established: explicit character creation is required before simulation.

### Finding 3: No Upper Bound Validation on totalHours/stepHours (MEDIUM)
**File**: `src/app/api/characters/[characterId]/life/simulate/route.ts`
**Issue**: Route only validated `<= 0` for totalHours and stepHours. No upper bound meant a user could request `totalHours: 999999, stepHours: 1` and get a confusing 720-step result silently.
**Fix**:
1. Added `MAX_TOTAL_HOURS = 720 * 24` (720 steps × max 24h per step).
2. Added validation: `totalHours > MAX_TOTAL_HOURS` → 422 with clear error message.
3. Added validation: `stepHours > 24` → 422.
**Impact**: Excessive requests are now clearly rejected with helpful messages.

## 5. Core Harness Governance

### Dry-Run Default Guarantee

- `LongitudinalCommitPolicy.enabled` defaults to `false` (via `?? false`).
- When disabled, `commitLifeTickProjection` is never called.
- `runLifeTickDryRun` is explicitly read-only: "NO state is ever mutated."
- Tests confirm: original state memory count, coordinate values, and boundary metrics are unchanged after dry-run.

### Determinism Policy

- Seed chain: `${baseSeed}:${index}:${absoluteHour}` — pure function of inputs.
- Same seed + same state → identical `lifeDecisionContext`, `differentiatedDecision`, aggregate metrics.
- API seed fallback: `${characterId}:${totalHours}:${stepHours}` — deterministic.
- **Known limitation**: `new Date().toISOString()` in internal trace metadata (not surfaced in API response). Core simulation logic is deterministic.

### Self-Action Safety

- `commitSelfActionCandidates` type is `?: false` — enforced at TypeScript level.
- `allowSelfActionCandidateMemorySeed` hardcoded to `false` in harness.
- `commitLifeTickProjection` validates: `c.evaluated !== false || c.executed !== false` triggers warning.
- Self-action candidates are signals only — never executed, never persisted as actions.

### Commit Deferred Rationale

- `commitLifeTickProjection` operates on a cloned state. The result includes the clone but the route never writes it back to the service repository.
- The route explicitly adds: `"V10.18: Commit is deferred..."` warning.
- Persistence is blocked until `finalStateForCommit` is explicitly designed — a future core enhancement, not V10.19 scope.

### Caps and Bounds

| Cap | Value | Enforcement |
|-----|-------|-------------|
| Max steps | 720 | `Math.min(rawSteps, 720)` |
| Max generated memories per step | 2 (default) | `commitPolicy.maxGeneratedMemoriesPerStep ?? 2` |
| Max total generated memories | `totalSteps * maxPerStep` | Checked per step against remaining budget |
| Max totalHours (API) | 17280 (720×24) | 422 rejection |
| Max stepHours (API) | 24 | 422 rejection |
| Memory snippet | 80 chars | `.slice(0, 80)` |
| Belief snippet | 80 chars | `.slice(0, 80)` |

## 10. Known Limitations

1. **Non-deterministic metadata**: `new Date().toISOString()` in `LifeTickTrace.createdAt` and `MemoryNode.timeStamp`. These are internal trace metadata, not surfaced in the API response. Core simulation logic is fully deterministic.
2. **Commit deferred**: `commitPolicy.enabled=true` runs the commit on an internal clone but does not persist to the service repository. This requires `finalStateForCommit` design in a future version.
3. **No `commitStatus` field**: The response communicates commit deferral via warning strings, not a structured field. A future contract enhancement should add `commitStatus: "deferred" | "applied"`.
4. **Auto-create pattern in other routes**: Several other API routes (`editor/apply`, `editor/preview`, `graph`, etc.) also call `getState()` which auto-creates default characters. The `hasCharacter()` gate was only applied to the simulate route in V10.19.

## 12. Governance Statements

1. **Longitudinal simulation is observation, not autonomous life loop.** Each invocation is a bounded, explicit request. No timers, no daemons, no recurring processes.

2. **API requests are explicit and bounded.** Every simulation has a declared `totalHours`, `stepHours`, and optional `seed`. Excessive requests are rejected at the API boundary.

3. **Self-action candidates are signals only.** The system generates action tendency observations but never executes them. This is enforced at the type level (`commitSelfActionCandidates?: false`) and runtime (`allowSelfActionCandidateMemorySeed: false`).

4. **Commit persistence remains deferred** until `finalStateForCommit` is explicitly designed. The current harness computes what WOULD be committed on an internal clone but never persists to the service repository.

5. **Single-character philosophy remains protected.** The simulation operates on one character state at a time. No cross-character dependencies, no multi-agent interactions, no world simulation.

6. **No multi-character/world simulation in V10.19.** This remains a single-character observational harness.

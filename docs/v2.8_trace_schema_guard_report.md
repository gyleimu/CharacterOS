# CharacterOS V2.8 Trace Schema Guard 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.8 Trace Schema Guard
```

这一阶段继续收敛 V2 Personality Galaxy 的 replay 基础设施：trace replay artifact 不再只是生成 JSON，而是在写出前具备运行时结构校验。

## 本阶段新增

### 1. Replay Artifact Runtime Guard

更新：

```text
src/core/trace/traceReplay.ts
```

新增：

```text
validateTraceReplayArtifact(value)
```

校验范围包括：

```text
schemaVersion
scenario
scenarioMeta
parameters
initialCoordinate
finalCoordinate
steps
boundary
force
velocity
coordinate
clusterForces
clusterMetrics
```

### 2. Trace Demo 写入前校验

更新：

```text
src/core/demo/traceReplayDemo.ts
```

`npm run demo:trace` 现在会先校验 artifact，再写入：

```text
outputs/trace_replay_abandonment_then_repair.json
outputs/trace_replay_betrayal_spiral.json
outputs/trace_replay_success_recovery.json
```

如果 artifact 结构异常，demo 会直接抛错，避免产生不可用的 replay seed。

### 3. 测试覆盖

更新：

```text
tests/core/traceReplay.test.ts
```

新增验证：

```text
合法 artifact 可以通过 validateTraceReplayArtifact
错误 schemaVersion 会被拒绝
破损 step trace 会被拒绝
```

## 验证结果


## 当前意义

V2 trace replay 已经完成从：

```text
可生成
```

升级到：

```text
可版本化
可解释
可校验
可作为未来 Benchmark seed 的原始材料
```

这一步仍然不实现 V5 Benchmark，只是在 V2 内核阶段为未来 Benchmark 留下稳定数据入口。

## 下一步建议

继续 V2，不跳到 V3/V4/V5：

```text
1. 增加 replay artifact snapshots
2. 将 outputs trace replay 迁移到 fixtures 或 benchmark seeds 草案目录
3. 增加 cluster variance / density 的边界测试
4. 增加更长序列的 repeated-event replay，用来观察 mass 与 momentum 的长期演化
```

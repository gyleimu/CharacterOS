# V12.3: Event Candidate Extractor Report

**Date:** 2026-07-02
**Version:** V12.3.0
**Previous:** V12.2 (Agent Input Adapter)

## 1. Module

`src/core/agent/eventCandidateExtractor.ts` — deterministic event extraction from `AgentTurnInput`

## 2. Extraction Pipeline

```
AgentTurnInput → hasEventSignal? → buildDraft → scoreConfidence → classifySafety → filter → AgentEventCandidate[]
```

## 3. Per-Mode Behavior

| Input Mode | Extraction | Confidence |
|-----------|-----------|------------|
| `chat` | 1 candidate if event signal detected | Moderate–High |
| `journal` | 1 candidate from entry | Moderate |
| `story` | 1 candidate (flagged: fictional) | Lower |
| `plugin` | Conservative — only if clear event | Lower |
| `tool` | Conservative — only if clear event | Lower |

## 4. Event Signal Detection

20 Chinese event keywords: 发生, 说, 问, 告诉, 来到, 离开, 回复, 解释, 陪伴, 失联, 等待, 抛弃, 背叛, 成功, 失败, 拒绝, 冲突, 支持, 改变, 发现, 感到, 决定, 终于, 突然, 开始, 结束

## 5. Confidence Scoring

| Factor | Effect |
|--------|--------|
| Content > 50 chars | +0.15 |
| Content < 10 chars | -0.20 |
| Location present | +0.10 |
| People present | +0.10 |
| Time present | +0.05 |
| Emotion keywords | +0.10 |
| Base | 0.50 |

## 6. Safety Flags

| Flag | Trigger |
|------|---------|
| `fictional_or_story_input` | story input mode |
| `plugin_or_tool_input` | plugin/tool input mode |
| `low_confidence` | confidence < 0.4 |
| `possible_diagnosis_claim` | medical/diagnosis keywords |
| `possible_multi_character_relationship` | relationship pattern keywords |
| `sensitive_personal_data` | PII patterns |
| `requires_user_confirmation` | consentForWriteback=false |

## 7. What Extraction Is NOT

- ❌ Not event preview (no state clone, no simulation)
- ❌ Not event apply (no writeback)
- ❌ Not policy decision (no block/allow logic)
- ❌ Not LLM-based extraction (deterministic keywords only)

CharacterOS V2.29 Continuous Tick Route Guard

## Goal

V2.29 adds route-level protection for the Continuous Tick API.

The purpose is to ensure the external time-passing entry point stays stable.

## Covered Route

```text
POST /api/characters/[characterId]/physics/tick
```

## New Test

```text
tests/app/continuousTickRoute.test.ts
```

The test verifies:

```text
route returns 200
response characterId matches request
trace.daysElapsed matches request body
state includes procedural routines
trace procedural routine count matches state
average procedural strength decays after tick
```

## Why This Matters

Continuous Tick is no longer only a core function.

It is now:

```text
core
-> service
-> DTO
-> API
-> Dashboard
```

Route-level tests protect the final external interface.

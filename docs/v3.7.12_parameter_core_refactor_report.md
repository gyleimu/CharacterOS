# CharacterOS V3.7.12 Parameter Core Refactor Report

## Summary

This update turns the new Parameter Evolution documentation into a small TypeScript foundation.

The goal was not to add more behavior.

The goal was to reduce duplicated low-level math and make parameter evolution, inertia, accumulation, and recovery available as reusable core primitives.

```text

```

## Recovery Foundation

The new recovery curve module provides:

```text
applyRecoveryCurve
```

It models recovery as movement toward baseline shaped by:

```text
elapsed time
safety factor
obstacle factor
scar retention
```

This supports the principle:

```text
Recovery is not erasure.
Recovery is one way change becomes livable.
```

## Intentional Non-Changes

`src/core/event/eventParser.ts` still keeps its local two-decimal clamp.

Reason:

```text
event parsing currently uses coarse two-decimal output
changing it would alter parser behavior without architectural benefit
```

`src/core/personality/coordinate.ts` still exports `clip01` and `round4`.

Reason:

```text
existing modules may depend on those names
the functions now delegate to parameterMath internally
```

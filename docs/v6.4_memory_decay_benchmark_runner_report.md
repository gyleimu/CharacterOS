## 1. 新增文件

### `src/core/benchmark/benchmarkRunner.ts`（310 行）

**第一个真实 benchmark runner。执行 memory_decay fixture 但不执行其他 categories。**

#### Exported functions

| 函数 | 用途 |
|------|------|
| `runBenchmarkCase({kase})` | 执行单个 benchmark case |
| `runBenchmarkCases({cases})` | 执行多个 benchmark cases，汇总统计 |

#### Supported categories

| Category | Status | Runner |
|----------|--------|--------|
| `memory_decay` | ✓ supported | `runMemoryDecayCase` — 创建 Lin Fan state, 处理 events, runContinuousTick, 收集 metrics, 评估 assertions |
| `event_impact` | ✗ skipped | returns error with "not supported" |
| `personality_drift` | ✗ skipped | returns error with "not supported" |
| `homeostasis_recovery` | ✗ skipped | returns error with "not supported" |
| `belief_evolution` | ✗ skipped | returns error with "not supported" |
| `behavior_decision` | ✗ skipped | returns error with "not supported" |

#### Memory decay runner flow

```
1. createCharacterStateFromBlueprint(createLinFanBlueprint(), { seedInitialExperiences: true })
2. CharacterPhysicsEngine.processEvent(state, event)  ← process benchmark events
3. Record memoryCountInitial
4. For each tick:
   a. runContinuousTick(state, { daysElapsed })
   b. Collect subprocess metrics into series arrays
5. Map each expected direction to trace/subprocess values:
   - continuousTick.averageMemoryRecency  → finalTrace.averageMemoryRecencyBefore/After
   - continuousTick.averageMemoryWeight   → finalTrace.averageMemoryWeightBefore/After
   - continuousTick.memoryCount           → memoryCountInitial / finalTrace.memoryCount
   - memoryDecay.averageRecencyAfter      → subProcess metrics (with monotonic series)
6. evaluateBenchmarkAssertion() for each expected direction
7. Build result: verdict, assertionResults, metrics, warnings, explanation
```

## 2. 为什么先从 memory decay 开始

```text
1. V5 memory_decay subprocess 提供了完整的 pre/post metrics
    - averageRecencyBefore/After
    - averageEffectiveWeightBefore/After
    - memoryCount
    → 数据源清晰，无需从 internal state field 提取

2. Memory decay 是纯时间驱动过程
    - 不需要事件处理（event → impact → coordinate pipeline）
    - 不需要多角色对比
    - 不需要复杂 assertion（只需 directional + bounded + monotonic）

3. Memory decay 的 benchmark value 最高
    - 可以直接测量 "memory decay 对 personality drift 的独立贡献"
    - monotonic 衰减是可证伪的 ("应该单调递减")
    - 任何 regression 都会立即表现为 monotonic failure

4. 验证了 V6 架构的正确性
    - fixtures + assertion engine + runner 的 pipeline 可行
    - 真实 physics 执行 + 自动 assertion 评估 = 可复现 benchmark
```

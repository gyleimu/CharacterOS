# CharacterOS V3.7.45 Import Validation API Report

## Summary

This update adds a read-only import validation API.

It does not import, overwrite, or mutate a character.

It only checks whether a submitted CharacterOS export package is valid and returns a compact summary.

## Updated Core

```text
src/core/export/characterExportValidation.ts
```

New summary type and function:

```text
CharacterExportPackageSummary
summarizeCharacterExportPackage()
```

The summary includes:

```text
source character id
export version
export timestamp
memory count
cluster count
particle count
manual adjustment count
governance override count
stability risk
governance recommendation
```

## Updated API Contract

```text
src/appContracts/characterPhysics.ts
```

New request/response contracts:

```text
ValidateCharacterImportRequest
ValidateCharacterImportResponse
```

The response always includes:

```text
mutatesState: false
```

This makes the safety boundary explicit.

## New Route

```text
POST /api/characters/[characterId]/import/validate
```

Behavior:

```text
valid package   -> 200
invalid package -> 422
```

The route returns validation errors and a package summary when valid.

## Architecture Meaning

CharacterOS now has the first half of an import pipeline:

```text
export package
↓
validation
↓
summary
↓
future explicit import decision
```

This keeps import safe by design.

The system can inspect a character artifact before it is allowed to affect live character state.

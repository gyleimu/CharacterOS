# V10.60 CharacterOS Demo Scenario Mini Flow + Density Report

**Date:** 2026-06-28
**Version:** 10.60.0
**Previous:** 10.59.0 (Decision Flow + Boundary Gauge + Energy/Fatigue Bars)

## Overview

V10.60 adds scenario-card mini-flows, intensity values to the decision flow, and a comprehensive desktop density polish for 1280px viewports.

## Changes

### 1. Scenario Card Mini-Flow (New)

Each scenario card now includes a compact horizontal flow diagram.

**Implementation details:**
- New `scenarioMiniFlow(s)` function renders: **Schema Basis → Need → Strategy → Action**
- Data sourced from real scenario fields: `s.strategySchemas`, `s.primaryNeed`, `s.strategy`, `s.action`
- Action text truncated at 28 chars to prevent card overflow
- Labeled "场景决策小链" with a muted section-kicker style label
- Rendered in a compact `.mini-flow` container with `.mf-item` nodes and `.mf-arrow` connectors
- Each node shows a tiny category em (`Schema Basis`, `Need`, `Strategy`, `Action`) + the value
- The existing full schemas and needs lists are visually de-emphasized inside `.scenario-detail-mini` with a subtle top border separator

**User value:** Reviewer can see each scenario's decision chain at a glance without scrolling through the full schemas/needs lists.

### 2. Decision Flow Intensity Values (Enhanced)

The V10.59 decision flow diagram now shows intensity values for schema/need/desire nodes.

**Data layer additions:**
- `decision.differentiated.topSchemaIntensity: number` (0-1)
- `decision.differentiated.topNeedIntensity: number` (0-1)
- `decision.differentiated.topDesireIntensity: number` (0-1)
- Extracted from `differentiated.schemas[0].intensity`, etc., rounded via `round2()`

**Implementation details:**
- `flowNode()` now accepts an optional 4th parameter `intensity`
- When present, renders a `.flow-node-intensity` span below the detail text
- Action and Strategy nodes do not show intensity (only Schema/Need/Desire)
- Intensity displayed as a 2-decimal value (e.g. "0.63")

**User value:** Reviewer can see not just which schema/need/desire is dominant, but how strongly it's activated.

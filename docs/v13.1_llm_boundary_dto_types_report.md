## 2. DTO List

### 2.1 `LlmBoundaryRequest`

Top-level request that packages a V12 `AgentReplyPlan` + `AgentGroundingBundle`
with session metadata. This is the **only** input to the LLM boundary pipeline.

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `requestId` | `string` (deterministic) | Hash of sessionId+turnId+characterId | Stable, reproducible |
| `sessionId` | `string` | from session | |
| `turnId` | `string` | from caller | |
| `characterId` | `string` | from session | Single character only |
| `replyPlan` | `AgentReplyPlan` | from V12 | Already mediated, no raw state |
| `groundingBundle` | `AgentGroundingBundle` | from V12 | Already mediated, no raw state |
| `policyDecisionSummary` | `string` | from caller | decision + safetyLevel only |
| `safetyMode` | `SafetyMode` | from session | |
| `locale` | `string` | `"zh-CN"` | |
| `allowLlm` | `boolean` | **`false`** | Must be explicitly enabled |
| `noMutation` | `true` | `true` | Invariant |
| `noRawState` | `true` | `true` | Invariant |

### 2.2 `LlmBoundaryPrompt`

A safe, bounded prompt constructed from the request. This is the **only** content
the LLM ever sees.

| Field | Type | Notes |
|-------|------|-------|
| `promptId` | `string` (deterministic) | Hash of requestId+"prompt" |
| `requestId` | `string` | Links back |
| `systemInstructions` | `string` | Always included; safety + boundary rules |
| `developerInstructions` | `string` | From replyPlan.llmBoundaryInstructions |
| `groundingFacts` | `string[]` | Only facts from groundedFacts |
| `uncertaintyNotes` | `string[]` | Must be preserved by LLM |
| `safetyNotices` | `string[]` | Must appear in output |
| `responseConstraints` | `string[]` | What the LLM SHOULD do |
| `forbiddenClaims` | `string[]` | What the LLM MUST NOT claim |
| `outputFormat` | `"plain_text" \| "markdown"` | Default `"plain_text"` |
| `noMutation` | `true` | Invariant |
| `noWritebackAuthority` | `true` | Invariant — LLM cannot authorize writeback |

### 2.3 `LlmProviderConfig`

Configuration for an LLM provider. **Stores `apiKeyRef` only — never the actual API key.**

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `providerId` | `string` (deterministic) | Hash of type+model | |
| `providerType` | `"mock" \| "openai_compatible" \| "local"` | **`"mock"`** | No network by default |
| `modelName` | `string` | `"mock-model"` | |
| `networkAllowed` | `boolean` | **`false`** | Must be explicitly enabled |
| `timeoutMs` | `number` | `30000` | Clamped [1000, 120000] |
| `temperature` | `number` | `0.3` | Clamped [0, 2] |
| `maxTokens` | `number` | `1024` | Clamped [1, 16384] |
| `safetyMode` | `SafetyMode` | `"strict"` | |
| `apiKeyRef` | `string` | `""` | Reference only — sanitized by builder |

**API key sanitization rules:**
- Rejects strings > 128 chars
- Rejects strings starting with `sk-`, `sk-or-`, `pk-`, `rk-`, `shh-`, `sess-`, `org-`, `key-`
- Accepts short reference strings like `"OPENAI_API_KEY"`

### 2.4 `LlmProviderResponse`

Raw output from an LLM provider — **unvalidated** at this stage.

| Field | Type | Notes |
|-------|------|-------|
| `responseId` | `string` (deterministic) | Hash of provider+request+rawText |
| `providerId` | `string` | |
| `requestId` | `string` | Links back |
| `rawText` | `string` | Unvalidated LLM output |
| `finishReason` | `"stop" \| "length" \| "error" \| "timeout"` | Default `"stop"` |
| `usage` | `LlmUsage \| null` | Token counts (if reported) |
| `latencyMs` | `number \| null` | |
| `providerWarnings` | `string[]` | Rate limits, etc. |
| `error` | `string \| null` | Sanitized (max 500 chars) |

### 2.5 `LlmOutputValidationResult`

Result of running validation rules against the LLM output.

| Field | Type | Notes |
|-------|------|-------|
| `valid` | `boolean` | `false` if any error-severity violation |
| `violations` | `LlmOutputViolation[]` | Rule ID, description, severity, excerpt |
| `warnings` | `string[]` | Non-blocking warnings |
| `blockedClaims` | `string[]` | Claims that were blocked |
| `preservedSafetyNotices` | `string[]` | Safety notices found in output |
| `mutationClaimDetected` | `boolean` | |
| `diagnosisClaimDetected` | `boolean` | |
| `unsupportedClaimDetected` | `boolean` | |
| `finalVerdict` | `"pass" \| "warn" \| "fail"` | Error violations → fail |

### 2.6 `GroundingCheckResult`

Result of verifying every factual claim against `groundedFacts` and `evidenceRefs`.

| Field | Type | Notes |
|-------|------|-------|
| `grounded` | `boolean` | `true` only if all claims have evidence |
| `checkedClaims` | `string[]` | All claims examined |
| `supportedClaims` | `string[]` | Claims with matching evidence |
| `unsupportedClaims` | `GroundingUnsupportedClaim[]` | Claims without evidence |
| `evidenceMatches` | `GroundingEvidenceMatch[]` | Claim-to-evidence mappings |
| `missingEvidence` | `string[]` | Evidence refs never cited |
| `confidence` | `number` | supportedClaims/checkedClaims (0–1) |
| `verdict` | `"grounded" \| "partially_grounded" \| "ungrounded"` | |

### 2.7 `AgentNaturalLanguageReply`

The final reply that reaches the user — or doesn't, if validation/grounding fails.

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `replyId` | `string` (deterministic) | Hash of requestId+text | |
| `requestId` | `string` | | |
| `source` | `"llm" \| "fallback"` | **`"fallback"`** | LLM path requires explicit validation/grounding |
| `text` | `string` | | Natural language text |
| `grounded` | `boolean` | `false` | Set by grounding result |
| `validationVerdict` | `ValidationVerdict` | `"pass"` | |
| `groundingVerdict` | `GroundingVerdict` | `"grounded"` | |
| `safetyNotices` | `string[]` | `[]` | |
| `uncertaintyNotes` | `string[]` | `[]` | |
| `noMutation` | `true` | `true` | Invariant |
| `writebackPerformed` | `false` | `false` | **Always false** — never changes |

### 2.8 `LlmFallbackReply`

Deterministic fallback triggered when LLM is unavailable or output fails validation/grounding.

| Field | Type | Notes |
|-------|------|-------|
| `fallbackId` | `string` (deterministic) | Hash of requestId+reason |
| `requestId` | `string` | |
| `reason` | `string` | e.g. `"llm_unavailable"`, `"validation_failed"` |
| `outlineUsed` | `string[]` | Outline items from reply plan |
| `text` | `string` | Deterministic template-based text |
| `safetyNotices` | `string[]` | Always included |
| `source` | `"fallback"` | Always `"fallback"` |
| `noMutation` | `true` | Invariant |

## 3. Builder Functions

| Builder | Input | Output |
|---------|-------|--------|
| `buildLlmBoundaryRequest()` | replyPlan, groundingBundle, session, turnId, + | `LlmBoundaryRequest` |
| `buildLlmBoundaryPrompt()` | request, optional overrides | `LlmBoundaryPrompt` |
| `buildLlmProviderConfig()` | optional overrides | `LlmProviderConfig` |
| `buildLlmProviderResponse()` | providerId, requestId, rawText, + | `LlmProviderResponse` |
| `buildLlmOutputValidationResult()` | violations, warnings, + | `LlmOutputValidationResult` |
| `buildGroundingCheckResult()` | checkedClaims, supportedClaims, + | `GroundingCheckResult` |
| `buildAgentNaturalLanguageReply()` | requestId, text, source?, validation?, grounding? | `AgentNaturalLanguageReply` |
| `buildLlmFallbackReply()` | requestId, reason, outlineUsed?, text?, safetyNotices? | `LlmFallbackReply` |
| `summarizeLlmBoundary()` | — | `string[]` |

All builders are:
- **Pure** — no side effects, no input mutation
- **Deterministic** — same input → same output every time
- **Synchronous** — no promises, no network
- **ID-safe** — all IDs from `stableHash()`, no `Date.now()` / `Math.random()`

## 4. Safe Defaults

| Behavior | Default | How to Enable |
|----------|---------|---------------|
| LLM path | `allowLlm = false` | Set `allowLlm: true` in `buildLlmBoundaryRequest()` |
| Network | `networkAllowed = false` | Set `networkAllowed: true` in `buildLlmProviderConfig()` |
| Provider | `providerType = "mock"` | Set `providerType: "openai_compatible"` |
| Reply source | `source = "fallback"` | Set `source: "llm"` + provide validation + grounding results |
| API key | `apiKeyRef = ""` | Provide a short reference string (not the key itself) |

---

## 5. Provider Boundary

- `apiKeyRef` is the **only** key-related field in any DTO
- The builder **sanitizes** the ref: rejects strings that look like actual API keys
- API key resolution happens at the **provider adapter** level (V13.3+), not in types
- Provider config is injectable — no hardcoded provider
- `networkAllowed = false` by default prevents accidental network calls in tests

---

## 6. No Mutation / No Writeback Authority

Every DTO with mutation-sensitive context includes hard invariants:

| DTO | Invariant Field |
|-----|----------------|
| `LlmBoundaryRequest` | `noMutation: true`, `noRawState: true` |
| `LlmBoundaryPrompt` | `noMutation: true`, `noWritebackAuthority: true` |
| `AgentNaturalLanguageReply` | `noMutation: true`, `writebackPerformed: false` |
| `LlmFallbackReply` | `noMutation: true` |

The LLM boundary layer has **zero** ability to trigger state changes. Writeback
remains in V12's deterministic pipeline, gated by `applyWriteback()`.

---

## 7. Validation / Grounding / Fallback Contracts

| Stage | DTO | Purpose |
|-------|-----|---------|
| Validation | `LlmOutputValidationResult` | Rule-based checks: no diagnosis, no mutation claims, safety notices preserved |
| Grounding | `GroundingCheckResult` | Every factual claim maps to groundedFacts or evidenceRefs |
| Fallback | `LlmFallbackReply` | Deterministic template-based reply when validation or grounding fails |

The pipeline: **LLM Output → Validate → Ground → (Reply or Fallback)**

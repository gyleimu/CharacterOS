# CharacterOS V2.4 Trace Standardization 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.4 Trace Standardization
```

这一阶段继续收敛 V2 内核，没有扩张 V3/V4，也没有做 3D 可视化。

核心目标：

```text
让单步事件和多步模拟使用同一套可审计 trace。
```

## 本阶段新增

### 1. Core Trace Module

新增：

```text
src/core/trace/galaxyTrace.ts
```

提供核心类型：

```text
GalaxyStepTrace
PhysicsStepTrace
```

以及转换函数：

```text
toGalaxyStepTrace
toPhysicsStepTrace
```

`GalaxyStepTrace` 包含：

```text
totalForce
previousVelocity
nextVelocity
before
after
clusterMetrics
```

`PhysicsStepTrace` 在此基础上增加：

```text
eventId
category
impactScore
boundaryImpact
galaxyTrace
```

### 2. API Trace 下沉到 Core

更新：

```text
src/appContracts/characterPhysics.ts
src/services/characterPhysicsDto.ts
```

`GalaxyStepTrace` 不再定义在 app contracts 里，而是由 `core/trace` 提供。API 只是复用 core trace 类型。

这避免了核心物理解释结构被绑定在 API 层。

### 3. SimulationSnapshot 接入完整 Trace

更新：

```text
src/core/simulation/runner.ts
```

每个 simulation snapshot 现在包含：

```text
boundaryImpact
galaxyTrace
```

因此多步模拟可以完整观察：

```text
Event
-> BoundaryImpact
-> ClusterMetrics
-> TotalForce
-> Velocity
-> Drift
```

### 4. SimulationPanel 显示边界状态

更新：

```text
src/components/character/SimulationPanel.tsx
src/app/styles.css
```

模拟面板现在显示：

```text
boundary phase
drift multiplier
force.trust / force.fear
velocity.trust / velocity.fear
coordinate trust / fear
```

## 当前意义

CharacterOS 现在不只是能输出状态，还能审计状态变化路径：

```text
为什么这次事件改变了速度？
哪个星团产生了力？
边界是否放大了漂移？
多步模拟里每一步的力和速度如何累积？
```

这是后续 Benchmark、调参、可视化和长期模拟的基础。

## 下一步建议

继续 V2，不跳 V3：

```text
1. 增加 trace replay demo
2. 为 cluster variance/density 写更多边界样例
3. 将 simulation trace 输出为 JSON artifact
4. 再考虑 Dashboard 中的 trace 折叠详情
```

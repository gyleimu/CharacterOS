## 1. V11's First Write Boundary

V11.3 is the **first write operation** in the Explorer platform. All previous V11 modules were read-only. This module establishes the pattern for all future write operations:

1. **Preview first** — must run full_preview
2. **Confirm explicitly** — confirmation phrase required
3. **Audit everything** — every apply produces an audit entry
4. **Clone by default** — no mutation unless explicitly requested

## 2. Confirmation Gate

| Check | Blocked? | Message |
|-------|----------|---------|
| Confirmation missing | ❌ Blocked | "confirmation 不匹配或不完整" |
| Confirmation wrong | ❌ Blocked | "confirmation 不匹配或不完整" |
| Preview not full_preview | ❌ Blocked | "预览模式不是 full_preview" |
| Reality Audit FAIL | ❌ Blocked* | "Reality Audit 判定 FAIL" |
| Reality Audit WARN | ⚠️ Applies | Warning recorded in audit entry |

*Can be overridden with `overrideAuditFail: true`

## 3. Apply Pipeline

```
EventStudioApplyInput
    │
    ├─ Confirmation Gate ──────────► blocked if missing/wrong
    ├─ Preview Mode Gate ──────────► blocked if not full_preview
    ├─ Reality Audit Gate ─────────► blocked if FAIL (unless override)
    │
    ├─ Clone state (default) ──────► serialize → deserialize
    │  or Allow Mutation ───────────► mutate baseline directly
    │
    ├─ V10 Engine processEvent ────► real physics engine
    ├─ Repeat N times ─────────────► repetitionCount
    │
    ├─ Compute fingerprints ───────► before + after state hash
    ├─ Build audit entry ──────────► full audit record
    ├─ Build rollback reference ───► rollback:{auditId}:before:{fingerprint}
    │
    └─ Return EventStudioApplyResult
```

## 4. Audit Entry

```typescript
EventStudioAuditEntry {
  auditId            // unique ID (deterministic with seed)
  eventDraftId       // which draft was applied
  sourceId           // original source reference
  actorId            // who applied
  applyReason        // why
  appliedAt          // when (ISO timestamp)
  beforeFingerprint  // state hash before
  afterFingerprint   // state hash after
  parsedEventSummary // what was parsed
  stateDeltaSummary  // trust/fear/boundary delta
  realityAuditVerdict // PASS/WARN/FAIL
  confirmationProvided // always true (blocked otherwise)
  rollbackReference  // rollback:{auditId}:before:{fp}
  warnings           // any warnings during apply
}
```

## 5. Rollback Boundary

V11.3 generates `rollbackReference: "rollback:{auditId}:before:{fingerprint}"` — sufficient to identify what needs to be rolled back. Actual rollback implementation (restoring previous state) is deferred to V11.4 or later.

## 6. Default: No Mutation

| Option | Default | Effect |
|--------|---------|--------|
| `allowMutation` | **false** | Clone state, apply to clone, return clone |
| `allowMutation: true` | — | Apply directly to input state |

Even with `allowMutation: true`, an audit entry is always created.

## 7. Fingerprint

```typescript
computeFingerprint(state) → "fp_{hash}"
// Hashes: identity.id, memory count, belief count, trust, fear, openness,
//         neuroticism, attachment, boundary stressLoad, integrity, phase
```

Used for:
- Stale preview detection (future V11.4)
- Rollback reference
- Before/after comparison in audit

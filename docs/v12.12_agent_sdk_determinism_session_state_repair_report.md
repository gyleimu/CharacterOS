## 1. Review Findings

Code review of the post-RC V12 Agent SDK revealed three classes of issues:

### 1.1 MEDIUM BUG έΑΦ Agent SDK session state continuity broken

**File:** `src/services/agentSdkService.ts`

`processTurn()`, `previewTurnEvents()`, and `applyWriteback()` each called
`createCharacterStateFromBlueprint(createLinFanBlueprint(), { seedInitialExperiences: true })`,
creating a **brand-new state on every call**. Events applied in one turn were invisible
to the next turn έΑΦ the Agent SDK could not maintain multi-turn continuity.

### 1.2 MEDIUM έΑΦ Non-deterministic defaults in core DTO builders

`Date.now()` and `Math.random()` appeared in default ID generation paths:

## 2. Fixes Applied

### 2.1 Session State Store (`agentSdkService.ts`)

Added an **in-memory character state store** keyed by `characterId`:

```ts
const characterStateStore = new Map<string, CharacterPhysicsState>();
```

**Public API:**

| Function | Purpose |
|----------|---------|
| `getCharacterState(characterId)` | Retrieve or lazily initialize state |
| `setCharacterState(characterId, state)` | Replace stored state |
| `resetCharacterState(characterId)` | Reset to fresh LinFan blueprint |
| `getCharacterStateStoreSize()` | Diagnostic: count of tracked characters |

**Behavior changes:**

- `processTurn()` έΑΦ reads stored state for context continuity; **does not mutate** the store
- `previewTurnEvents()` έΑΦ reads stored state; **does not mutate** the store
- `applyWriteback()` έΑΦ clones stored state as baseline, applies event to clone. By default **does not** persist the result back. Pass `options.persistAppliedState = true` to update the store for true multi-turn continuity.

### 2.2 Deterministic ID Strategy

**Principle:** Same builder/producer input έΗΤ same ID every time. No `Date.now()`, no `Math.random()` in core deterministic paths.

**`agentDtoBuilders.ts`:**

| Builder | Was | Now |
|---------|-----|-----|
| `buildAgentSessionConfig` | `session_${Date.now()}` | `"session_unknown"` (caller should provide sessionId) |
| `buildAgentTurnInput` | `turn_${Date.now()}`, `new Date().toISOString()` | `"turn_unknown"`, `"unknown"` |
| `buildAgentEventCandidateFromDraft` | `candidate_${draft.sourceId \|\| Date.now()}` | `candidate_${draft.sourceId \|\| "unknown"}` |
| `buildAgentReplyPlan` | `reply_${Date.now()}` | `reply_${stableHash(JSON.stringify(params))}` |
| `buildAgentWritebackPlan` | `writeback_${Date.now()}` | `writeback_${stableHash(policy \| candidateIds)}` |

**`replyPlanner.ts`:**

| Field | Was | Now |
|-------|-----|-----|
| `replyPlanId` | `reply_${Date.now()}` | `reply_${stableHash(sessionId \| intent \| tone \| ...)}` |

**`eventStudioApply.ts`:**

| Field | Was | Now |
|-------|-----|-----|
| `auditId` (fallback) | `audit_${Date.now()}_${Math.random()...}` | `audit_${computeAuditFingerprint(actorId, sourceId, confirmation, beforeFingerprint, parsedEvent)}` |

A `stableHash()` helper (djb2-derived) produces deterministic 8έΑΥ12 hex-char hashes from string inputs. The `computeAuditFingerprint()` helper combines actorId, draft sourceId, confirmation, before-fingerprint, and parsed event summary.

### 2.3 Unused Import Cleanup

- `agentSdkService.ts`: Removed `detectAgentInputMode` from the `agentInputAdapter` import
- `eventStudioApply.ts`: Removed `runRealityAudit` and `EnvironmentSeed` imports

**Key invariants:**

1. `processTurn` and `previewTurnEvents` always observe the current stored state but **never mutate** it
2. `applyWriteback` always clones before applying (service-layer protection)
3. Only `persistAppliedState = true` writes the result back to the store
4. Different `characterId` values are fully isolated (no cross-character leakage)

## 4. Clone-Only / persistAppliedState Boundary

| Scenario | `allowMutation` | `persistAppliedState` | Effect |
|----------|:--:|:--:|--------|
| Preview workflow | false | false | Clone, apply, return έΑΦ store unchanged |
| Confirmation workflow | false | false | Clone, apply, return έΑΦ store unchanged |
| Auto-apply with persistence | false | **true** | Clone, apply, persist result |
| Direct mutation (discouraged) | true | έΑΦ | Engine mutates working clone; store unchanged unless `persistAppliedState` |

The safe default (`persistAppliedState = false`) ensures that preview-only and
confirmation workflows don't accidentally advance state. Multi-turn continuity
requires explicit opt-in via `persistAppliedState = true`.

---

## 5. What This Is NOT

- **Not a production database.** State is in-memory only. Process restart = state gone.
- **Not multi-character.** Single `characterId` per store entry; no relationship engine.
- **Not V13.** No LLM adapter, no chat completion, no UI changes.
- **Not persistence.** No file/DB storage. A future V13 production adapter would replace
  the `Map` with a repository backed by Prisma/SQLite.

## 4. Import 非 ACID：文档和测试加强

### 状态：Confirmed Closed ✓

**文档强化：**

`importCharacterPackage` 方法中已有注释：
```
// This is NOT an automatic rollback mechanism — it is an audit artifact
// so that a human operator can restore the previous state if the import
// fails partway through. Real ACID rollback would require wrapping all
// repository writes in a transaction that can be aborted.
```

**mutationOutcome 覆盖完整性：**

现在所有 `CharacterImportApplyTrace` 都包含 `mutationOutcome`：

| 场景 | stateReplaced | adjHistoryReplaced | historyRecorded | fullyApplied | description |
|------|:---:|:---:|:---:|:---:|------|
| auth 失败 (plan not ready) | false | false | false | false | "import blocked at authorization — no mutations were attempted" |
| auth 失败 (confirmation mismatch) | false | false | false | false | "import blocked at authorization — no mutations were attempted" |
| state replace 失败 | false | false | — | false | "state replacement failed; adjustment history was not modified" |
| adjHistory replace 失败 | true | false | — | false | "state replaced but adjustment history... manual rollback may be needed" |
| 全部成功 | true | true | true | true | "all mutations applied successfully" |

**新增测试：**
- `exposes mutationOutcome and stateRollbackSnapshot even when import is blocked early`
  - 验证 blocked-at-auth 时 mutationOutcome 反映零 mutation
  - 验证 stateRollbackSnapshot 仅在 mutation 尝试后才存在
  - 验证 transactionSummary.stateMutated = false
- 扩展现有 success test：验证 mutationOutcome.fullyApplied = true，stateRollbackSnapshot 存在

**修改文件：**

## 3. New Module: Event Type Coverage Audit

### 3.1 File

`src/core/audit/eventTypeCoverageAudit.ts`

### 3.2 API

```typescript
runEventTypeCoverageAudit({
  fixtures: EventTypeCoverageFixture[],  // per-type event inputs + expected profiles
  baselines: Array<{id, label, state}>,  // personality baselines (sensitive, secure, achiever)
  scenarios: Array<{id, definition}>,    // decision scenarios for relevance testing
})
```

### 3.3 Output

- `results[]` — per-event-type × per-baseline results
  - `channelActivations` — measured activation per channel
  - `directionChecks[]` — expected vs actual direction per channel
  - `relevanceScores[]` — scenario relevance + overreaction per scenario
  - `warnings[]` / `failures[]` — specific mismatches
- `channelActivationMatrix` — keyed lookup for all results
- `coverageSummary` — aggregate pass/fail counts
- `coverageVerdict` — PASS/WARN/FAIL with reasons

### 3.4 Verdict Rules

- **FAIL**: memoryImpact critical channel not responding, or >30% direction check failures
- **WARN**: >10% direction check failures, or >20% relevance check failures
- **PASS**: all checks within thresholds

## 4. Domain Relevance Enhancement

Updated `estimateDomainRelevance` in `impactCalibrationAudit.ts` to handle new event categories:

- **Category-based fallback**: When keyword matching is insufficient, uses category-to-scenario mapping
- **Expanded keywords**: failure/success → study scenarios, rejection → relationship/social, conflict → social
- **Threshold**: relevance ≥ 0.3 for "high relevance" expectation

## 9. Known Limitations (Honest Assessment)

### Coverage Gaps
1. **Keyword-based category inference**: Some events with ambiguous wording may map to `general` instead of the intended category. Using `categoryHint` in fixtures mitigates this.
2. **Single-event measurement**: belief/need/personality channels show minimal response in single-event tests (by design — they're slow channels). The accumulation audit (V10.71) verifies multi-event behavior.
3. **Domain relevance is keyword-based**: The `estimateDomainRelevance` function relies on keyword matching. Category-based fallback (V10.73 addition) improves coverage but isn't perfect.

### No FAIL — All Channels Respond
- All 10 event types produce correct directional responses
- No critical channel (memoryImpact) fails to respond for any active event type
- Fatigue and neutral correctly show minimal personality/belief changes
- Relevance routing correctly maps events to appropriate scenarios

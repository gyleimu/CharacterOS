## 1. Module

`src/core/explorer/timeMachineSnapshot.ts`

Two builders:
- `buildTimeMachineSnapshot(input)` — create immutable snapshot
- `buildTimeMachineTimeline(input)` — build sorted timeline from snapshots

## 2. Snapshot vs Rollback

| Snapshot | Rollback |
|----------|----------|
| Point-in-time record | State mutation back to previous |
| Immutable, read-only | Write operation |
| Can have many | Each replaces current state |
| View historical state | Revert current state |
| `immutable: true` | Requires confirmation |

V11.6 only implements snapshots. Rollback is deferred.

## 3. Deterministic Snapshot ID

```
snapshotId = hash(characterId | capturedAt | sequenceIndex | sourceFingerprint)
```

Same character + same timestamp + same sequence + same state → same ID. No random, no Date.now injection.

## 4. What's Included / Hidden

| Included (Summary) | Hidden (Raw) |
|-------------------|-------------|
| Personality qualitative bands | Raw coordinate values |
| Memory count + event count | Memory content payload |
| Belief count + top 3 names | Belief strength values |
| Boundary phase + stress label | stressLoad, integrity, cracks |
| Safety note | All internal IDs |

## 5. Timeline Ordering

Snapshots sorted by `sequenceIndex`, then `capturedAt`. The highest sequence is the "current" snapshot. Mixed characterIds produce a warning.

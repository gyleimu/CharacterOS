# CharacterOS V2.14 Trace Summary Guard 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.14 Trace Summary Guard
```

这一阶段继续收敛 V2 Personality Galaxy replay 数据链路。V2.8 已经为 artifact 加了 schema guard，V2.14 将同样的运行时校验扩展到 summary 和 summary index。

## 本阶段新增

### 1. Summary Runtime Guard

更新：

```text
src/core/trace/traceReplaySummary.ts
```

新增：

```text
validateTraceReplaySummary(value)
```

校验范围：

```text
scenario
title
steps
coordinateDelta
firstForce
lastForce
firstVelocity
lastVelocity
clusterTrends
dominantDirection
```

### 2. Summary Index Runtime Guard

新增：

```text
validateTraceReplaySummaryIndex(value)
```

校验范围：

```text
title
scenarioCount
summaries
directionCounts
```

并验证：

```text
scenarioCount 必须等于 summaries.length
每个 summary 都必须通过 validateTraceReplaySummary
```

### 3. Demo 写入前校验

更新：

```text
src/core/demo/traceReplayDemo.ts
```

现在 `npm run demo:trace` 的写出流程是：

```text
build artifact
validate artifact
write artifact JSON
build summary
validate summary
write summary JSON
build summary index
validate summary index
write summary index JSON
```

### 4. 测试覆盖

更新：

```text
tests/core/traceReplay.test.ts
```

新增验证：

```text
合法 summary 可以通过运行时校验
合法 summary index 可以通过运行时校验
破损 coordinateDelta 会被拒绝
空 clusterTrends 会被拒绝
scenarioCount 与 summaries.length 不一致会被拒绝
```

## 当前意义

V2 replay 数据链路现在具备三层稳定性：

```text
TraceReplayArtifact       可校验
TraceReplaySummary        可校验
TraceReplaySummaryIndex   可校验
```

这意味着未来把 replay 数据迁移到 fixtures、Dashboard 或 Benchmark seed 时，不需要相信 JSON 是正确的，可以先经过结构 guard。

## 3. Classification and Resolution

### Warning 1: Betrayal near-linear growth

- **Type:** Known limitation (documented since V10.72)
- **Root cause:** Galaxy force sqrt saturation helps but doesn't fully eliminate near-linear personality distance growth when cluster mass is amplified by seeded experiences
- **Absolute magnitude:** 0.097 over 5 events — gradual and directionally correct
- **Resolution:** **Allowed** — acknowledged limitation with clear rationale
- **Status:** allowed

### Warning 2: Neutral near-linear growth

- **Type:** Audit threshold sensitivity
- **Root cause:** Accumulation audit's linear-growth check triggered at distance > 0.02. Neutral events accumulate 0.023 — trivially small but above the absolute threshold.
- **Fix:** Raised linear-growth detection threshold from 0.02 to 0.04 (`longTermAccumulationAudit.ts:394`). The check still catches real problems (e.g., 0.05+ drift) but doesn't flag trivial neutral accumulation.
- **Resolution:** **Resolved** in V10.77
- **Status:** resolved

### Warning 3: Coverage neutral relevance

- **Type:** Audit threshold sensitivity
- **Root cause:** Coverage audit's irrelevant-scenario check used uniform `overreactionScore < 0.35`. Neutral events with 0.18 relevance (correctly low) had borderline 0.35 overreaction.
- **Fix:** Tiered overreaction check (`eventTypeCoverageAudit.ts:214`): relevance < 0.3 → overreaction limit 0.45; relevance 0.3–0.5 → overreaction limit 0.35
- **Resolution:** **Resolved** in V10.77
- **Status:** resolved

# CharacterOS V10.24 - Longitudinal Commit Audit Repository Report

**Date:** 2026-06-28
**Version:** V10.24
**Status:** Complete
**Scope:** Audit repository and service wiring for future longitudinal commit apply

---

## 1. Executive Summary

V10.24 adds the audit ledger needed before longitudinal simulation commits can become durable character history.

This version does not apply simulated final state. It does not add an apply API. It does not execute self-actions.

It creates the storage and service boundary for future V10.25/V10.26 work.

## 3. Core Types

`LongitudinalCommitAuditEntry` records:

- version
- audit id
- character id
- simulation id
- status
- timestamps
- request digest
- base and final state fingerprints
- commit policy
- changed paths
- generated memory ids
- compact before/after summaries
- governance status, blockers, warnings
- rollback plan
- warnings and reasons

Status values:

- `previewed`
- `applied`
- `rolled_back`
- `blocked`

The entry intentionally does not store full `finalState`.

---

## 4. Pure Helpers

Added functions:

- `createLongitudinalCommitAuditEntry(handoff, options?)`
- `markLongitudinalCommitAuditApplied(entry, appliedAt)`
- `markLongitudinalCommitAuditRolledBack(entry, rolledBackAt)`
- `summarizeLongitudinalCommitHistory(entries)`
- `findLongitudinalCommitAuditBySimulationId(entries, simulationId)`

These helpers are deterministic and do not mutate input entries.

---

## 5. Repository Contract

`LongitudinalCommitAuditRepository` supports:

- `list(characterId)`
- `get(characterId, auditId)`
- `getBySimulationId(characterId, simulationId)`
- `append(entry)`
- `update(characterId, auditId, updater)`
- `clear(characterId?)`

Implementations:

- `InMemoryLongitudinalCommitAuditRepository`
- `FileLongitudinalCommitAuditRepository`

File storage path:

`data/longitudinal_commit_audit.json`

---

## 6. Service Wiring

`CharacterPhysicsService` now exposes:

- `recordLongitudinalCommitAudit(entry)`
- `getLongitudinalCommitAuditHistory(characterId)`
- `getLongitudinalCommitAuditBySimulationId(characterId, simulationId)`

`resetCharacter(characterId)` clears the longitudinal commit audit for that character.

This prevents stale preview/apply history from remaining attached to a newly reset state.

---

## 7. Safety Boundaries

V10.24 does not:

- persist simulated final state
- expose final state through API
- add an apply route
- add a rollback route
- execute self-action candidates
- start background simulation
- add new benchmark cases

It only records audit metadata.

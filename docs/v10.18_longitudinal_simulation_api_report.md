# V10.18: Longitudinal Simulation API — Report

**Date:** 2026-06-24
**Phase:** V10.18 — Read-Only API for Single-Character Time Projection
**Status:** ✓ Complete

---

## Objective

Expose the V10.17 Longitudinal Single-Character Simulation Harness through a controlled, read-only HTTP API. This is NOT a background loop, NOT autonomous execution, NOT a daemon. It is a one-time, explicit, caller-initiated simulation endpoint.

---

## API

### Route

```
POST /api/characters/[characterId]/life/simulate
```

### Request Body

```json
{
  "totalHours": 24,
  "stepHours": 6,
  "seed": "optional-seed",
  "observed": true,
  "includeDecision": true,
  "includeExplanation": false,
  "commitPolicy": {
    "enabled": false,
    "commitDreams": false,
    "commitRandomThoughts": false,
    "commitInspirationSeeds": false,
    "commitSelfActionCandidates": false,
    "maxGeneratedMemoriesPerStep": 2,
    "maxTotalGeneratedMemories": 20
  },
  "lifeOptions": {
    "stimulationLevel": 0.4,
    "socialContactLevel": 0.3,
    "localHour": 22
  }
}
```

| Field | Required | Default | Notes |
|---|---|---|---|
| `totalHours` | **Yes** | — | Must be > 0 |
| `stepHours` | **Yes** | — | Must be > 0 |
| `seed` | No | `{characterId}:{totalHours}:{stepHours}` | Deterministic |
| `observed` | No | `true` | — |
| `includeDecision` | No | `false` | Includes `differentiatedDecision` per step |
| `includeExplanation` | No | `false` | Includes `differentiatedExplanation` per step |
| `commitPolicy` | No | `{enabled: false}` | See commit policy below |
| `lifeOptions` | No | — | `stimulationLevel`, `socialContactLevel`, `localHour` |

### Response (200 OK)

```json
{
  "characterId": "...",
  "result": { "...LongitudinalSimulationResult..." },
  "warnings": ["...", "Dry-run only — no state mutation, no persistence."],
  "reasons": ["..."]
}
```

The `result` field is the full `LongitudinalSimulationResult` (compact summaries, not full state).

### Validation Errors

| Status | Condition |
|---|---|
| 400 | Invalid JSON body |
| 422 | `totalHours` or `stepHours` missing or ≤ 0 |
| 401/403 | `commitPolicy.enabled=true` without auth |

---

## Commit Policy

| Policy | Status |
|---|---|
| Dry-run (default) | **Fully supported** — no state mutation, no persistence |
| Commit with memory seeds | **Deferred** — simulation runs with internal commit tracking on a working clone, but the final committed state is not yet exposable for external persistence |

**Why commit is deferred:** V10.17 `runLongitudinalSimulation` updates an internal working clone when commit is enabled, but `LongitudinalSimulationResult` does not expose the final committed state. A future core enhancement should add an optional `finalStateForCommit` field, excluded from the public DTO, to enable safe persistence.

---

## Safety Boundaries

| Boundary | Enforcement |
|---|---|
| Dry-run by default | No `commitPolicy` → no mutation |
| Auth gate for commit | `requireAuth` called when `commitPolicy.enabled=true` |
| Self-action never executed | Enforced at type level + core harness |
| No full state leak | Response includes `CompactStateSummary` only, not `CharacterPhysicsState` |
| Steps capped | Max 720 (core harness) |
| No background loop | One-time POST only |
| No LLM | Template-based explanation only |

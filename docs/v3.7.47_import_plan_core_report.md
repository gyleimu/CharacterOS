# CharacterOS V3.7.47 Import Plan Core Report

## Summary

This update moves import work back into the core architecture.

No new frontend workflow was added.

The system now has a pure domain import plan that can evaluate a validated character package before any future state mutation is allowed.

## Import Plan Status

```text
ready

```

## Import Risk

```text
low
medium
high
```

Current rules:

```text
invalid package -> blocked / high
same character, stable package -> ready / low
cross-character package -> ready / medium
package with overrides or repeated manual changes -> medium
package with high stability risk or pause governance -> needs_review / high
```

## Explicit Mutation Intent

The plan declares future behavior without executing it:

```text
requiresExplicitConfirmation
willReplaceState
willReplaceAdjustmentHistory
```

This keeps the import pipeline honest:

```text
planning is not applying
validation is not mutation
future import must be explicit
```

## Updated API

```text
POST /api/characters/[characterId]/import/validate
```

The validation response now includes:

```text
plan
```

The route still does not mutate state.

## Repository Stability

This update also strengthens:

```text
src/db/repositories/characterPhysicsRepository.ts
```

The file-backed character state repository now uses a lightweight file lock for:

```text
get
set
delete
clear
```

This prevents concurrent route tests or local operations from reading during a write window.

## Architecture Meaning

CharacterOS now has a safer import architecture:

```text
package
↓
validation
↓
summary
↓
import plan
↓
future explicit apply
```

This keeps the core direction correct:

Do not let external artifacts rewrite a character directly.

Always plan, audit, and require explicit confirmation first.

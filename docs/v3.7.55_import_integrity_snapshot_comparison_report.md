# CharacterOS V3.7.55 Import Integrity Snapshot Comparison Report

## Goal

V3.7.54 added an embedded `stateIntegrity` snapshot to exported packages.

This update makes import planning compare that embedded snapshot with recomputed integrity.

The embedded snapshot is useful for audit, but import should not blindly trust it.

## Core Change

Updated:

```text
src/core/export/characterImportPlan.ts
```

`CharacterImportPlan` now includes:

```text
stateIntegritySnapshotComparison
```

The comparison status can be:

```text
missing
matched
mismatch
```

## Behavior

Old packages without embedded integrity snapshots remain compatible:

```text
status: missing
risk: unchanged
```

Packages with matching snapshots are recorded as:

```text
status: matched
```

Packages whose embedded snapshot differs from recomputed integrity become high risk:

```text
risk: high
status: needs_review
```

That means they cannot be applied through the normal confirmed import path.

## Compared Fields

The comparison checks:

```text
valid
errorCount
warningCount
summary.memoryCount
summary.particleCount
summary.clusterCount
summary.beliefCount
summary.proceduralRoutineCount
```

This catches cases where the exported state was changed after the integrity snapshot was generated.

## Apply Gate

`authorizeCharacterImportApplication` still depends on the import plan.

Therefore:

```text
snapshot mismatch
-> import plan needs_review
-> apply authorization blocked
```

Even a correct confirmation phrase cannot bypass a mismatched embedded integrity snapshot.

## Architecture Meaning

CharacterOS now distinguishes three different questions:

```text
Is the package shape valid?
Is the recomputed state integrity valid?
Does the embedded integrity snapshot match the current package state?
```

This makes export/import safer for future human editing, migration, benchmark packaging and long-running character state transfer.

# CharacterOS V2.3 Core Convergence 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.3 Personality Galaxy Core Convergence
```

这一阶段没有继续扩张 V3 / V4，也没有做 3D 可视化。重点是收敛 V2 内核，让 Memory Decay、Cluster Metrics、Cluster Force、Momentum 和 Simulation 使用更统一的物理量。

## 本阶段新增与调整

### 0. Galaxy Step Trace

更新：

```text
src/core/physics/physicsEngine.ts
src/appContracts/characterPhysics.ts
src/services/characterPhysicsDto.ts
src/components/character/PhysicsDashboard.tsx
src/core/demo/physicsDemo.ts
```

`PhysicsStepResult` 现在直接包含：

```text
galaxyStep
```

API 的 `ProcessEventResponse` 现在返回轻量 trace：

```text
galaxyTrace.totalForce
galaxyTrace.previousVelocity
galaxyTrace.nextVelocity
galaxyTrace.before
galaxyTrace.after
galaxyTrace.clusterMetrics
```

Dashboard 的“本次物理结果”会显示：

```text
totalForce.trust
totalForce.fear
nextVelocity.trust
nextVelocity.fear
```

`npm run demo:physics` 也会输出完整 V2 step trace。

### 1. Cluster Mass 统一到 Galaxy Metrics

更新：

```text
src/core/galaxy/clusterMetrics.ts
src/core/physics/physicsEngine.ts
src/core/simulation/runner.ts
```

新增：

```text
syncClusterWithGalaxyMetrics
syncClustersWithGalaxyMetrics
```

运行态 `CharacterPhysicsState.clusters` 现在会同步到 Galaxy Metrics：

```text
mass = sum(importance * repetitionCount)
density = concentration of cluster memories
stability = 1 / (1 + variance)
```

`absorbImpactParticle` 仍然保留为低层粒子吸收函数，但 API、Dashboard、Simulation 看到的运行态星团质量来自 Galaxy Metrics。

### 2. Memory Decay 公式单一来源

更新：

```text
src/core/memory/decay.ts
src/core/galaxy/memoryDecay.ts
```

`memory/decay.ts` 现在复用 V2 的 Ebbinghaus helper：

```text
M(t)=M0*exp(-k*t)
```

避免项目里出现两套衰减公式。

### 3. Simulation Snapshot 增加 Force / Velocity

更新：

```text
src/core/simulation/runner.ts
src/components/character/SimulationPanel.tsx
```

每一步模拟现在记录：

```text
force
velocity
coordinate
```

Dashboard 模拟结果会显示：

```text
force.trust
force.fear
velocity.trust
velocity.fear
trust
fear
```

这让连续事件下的“星团合力 -> 人格速度 -> 人格漂移”可以直接被观察。

## 当前 V2 状态

V2 内核现在更加收敛：

```text
Memory Decay -> 单一 Ebbinghaus 公式
Cluster Mass -> 单一 Galaxy Metrics 来源
Cluster Force -> 基于同步后的 Galaxy cluster
Momentum -> state.velocity 持久化
Simulation -> 可观察 force / velocity / drift
```

## 下一步建议

继续做 V2，不跳 V3：

```text
1. 为 cluster density / variance 增加更多测试样例
2. 增加 support cluster 对 trust/fear 的反向修复测试
3. 让 SimulationPanel 同时显示 fear force / fear velocity
4. 增加一个纯核心 demo 输出完整 V2 step trace
5. 再考虑 API 层是否需要独立返回 galaxyStep
```

## 3. Derived State Audit

### File: `src/core/state/derivedCharacterState.ts`

| Check | Result |
|---|---|
| Legacy decision fully preserved | ✅ `decision: BehaviorDecision` unchanged |
| differentiatedDecision failure behavior | ✅ `try/catch` → silently `undefined` |
| Legacy test semantics | ✅ No existing tests modified |
| Does differentiatedDecision failure need a warning? | ⚠️ Design note — see below |

**Design note (not a bug):** When `buildDifferentiatedDecisionForState(state)` throws (e.g., malformed coordinate values), `differentiatedDecision` is silently set to `undefined`. This is intentional — differentiation is additive; the legacy `BehaviorDecision` remains the primary decision. Adding a `console.warn` or propagating a warning to the trace would increase observability without changing behavior. **Decision: defer to future API-level observability work. No fix needed for V10.14.**

**Verdict: CLEAN (1 design note)**

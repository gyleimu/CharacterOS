# CharacterOS V10.26 - Longitudinal Commit Apply API Report

**Date:** 2026-06-28
**Version:** V10.26
**Status:** Complete
**Scope:** Public guarded apply API for longitudinal commit

---

## 1. Executive Summary

V10.26 adds the public apply route for longitudinal simulation commit.

The route does not accept a client-provided final state. It reruns the same simulation server-side, rebuilds the private `finalStateForCommit` handoff, verifies preview digests, and then delegates the write to `CharacterPhysicsService.applyLongitudinalCommit()`.

---

## 2. Route

```text
POST /api/characters/[characterId]/life/simulate/commit/apply
```

Required:

- auth
- `commitPolicy.enabled=true`
- `requestDigest` from preview
- `baseStateFingerprint` from preview
- `confirmation="APPLY_LONGITUDINAL_COMMIT"`

Optional:

- `simulationId` from preview
- `allowWarnings`

---

## 3. Apply Flow

1. Require auth.
2. Reject unknown character.
3. Validate simulation bounds.
4. Validate preview digest/fingerprint fields are present.
5. Load current state.
6. Rerun simulation with the supplied request.
7. Rebuild private `LongitudinalFinalStateForCommit`.
8. Compare request digest.
9. Compare base state fingerprint.
10. Compare simulation id if supplied.
11. Call service-layer apply core.
12. Return compact response.

---

## 4. Response Privacy

The route response excludes:

- `finalState`
- raw `LongitudinalSimulationResult`
- raw per-step `commitResult`
- full `CharacterPhysicsState`
- clusters / particles / procedural routines

The response includes only:

- apply status
- compact audit dto
- compact rollback dto
- readiness summary
- warnings and reasons

---

## 5. Status Codes

| Status | Meaning |
|---|---|
| 200 | Applied |
| 401 | Unauthorized |
| 404 | Character not found |
| 409 | Preview base state is stale |
| 422 | Invalid request, digest mismatch, missing confirmation, governance block |

---

## 6. Safety Boundaries

V10.26 still does not:

- accept client final state
- execute self-action candidates
- start background life loops
- add multi-character simulation
- add world simulation
- store full final state in public history

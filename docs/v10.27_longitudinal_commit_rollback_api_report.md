# CharacterOS V10.27 - Longitudinal Commit Rollback API Report

**Date:** 2026-06-28
**Version:** V10.27
**Status:** Complete
**Scope:** Guarded rollback path for applied longitudinal commits

---

## 1. Executive Summary

V10.27 adds rollback for V10.26 longitudinal commits.

Rollback is intentionally narrow. It does not restore a full state snapshot. It only removes generated memory seeds listed in the applied commit audit's rollback plan.

---

## 2. Route

```text
POST /api/characters/[characterId]/life/simulate/commit/rollback
```

Required:

- auth
- `simulationId` or `auditId`
- `confirmation="ROLLBACK_LONGITUDINAL_COMMIT"`

---

## 3. Rollback Flow

1. Require auth.
2. Reject unknown character.
3. Load the matching longitudinal commit audit.
4. Require audit status `applied`.
5. Require explicit rollback confirmation.
6. Compare current state fingerprint with audit `finalStateFingerprint`.
7. If fingerprints differ, return conflict.
8. Remove generated memory ids from the rollback plan.
9. Persist the cloned rolled-back state.
10. Mark the audit entry `rolled_back`.
11. Return a compact response.

---

## 4. Response Privacy

The rollback response excludes:

- full `CharacterPhysicsState`
- `finalState`
- raw simulation result
- raw commit result
- clusters / particles / procedural routines
- generated memory contents

The response includes:

- rollback status
- compact audit dto
- rollback plan summary
- readiness summary
- removed / missing memory counts
- warnings and reasons

---

## 5. Status Codes

| Status | Meaning |
|---|---|
| 200 | Rolled back |
| 401 | Unauthorized |
| 404 | Character or audit not found |
| 409 | Current state differs from applied final fingerprint |
| 422 | Missing confirmation, missing target, already rolled back, or invalid audit status |

---

## 6. Safety Boundaries

V10.27 does not:

- restore full snapshots
- accept client-provided state
- delete arbitrary memories
- execute self-action candidates
- start background simulation
- create multi-character rollback
- erase audit history

Rollback changes audit status from `applied` to `rolled_back`. Failed rollback attempts do not erase the applied audit state.

V10.27 completes the minimal public commit lifecycle:

```text

```

## 10. Final Statement

Rollback should not pretend the past did not happen.

It should remove the generated residue, preserve the audit trail, and tell the operator exactly what was done.

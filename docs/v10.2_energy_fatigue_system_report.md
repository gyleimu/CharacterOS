## Objective

Establish the first real subsystem for Continuous Life: energy and fatigue modeling. V10.2 moves beyond type definitions (V10.1) into pure-function logic — computing how a character's energy, fatigue, sleep pressure, and rest debt change over time.

No state is written. No APIs are added. This is the physics of living, not the execution of it.

---

## Why Energy/Fatigue Is the First Subsystem

Every living being experiences energy depletion and fatigue. Before a character can:
- Think (attention drift)
- Feel (emotional settling)
- Remember (memory resurfacing)
- Decide (self-action candidates)

...they must have a body that gets tired.

Energy/fatigue is the **rhythm layer** — it underlies all other life tick phases. A character who is exhausted thinks differently, feels differently, and acts differently. V10.2 establishes the baseline model without wiring those consequences yet.

## Architecture

### EnergyFatigueState

```typescript
{
  energy: number;        // [0,1] — higher = more capable
  fatigue: number;       // [0,1] — higher = more tired
  sleepPressure: number; // [0,1] — higher = stronger sleep urge
  restDebt: number;      // [0,1] — accumulated rest deficit
}
```

Default: `energy=0.65, fatigue=0.25, sleepPressure=0.3, restDebt=0.2`

### EnergyFatigueContext

External factors that modulate the energy/fatigue computation:

| Field | Source | Effect |
|---|---|---|
| `elapsedHours` | Tick request | Time factor scaling |
| `isResting` | Caller choice | Determines active vs. recovery mode |
| `stressLoad` | `boundary.stressLoad` | Amplifies fatigue gain, slows recovery |
| `selfControl` | `metaState.selfControl` | Low SC = faster fatigue accumulation |
| `resilience` | `metaState.resilience` | High resilience = faster recovery |
| `emotionalSensitivity` | `metaState.emotionalSensitivity` | Small fatigue overhead |

All fields clamped to [0,1].

### Active Mode Rules (isResting=false)

```
fatigueDelta   = base_rate × timeFactor × stressAmp × selfControlAmp × emotionalAmp
energyDelta    = -base_rate × timeFactor × stressAmp × (1 + fatiguePressure)
sleepPressure  = base_rate × timeFactor × (1 + fatigue × 0.5)
restDebt       = base_rate × timeFactor × (1 + fatigue × 0.3)  // slow
```

Modifiers:
- `stressAmplifier = 1 + stressLoad × 0.35`
- `selfControlAmplifier = 1 + (1 - selfControl) × 0.30`
- `emotionalAmplifier = 1 + emotionalSensitivity × 0.15`

### Resting Mode Rules (isResting=true)

```
recoveryMultiplier = 1 + resilience × 0.30 - stressLoad × 0.30
effectiveRecovery  = max(0.05, recoveryMultiplier)

fatigueDelta   = -base_rate × timeFactor × effectiveRecovery
energyDelta    = +base_rate × timeFactor × effectiveRecovery
sleepPressure  = -base_rate × timeFactor × effectiveRecovery
restDebt       = -base_rate × timeFactor × effectiveRecovery  // very slow
```

### Time Factor

```
timeFactor = min(elapsedHours / 24, 1)
```

Saturates at 24 hours — beyond a day, the delta doesn't grow further per tick (multi-day ticks should be split).

### Rate Constants (per 24h equivalent)

| Rate | Active | Resting |
|---|---|---|
| Fatigue | +0.30 | -0.35 |
| Energy | -0.25 | +0.40 |
| Sleep Pressure | +0.28 | -0.38 |
| Rest Debt | +0.08 | -0.10 |

All deltas are small and bounded. A 1-hour active tick changes energy by ~1-2%, not 25%.

## Functions

| Function | Purpose |
|---|---|
| `computeEnergyFatigueDelta(state, context)` | Pure delta computation |
| `applyEnergyFatigueDelta(state, delta)` | Returns new clamped state (immutable) |
| `tickEnergyFatigue(state, context)` | Full tick: delta + apply + trace |
| `buildEnergyFatigueContextFromCharacter(char, hours, resting)` | Read-only context extraction |

All functions are pure. No state mutation. No side effects.

## Design Philosophy

> Continuous Life cannot be only about thoughts. Characters need embodied limits.

Fatigue, energy, and sleep pressure are the bottom rhythm of being alive. A character is not an always-awake, always-rational, always-capable agent. Humans get tired. CharacterOS must also allow characters to get tired.

V10.2 establishes this as a **pure model** — the physics of living, not the complexity of psyche. Later subsystems (attention, boredom, memory, self-action) will consume energy/fatigue state to modulate their own behavior.

> First the body. Then the mind.

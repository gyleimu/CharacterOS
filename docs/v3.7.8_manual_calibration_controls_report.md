# CharacterOS V3.7.8 Manual Calibration Controls Report

## Purpose

Create the first place where a human can participate in data adjustment.

This pass does not expose raw personality formulas yet.

It exposes calibration thresholds safely through Dashboard controls.

## Adjustable Calibration Parameters

Replay calibration now accepts:

```text
daysPerStep
maxDriftMagnitude
minObservableDrift
maxVelocity
```

Meaning:

```text
daysPerStep          how many days pass between replay events
maxDriftMagnitude   maximum acceptable total trust/fear drift
minObservableDrift  minimum drift before a scenario is considered inspectable
maxVelocity         maximum acceptable final trust/fear velocity
```

## API

`GET /api/trace/replay/calibration` now supports query params:

```text
?daysPerStep=30
&maxDriftMagnitude=0.2
&minObservableDrift=0.01
&maxVelocity=0.05
```

Parameters are normalized to safe ranges.

## Dashboard

The Replay 校准 panel now includes manual inputs for:

```text
时间步长
最大漂移
最小可见漂移
速度上限
```

Click:

```text
重新校准
```

to run calibration with the current values.

## Result

CharacterOS now has a small human-in-the-loop calibration surface.

This is the first safe data-adjustment point:

```text
human changes calibration standards
system reruns fixed replay scenarios
Dashboard shows pass / warn / fail
```

Future tuning can expose actual physics parameters only after this calibration workflow feels useful.

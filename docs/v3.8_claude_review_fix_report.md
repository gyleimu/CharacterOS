# CharacterOS V3.8 Claude Review Fix Report

## 修复范围

本轮修复针对 Claude review 中指出的基础设施风险点。

没有新增前端。

没有扩张人格理论模块。

没有改变 Character Physics Core 的主线方向。

重点是让 V3.8 的导入、状态、文件仓库和文档边界更稳。

## 已修复问题

### 1. getState 防御性拷贝

`InMemoryCharacterPhysicsService.getState()` 现在返回完整 clone。

调用方拿到状态后，即使修改：

```text
memories
clusters
coordinate values
```

也不会污染 service 内部真实状态。

对应测试：

```text
returns a defensive copy of character state
```

### 2. 导入包 apply 阶段类型守卫

`importCharacterPackage()` 不再在 apply 阶段直接把 unknown package 强制断言成导入包。

新增 apply-time guard：

```text
toImportableCharacterPackage
isSerializedCharacterPhysicsStateShape
parseParameterAdjustmentHistory
isParameterAdjustmentHistoryEntry
```

如果导入包在授权后仍无法安全解释，系统会记录：

```text
apply_failed
history_recorded
```

并保持：

```text
stateMutated = false
adjustmentHistoryMutated = false
```

### 3. Adjustment History Entry 验证

导入时不再只检查 `adjustmentHistory.history` 是数组。

现在会检查每条 entry 的核心字段：

```text
id
characterId
action
status
snapshotId
operationCount
targetPaths
createdAt
reasons
governanceOverride
```

对应测试：

```text
blocks a confirmed package when adjustment history entries are not safely shaped
```

### 4. 文件锁 stale owner 处理

共享 file lock helper 现在会写入 owner pid。

当遇到已有 lock 时：

```text
owner pid 已死亡 -> 可清理 stale lock
owner pid 仍存活 -> 保留 lock
owner pid 状态无法确认 -> 保守保留 lock
```

这减少了 Windows 本地 JSON 仓库因残留 lock 卡死的风险，也避免过度激进删除活跃 lock。

### 5. 测试环境 singleton 隔离

`characterPhysicsServiceSingleton` 在 Vitest 环境下使用内存仓库。

开发 / 生产仍使用文件仓库。

这样 API route 测试不会并发争抢同一组本地 JSON 文件，同时不改变正常运行路径。

### 6. 架构文档校准

`docs/architecture_bible.md` 和 `docs/latest_development_flow.md` 已更新。

关键校准：

```text
概念链路 != 当前代码单函数链路
```

当前代码是两阶段结构：

```text
Phase 1: Event ingestion and physics mutation
Phase 2: Derived decision read model
```

并明确：

```text
核心逻辑完成前不恢复用户可见前端
Dashboard / 高级可视化后置
```

### 7. Repository Update Boundary

`CharacterPhysicsRepository` 新增：

```text
update(characterId, updater)
```

文件仓库会在同一把 repository lock 内完成：

```text
read current state
run mutation
write next state
```

已经接入：

```text
processEvent
simulateEvents
tickCharacter
applyParameterAdjustment
rollbackParameterAdjustment
```

这不能替代真正数据库事务，也不覆盖跨仓库导入 ACID，但已经修复了核心状态变更路径中最明显的 get -> mutate -> set 松散边界。

### 8. API Route Boundary

新增共享工具：

```text
src/app/api/_shared/routeUtils.ts
```

用于：

```text
resolveCharacterRouteParams
readJsonBody
```

所有 character route 中重复的 `resolveParams` 已移除。

`.catch(() => ({}))` 静默吞 JSON 错误模式已移除。

非法 JSON 现在明确返回：

```text
400 Invalid JSON body
```

### 9. Lin Fan Blueprint Wiring

`createLinFanBlueprint()` 现在接入：

```text
linFanRewardState()
linFanHomeostasisState()
```

林凡不再只使用默认 reward/homeostasis。

### 10. Derived World Interpretation Time Perception

`deriveCurrentWorldInterpretation()` 不再使用硬编码 normal time perception。

现在使用：

```text
perceiveEventTime()
```

等待、失联、孤独、边界压力等会影响派生世界模型中的主观时间证据。

### 11. Trace Replay Summary Robustness

`traceReplaySummary` 遇到缺失 cluster metric 时不再直接抛出未捕获异常。

缺失 metric 的 cluster trend 会被跳过，让 summary 构建更适合作为 API 基础设施。

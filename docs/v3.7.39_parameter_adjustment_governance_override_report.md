# CharacterOS V3.7.39 Parameter Adjustment Governance Override Report

## Summary

This update adds governance override auditing for manual parameter adjustment.

Manual adjustment is now blocked during active governance cooldown unless the caller provides an explicit override reason.

Dashboard does not expose override input yet.

This is a core/API safeguard for future operator workflows.

## Updated Core

```text
src/core/parameters/parameterAdjustmentGovernanceOverride.ts
src/services/characterPhysicsService.ts
```

New function:

```text
evaluateParameterAdjustmentGovernanceOverride()
```

## Override Rules

If cooldown is not active:

```text
allowed = true
usedOverride = false
```

If cooldown is active and no override is provided:

```text
allowed = false
```

If cooldown is active and override is provided:

```text
override.enabled must be true
override.reason must be meaningful
```

Current minimum reason length:

```text
12 characters
```

## Updated API Behavior

The service checks current adjustment history and governance before applying.

## Architecture Meaning

Governance now has teeth.

It is no longer only a Dashboard warning.

During active cooldown, manual adjustment is blocked unless a future operator explicitly records why they are overriding the guard.

This protects character continuity while still leaving room for deliberate human intervention.

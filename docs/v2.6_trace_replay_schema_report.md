# CharacterOS V2.6 Trace Replay Schema 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.6 Trace Replay Schema
```

这一阶段继续推进 V2 内核实验能力：把 trace replay 从临时 demo 升级成可复用的 core schema 和多场景 artifact 生成器。

## 本阶段新增

### 1. Trace Replay Core Module

新增：

```text
src/core/trace/traceReplay.ts
```

提供：

```text
TraceReplayScenarioId
TraceReplayArtifact
TraceReplayStepArtifact
buildTraceReplayArtifact
traceReplayScenarios
```

Artifact 标准结构：

```text
title
scenario
initialCoordinate
steps[]
finalCoordinate
```

每个 step 包含：

```text
eventId
category
impactScore
boundary
force
clusterForces
velocity
coordinate
clusterMetrics
```

### 2. 多场景 Replay

当前内置场景：

```text
abandonment_then_repair
betrayal_spiral
success_recovery
```

运行：

```text
npm run demo:trace
```

会生成：

```text
outputs/trace_replay_abandonment_then_repair.json
outputs/trace_replay_betrayal_spiral.json
outputs/trace_replay_success_recovery.json
```

### 3. Demo 简化

更新：

```text
src/core/demo/traceReplayDemo.ts
```

Demo 现在只负责：

```text
选择 replay scenarios
调用 buildTraceReplayArtifact
写入 outputs
打印关键摘要
```

核心 artifact schema 不再写死在 demo 里。

## 测试新增

新增：

```text
tests/core/traceReplay.test.ts
```

验证：

```text
所有 replay scenario 都能生成 schema-stable artifact
betrayal_spiral 产生 trust < 0 / fear > 0 的背叛力
success_recovery 产生 trust > 0 / fear < 0 的恢复力
```

## 验证结果

```text
npm test          40 passed
npm run demo:trace OK
```

## 当前意义

CharacterOS 现在有了最小实验数据格式：

```text
Event sequence
-> TraceReplayArtifact
-> JSON artifact
```

这为后续 Benchmark、调参、回归测试和可视化准备了稳定输入。

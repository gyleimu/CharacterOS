# V10.22 Longitudinal Commit Preview Report

## Executive Summary

V10.22 adds a guarded commit preview surface for longitudinal single-character simulation.

The route builds the private `finalStateForCommit` handoff introduced in V10.21, strips the private `finalState`, and returns only a public `LongitudinalCommitPreview`.

No state is persisted. No self-action is executed. No background loop is started.

## Route

`POST /api/characters/[characterId]/life/simulate/commit/preview`

Request body:

```json
{
  "totalHours": 8,
  "stepHours": 4,
  "seed": "optional-seed",
  "includeDecision": false,
  "includeExplanation": false,
  "commitPolicy": {
    "enabled": true,
    "commitDreams": true,
    "commitRandomThoughts": true,
    "commitInspirationSeeds": true
  }
}
```

Response shape:

```json
{
  "characterId": "lin_fan",
  "preview": {
    "version": "v10.21",
    "simulationId": "...",
    "requestDigest": "...",
    "baseStateFingerprint": "...",
    "finalStateFingerprint": "...",
    "commitSurface": "...",
    "auditSummary": "...",
    "rollbackSummary": "...",
    "governance": "..."
  },
  "warnings": [],
  "reasons": []
}
```

The response intentionally does not include `finalState`, raw `result`, raw `commitResult`, or full `CharacterPhysicsState`.

## Governance

- Auth is required because this is a write-intent preview.
- `commitPolicy.enabled=true` is required.
- Unknown characters return 404.
- `totalHours` is capped at 17280.
- `stepHours` is capped at 24.
- The route does not call `replaceState`.
- The route does not record audit history yet.

## Privacy Boundary

Internal route flow:

1. Load base state.
2. Run longitudinal simulation with commit tracking on an internal clone.
3. Extract final committed clone privately.
4. Build `LongitudinalFinalStateForCommit`.
5. Strip via `stripFinalStateForPublicPreview`.
6. Return `LongitudinalCommitPreview`.

This keeps V10.21's private handoff available for future apply work without exposing it through the API.

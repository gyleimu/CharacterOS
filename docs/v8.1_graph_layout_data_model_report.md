## 1. 新增文件

### `src/core/graph/mindGraphLayout.ts`（370 行）

**从 MindGraphSnapshot 构建 GraphLayoutSnapshot。Pure data layer — 无渲染。**

| 接口 | 用途 |
|------|------|
| `LayoutPosition` | { x, y } — 逻辑坐标 |
| `LayoutSize` | { radius } — 逻辑大小 |
| `LayoutStyle` | { fill, stroke, strokeWidth, opacity } — 视觉样式 |
| `LayoutNode` | nodeId, position, size, style, weight, risk, zIndex |
| `LayoutEdge` | edgeId, sourceNodeId, targetNodeId, weight, style, directed |
| `GraphLayoutSnapshot` | version, nodes[], edges[], summary, warnings, reasons |

**`buildGraphLayoutSnapshot(snapshot, options?)`** — 将 MindGraphSnapshot 转换为布局数据。

### 布局策略

| Node type | 位置策略 |
|-----------|---------|
| personality_core | 中心 (0, 0) |
| impact_cluster | 按 category 预置位置 + 随机序列 |
| memory | 依附 cluster 或自由轨道 |
| impact_particle | 依附 cluster 或随机散布 |
| belief | 环形分布 (radius=0.55) |
| need | 内环 (radius=0.25) |
| desire | 中环 (radius=0.35) |
| behavior_bias | 外环 (radius=0.45) |
| temporal_process | 最外环 (radius=0.85) |
| internal_state_variable | 环形 + 域偏移 |
| benchmark_signal | 最外圈随机角度 |

### 颜色方案

- 11 种节点类型有独立的 fill/stroke 颜色对
- 高/中风险节点有额外的颜色强调
- 14 种边类型有独立的线样式

### 确定性

使用 `hashToFloat(nodeId)` 生成稳定的 jitter 和环形位置。相同输入 → 相同布局。

## 1. Module

`src/core/agent/agentInputAdapter.ts` — 5 input modes → normalized `AgentTurnInput`

## 2. Supported Input Modes

| Mode | Raw Type | Content Source | Speaker |
|------|----------|---------------|---------|
| `chat` | `ChatRawInput` | `message` | `speakerLabel` / "unknown" |
| `journal` | `JournalRawInput` | `entry` | "self" |
| `story` | `StoryRawInput` | `sceneText` | `narrator` / "narrator" |
| `plugin` | `PluginRawInput` | summarized payload | `plugin:{id}` |
| `tool` | `ToolRawInput` | `result` | `tool:{name}` |

## 3. Normalization Strategy

```
RawInput → detect mode → extract content → sanitize → deterministic turnId → AgentTurnInput
```

## 4. Deterministic Turn ID

```
turnId = stableHash(sessionId | inputType | content[0:100] | occurredAt)
Same input across runs → same turnId. No Date.now in hash.
```

## 5. Sanitization Rules

- Remove control characters (0x00–0x1F except \n, \t)
- Remove null bytes
- Normalize line endings (\r\n → \n)
- Trim leading/trailing whitespace
- Cap at 8000 characters
- Preserve Chinese text and meaningful punctuation

## 6. Metadata Redaction

| Allowed Keys | Redacted Keys |
|-------------|---------------|
| source, locale, device, author, tags, location, chapterId, pluginId, toolName, mood | apiKey, token, password, secret, authorization, cookie, key, bearer |

Redacted values replaced with `[REDACTED]`.

## 7. Safety Defaults

- `consentForWriteback: false` on every normalized input
- `userConfirmation: undefined` unless explicitly provided
- Plugin payloads summarized, not leaked
- No LLM fields created
- No multi-character relationship fields

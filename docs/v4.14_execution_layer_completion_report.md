## 5. 为什么 V4 不做 Decay and Recovery

```text
decay_and_recovery (Phase 3) 是 V3 tick 中唯一的 composite phase。

它与前 4 个 delegated phase 的根本区别：

1. 4 个 V3 函数调用（不是 1 个）
   - decayMemory, decayProceduralRoutine, recoverBoundary, recoverRewardBaseline

2. 集合映射操作
   - memories 和 proceduralRoutines 使用 .map() 遍历
   - 不是整体替换

3. 硬编码 reasons
   - 不来自任何 V3 函数的返回值

4. 两种方案都有显著缺陷：
   - 方案 A (整块 composite): V4 抽象价值低，黑箱包装
   - 方案 B (拆分为 4 个 adapter): 破坏 V3 17-phase trace 兼容性

5. 4/5 mutation phases delegated 已经足够证明 V4 adapter 框架的正确性。
   第 5 个不需要为了"完成"而强行适配。

因此 deferred to V5。
V5 可以在不受 17-phase trace 兼容约束的情况下，设计新的 trace format。
```

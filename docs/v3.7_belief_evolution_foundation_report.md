# CharacterOS V3.7 Belief Evolution Foundation Report

## Version

```text
CharacterOS V3.7
```

## Goal

Make beliefs persistent slow variables instead of purely derived labels.

This version keeps the original rule:

```text
Event -> Memory -> Belief
```

but adds time:

```text
Memory support over time -> Belief Evolution
```

## New Core Module

```text
src/core/belief/beliefEvolution.ts
```

New functions:

```text
assimilateMemoryIntoBeliefs()
evolveBeliefsForTick()
```

New trace:

```text
BeliefEvolutionTrace
- before
- after
- strengthened
- weakened
- reasons
```

## State Changes

`CharacterPhysicsState` now stores:

```text
beliefStates
```

Event processing now does:

```text
MemoryNode -> belief evidence assimilation
```

Continuous Tick now does:

```text
decayed memory support
-> slow belief strengthening / weakening
-> beliefEvolution trace
```

Serialization now preserves `beliefStates`.

Derived state now prefers persistent `beliefStates` and only falls back to memory-derived beliefs when no persistent beliefs exist.

## Dashboard

Continuous Tick panel now displays:

```text
信念增强
信念松动
belief evolution reasons
```


Covered behavior:

```text
new memory becomes belief evidence

unsupported beliefs weaken slowly

```

## Design Boundary

This version intentionally does not:

```text
use Bayesian math yet
call LLM for belief interpretation
rewrite personality directly
implement complex contradiction resolution
model belief networks
```

V3.7 is a foundation layer. It prepares the system for future Bayesian belief update, contradiction handling and long-term identity formation.

## 1. Objective

把所有核心真实性审计统一成一个可运行、可报告、可失败的 regression gate。

任何核心逻辑变更都必须通过这个 gate，防止回归：
- state changed but decision did not respond
- explanation ungrounded
- positive support boundary over-response
- repeated event force unbounded growth
- trust repair failure
- event category routing degradation
- neutral overreaction
- personality one-step over-flip

## 3. Gate Rules

### FAIL Conditions
- Any audit suite has FAIL verdict
- Reality Audit case has state unchanged after meaningful event
- Decision responsiveness FAIL
- Explanation grounding missing
- Support boundary over-response detected (V10.70 regression)
- Required event category missing from coverage

### WARN Conditions
- Any suite has WARN but no FAIL
- Trust repair below minimum threshold after 5 support events
- Neutral events accumulate personality distance near threshold
- Known limitations present (documented)

### PASS Conditions
- All required suites pass or have only documented/allowable warnings
- No FAIL conditions met

## 5. Usage

```bash
# Run the gate
npx tsx scripts/run-core-reality-gate.ts

# Run gate tests
npx vitest run tests/core/audit/coreRealityRegressionGate.test.ts

# Run all targeted audits
npx vitest run tests/core/audit/coreRealityRegressionGate.test.ts \
  tests/core/audit/realityAudit.test.ts \
  tests/core/audit/longTermAccumulationAudit.test.ts \
  tests/core/audit/eventTypeCoverageAudit.test.ts \
  tests/core/audit/impactCalibrationAudit.test.ts
```

Exit code: 0 for PASS/WARN, 1 for FAIL (CI-compatible).

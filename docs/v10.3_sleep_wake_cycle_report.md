## Objective

Establish the sleep-wake circadian rhythm system for Continuous Life. V10.3 models how a character transitions between awake, drowsy, sleeping, and waking states — driven by sleep pressure, fatigue, energy, and time of day.

Sleep is not a button. It emerges from the interaction of bodily signals.

---

## Why Sleep Is Central to Continuous Life

Energy/fatigue (V10.2) tells us **how tired** the character is. Sleep-wake (V10.3) tells us **what the character is doing about it**.

Sleep is the primary recovery mechanism. During sleep:
- Energy recovers
- Fatigue decreases
- Sleep pressure dissipates
- Rest debt slowly unwinds

But sleep is not guaranteed. A character can be exhausted (high fatigue, high sleep pressure) yet unable to fall asleep — too stressed, too alert, or fighting their circadian rhythm.

V10.3 models this as a **phase transition system** with multiple factors, not a simple threshold.

## Architecture

### SleepWakePhase (6 phases)

```
awake → drowsy → falling_asleep → light_sleep → deep_sleep → waking → awake
```

| Phase | Meaning |
|---|---|
| `awake` | Fully alert, not sleeping |
| `drowsy` | Getting tired, attention fading |
| `falling_asleep` | Transitioning into sleep |
| `light_sleep` | Shallow sleep, easily disrupted |
| `deep_sleep` | Restorative sleep |
| `waking` | Emerging from sleep |

### SleepWakeState

```typescript
{
  phase: SleepWakePhase;
  sleepQuality: number;       // [0,1] — restfulness of current/last sleep
  circadianAlignment: number; // [0,1] — sleep timing vs circadian rhythm
  hoursSinceSleep: number;    // accumulated awake time
  hoursAsleep: number;        // accumulated asleep time in current period
}
```

Default: `awake, quality=0.65, alignment=0.7, hoursSinceSleep=8, hoursAsleep=0`

### SleepWakeContext

External factors driving sleep transitions:

| Field | Source | Role |
|---|---|---|
| `elapsedHours` | Tick | Time step |
| `localHour` | World clock | Circadian drive |
| `energy` | EnergyFatigueState | High energy resists sleep |
| `fatigue` | EnergyFatigueState | High fatigue promotes sleep |
| `sleepPressure` | EnergyFatigueState | Primary sleep driver |
| `stressLoad` | Boundary | Disrupts quality |
| `resilience` | MetaState | Improves recovery |

## Circadian Sleep Drive

Piecewise model mapping local hour → [0,1] drive:

```
 0– 5:  High (peak ~3am, 1.0)
 6– 8:  Medium (0.35–0.55, morning decline)
 9–17:  Low (0.17–0.20, daytime alert)
18–21:  Medium (0.30–0.54, evening rise)
22–23:  High (0.65–0.86, approaching peak)
```

Circular distance from peak (3am) with wrap-around ensures smooth 24h cycle.

---

## Sleep Readiness

Weighted combination of five factors:

| Factor | Weight | Direction |
|---|---|---|
| `sleepPressure` | 35% | ↑ readiness |
| `fatigue` | 22% | ↑ readiness |
| `circadian drive` | 22% | ↑ readiness |
| `energy deficit (1-energy)` | 15% | ↑ readiness |
| `hoursSinceSleep / 24` | 6% | ↑ readiness |

High stress (>0.7) adds a small exhaustion bonus.

---

## Phase Transition Rules

### From wake states (awake / drowsy / falling_asleep):

| Readiness | Phase |
|---|---|
| < 0.35 | `awake` |
| 0.35–0.55 | `drowsy` |
| 0.55–0.75 | `falling_asleep` |
| > 0.75 | `light_sleep` |

### From sleep states (light_sleep / deep_sleep):

| Condition | Next Phase |
|---|---|
| `hoursAsleep < 1` | `light_sleep` (just fell asleep) |
| `stress < 0.6, resilience > 0.4, hoursAsleep ≤ 5` | `deep_sleep` (good conditions) |
| `hoursAsleep ≥ 6` or `fatigue < 0.2` or `stress > 0.8 & hoursAsleep ≥ 3` | `waking` |
| Otherwise | `light_sleep` (default) |

### From waking:
Always transitions to `awake`.

---

## Tick Behavior

`tickSleepWake` is a pure function that:

1. Computes `circadianSleepDrive` from local hour
2. Computes `sleepReadiness` from state + context
3. Selects next `SleepWakePhase`
4. Updates `hoursSinceSleep` / `hoursAsleep` counters
5. Adjusts `sleepQuality` (stress ↓, resilience ↑ during sleep)
6. Adjusts `circadianAlignment` (night sleep ↑, day sleep ↓)
7. Produces a `SleepWakeTrace` with reasons

## Design Philosophy

> Sleep is not turning the character off. Sleep is another way the character continues to live.

Humans recover, forget, reorganize, and dream during sleep. V10.3 establishes the rhythm of sleeping and waking. It answers: *is the character asleep right now?* and *should they be?*

V10.4 will let dreams begin.

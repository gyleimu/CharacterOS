## 1. Executive Summary

V10.34 implements the static Canvas renderer as a **render command list builder**. Instead of drawing to a real Canvas 2D context, it produces a deterministic array of plain-object draw commands (circle, line, text, glow, driftVector, background). This keeps the renderer testable without browser APIs and completely deterministic.

**New files:**
- `src/core/graph/mindGalaxyCanvasRenderer.ts` — 493 lines
- `tests/core/graph/mindGalaxyCanvasRenderer.test.ts` — 34 tests

## 2. Render Command Types

| Command | Kind | Key Fields | Draw Order (zIndex) |
|---------|------|-----------|---------------------|
| Background | `background` | color, width, height | 0 (first) |
| Glow | `glow` | x, y, radius, color, opacity | 5–7 |
| Line (edge) | `line` | x1, y1, x2, y2, stroke, strokeWidth, directed | 10 |

| Text (label) | `text` | x, y, content, fontSize, color, opacity | 200 |
| Drift Vector | `driftVector` | x, y, dx, dy, endX, endY, color, opacity | 300 (last) |

All commands include:
- `kind` — discriminator
- `zIndex` — draw ordering (sorted ascending)
- Optional `nodeId` / `edgeId` — for future hit-testing

## 4. Rendering Rules

### Draw order (by zIndex)

```
0:    background fill
5-7:  nebula glow + cluster halos + core glow
10:   edges (lines)
50+:  nodes (circles, zIndex from layout)
200:  text labels (when showLabels=true)
300:  drift vectors (L4 only, when showDriftVectors=true && !reducedMotion)
```

### Zoom gating

| Zoom | Nodes Visible | Edges Visible | Labels | Drift Vectors |
|------|--------------|---------------|--------|---------------|
| L0 | personality_core | minimal | core only | No |
| L1 | + impact_cluster, temporal_process | + belongs_to_cluster, clusters_around | No | No |
| L2 | + memory, belief, need, desire, impact_particle | + 6 relationship types | edge labels (weight≥0.5) | No |
| L3 | + behavior_bias, internal_state_variable, benchmark_signal | + all 14 types | node labels (radius≥8) | No |
| L4 | all nodes | all edges | all labels | Yes (when allowed) |

### Drift vector gating

Drift vectors only render when ALL conditions are true:
1. `zoomLevel === "L4"`
2. `showDriftVectors !== false`
3. `reducedMotion !== true`
4. Node has a `drift` object (all nodes in the view snapshot do, by default)

## 5. Determinism Guarantees

- No `Math.random()` — drift angles from `hash(nodeId + seed)` in V10.33
- No `Date.now()` — all timestamps from snapshot
- No `window`, `document`, `CanvasRenderingContext2D`, `requestAnimationFrame`
- Same snapshot + same options → byte-identical JSON output
- Stable sort: commands with equal zIndex preserve insertion order

## 7. Data Flow

```text
MindGalaxyViewSnapshot (V10.33)
        │
        ▼
buildGalaxyCanvasRenderCommands(snapshot, { zoomLevel, showLabels, ... })
        │
        ▼
GalaxyCanvasRenderCommand[]   ← sorted by zIndex, ready for Canvas 2D ctx.* calls
        │
        ├──▶ summarizeGalaxyCanvasRender(commands) → counts/stats
        │
        ▼
   Future Canvas 2D consumer (V10.35+):
   ctx.fillStyle = cmd.color; ctx.fillRect(0, 0, cmd.width, cmd.height);
   ctx.arc(cmd.x, cmd.y, cmd.radius, 0, Math.PI*2); ctx.fill();
   ...
```

# CharacterOS V3.7.32 Parameter Adjustment Apply Report

## Summary

This update adds pure function application and rollback for parameter adjustment patches.

The system can now apply a ready patch to a cloned character state, and rollback from a captured snapshot.

This is still not connected to Dashboard buttons or automatic Tick execution.

## Updated Core

```text
src/core/parameters/parameterAdjustmentApply.ts
```

## New Functions

```text
applyParameterAdjustmentPatch()
rollbackParameterAdjustmentPatch()
```

## Safety Rules

Patch application is blocked unless:

```text
patch.status === ready
snapshot.status === captured
snapshot values match patch from-values
all paths can be safely written
```

Application writes only to a cloned state.

The input state is not mutated.

## Rollback

Rollback uses:

```text
snapshot.rollbackOperations
```

and also returns a cloned state.

## Current Flow

```text
observe
↓
accumulate
↓
draft
↓
preview
↓
audit
↓
patch plan
↓
pre-application snapshot
↓
explicit apply
↓
rollback if needed
```

## Design Constraint

This version only exposes pure core logic.

There is still no automatic application.
There is still no Dashboard apply button.
There is still no route that mutates persisted character data.

## Architecture Meaning

CharacterOS now has executable calibration logic, but only as a pure function.

This is the correct order:

```text
make changes visible
make changes reviewable
make changes reversible
then make changes executable
only later expose them through UI or API
```

The engine can now support manual parameter editing without sacrificing safety.

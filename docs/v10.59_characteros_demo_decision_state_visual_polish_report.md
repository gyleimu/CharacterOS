# V10.59 CharacterOS Demo Decision Flow + State Signal Visual Polish Report

**Date:** 2026-06-28

## Overview

V10.59 adds three visual improvements focused on internal decision mechanics and state signals: a deterministic decision flow diagram, a boundary pressure gauge, and energy/fatigue dual bars.

## Changes

### 1. Decision Flow Diagram (New)

A horizontal pipeline diagram visualizes the deterministic decision chain on the Decision tab.

**Implementation details:**
- New `renderDecisionFlow(diff)` function renders: Schema → Need → Desire → Strategy → Action
- Each node shows a category label + the top-ranked item (e.g. "安全寻求图式")
- `topSchema`, `topNeed`, `topDesire` extracted from `differentiated` data — only one top item per node to keep layout clean
- Arrow connectors between nodes
- Action text truncated at 36 chars to prevent overflow
- Annotated with "deterministic pipeline — 不是 LLM 思考链"
- Color-coded: Schema (purple), Need (blue), Desire (orange), Strategy (green/red by direction), Action (white)
- `flowNode()` helper renders each node with label + detail
- Responsive: on narrow screens, flow stacks vertically with rotated arrows

**User value:** Reviewer can trace the full causation chain from activated schema to final action in one glance.

### 2. Boundary Pressure Gauge (New)

Text-only boundary status replaced with a visual gauge on the Today tab.

**Implementation details:**
- New `boundarySignal` data structure: `{ phase, phaseLabel, stressLoad }` — all values clamped to [0, 1]
- `boundaryGauge(signal)` renders: colored phase label + percentage + horizontal bar
- Bar color by phase: overflow=red (`#d96c6c`), strained=amber (`#d4a85e`), stable=green (`#5bae7a`)
- Bar background tinted to match severity level
- Footer text: "数值已 bounded 至 [0,1] 区间" — explicitly states bounds
- Raw stressLoad is clamped via `clamp01()` before reaching the data layer
- The old `boundary` text field is preserved for backward compatibility

**User value:** Boundary status is scannable by color; the bounded annotation makes it clear values are safe for display.

### 3. Life Preview Energy/Fatigue Dual Bars (New)

Plain energy/fatigue numbers replaced with paired visual bars on the Life Preview tab.

**Implementation details:**
- New `.energy-fatigue-dual` container with two `.ef-bar` children
- Energy bar: green (`#5bae7a`), Fatigue bar: orange (`#d4855e`)
- Each bar shows label + percentage + horizontal bar proportional to value
- Sleep phase rendered as a colored badge: awake=green, drowsy=amber, asleep=blue
- Existing "候选动作不会自动执行" warning preserved

**User value:** Energy budget is immediately scannable — reviewer sees at a glance that energy is moderate (51%) and fatigue is moderate (42%).

## Data Layer Additions

| Field | Type | Description |
|---|---|---|
| `today.boundarySignal` | `{ phase, phaseLabel, stressLoad }` | Structured, bounded boundary status |
| `decision.differentiated.topSchema` | `string` | Top-ranked schema label (for flow node) |
| `decision.differentiated.topNeed` | `string` | Top-ranked need label (for flow node) |
| `decision.differentiated.topDesire` | `string` | Top-ranked desire label (for flow node) |

All values are clamped to [0, 1] before serialization. The `boundarySignal.stressLoad` is guaranteed ≤ 1.

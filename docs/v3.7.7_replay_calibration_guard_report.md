# CharacterOS V3.7.7 Replay Calibration Guard Report

## Purpose

Polish the replay calibration layer by adding runtime structure validation.

This pass did not change physics behavior.

## Change

Added:

```text
validateReplayCalibrationSuite()
```

in:

```text
src/core/trace/replayCalibration.ts
```

The validator checks:

```text
suite title
suite passed flag
non-empty results
result scenario/title/summary/checks
check name/severity/message/value
passed flags matching fail severities
```

## API Integration

`GET /api/trace/replay/calibration` now validates the generated suite before returning it.

Invalid structure returns:

```text
500
```

## Tests

Updated:

```text
tests/core/replayCalibration.test.ts
```

Covered:

```text
valid suite passes validation
suite with failed check but passed=true is rejected
```

## Verification

Targeted verification passed:

```text
npm test -- tests/core/replayCalibration.test.ts tests/app/traceReplayCalibrationRoute.test.ts
npm run build
```

## Result

Replay calibration now has the same kind of guardrail as replay artifacts and replay summaries.

This makes the calibration dashboard safer because a malformed calibration payload will fail loudly instead of being silently rendered.

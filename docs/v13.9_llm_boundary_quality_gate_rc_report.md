## 质量门

新增 `src/core/audit/llmBoundaryQualityGate.ts`，统一验证以下风险：

- Grounded Mock 成功路径。
- `allowLlm=false` 默认关闭路径。
- Provider timeout、error、throw 和空输出。
- 诊断、伪状态修改、缺安全声明和过度确定。
- 无证据断言、安全标签伪装、真假混合句。
- Provider/request 身份错配。
- 带密钥与 raw state 文本的 Provider exception。
- Unsafe source preflight。
- 非 Mock Provider 和 `networkAllowed=true` 配置。
- `finishReason=length` 截断输出。
- 每个 case 双次确定性重放和输入不可变验证。

每个最终交付结果必须同时满足：

```text
deliveredValidation = pass
deliveredGrounding = grounded
noMutation = true
noWritebackAuthority = true
networkUsed = false
```

## 最终边界加固

- `no_unsupported_claim` 仍交由 Grounding Checker 给出证据级结论。
- 截断、过度确定等其他 validation WARN 改为阻断交付并触发 fallback。
- execution ID 纳入 Provider config、network flag、完整 response 和 fallback provenance。
- Provider 只接收深拷贝且递归冻结的 DTO。
- Deterministic fallback 复检失败时抛出 `LlmBoundaryDeliveryError`，不返回文本。

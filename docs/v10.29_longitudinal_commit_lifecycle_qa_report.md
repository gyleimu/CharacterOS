## 4. V10 Commit Lifecycle

```
┌──────────┐     ┌──────────┐     ┌──────────┐
│  Preview  │────▶│   Apply   │────▶│ Rollback  │
│ (no mut)  │     │ (mutate)  │     │ (cleanup) │
└──────────┘     └──────────┘     └──────────┘
     │                │                  │
     ▼                ▼                  ▼
  Handoff          State +           Memories
  computed         Audit              removed
  (private)        recorded          Fingerprint
                                     restored
```

### Preview
- Computes `finalStateForCommit` from simulation result
- Strips `finalState` before public response (never leaks full state)
- Does NOT mutate state or persist anything
- Returns `LongitudinalCommitPreview` with commit surface, audit summary, rollback plan, governance decision

### Apply
- Requires `confirmation === "APPLY_LONGITUDINAL_COMMIT"`
- Validates `requestDigest` and `baseStateFingerprint` against server-recomputed values
- Checks governance (commitPolicy.enabled, no self-action, integrity valid)
- Writes state to repository, records audit entry
- Returns `applied` status with audit DTO and rollback plan reference

### Rollback
- Requires `confirmation === "ROLLBACK_LONGITUDINAL_COMMIT"`
- Looks up audit entry by `auditId` or `simulationId`
- Validates current state fingerprint matches `finalStateFingerprint` (stale write protection)
- Removes only generated memory seeds (not full state restore)
- Records rolled_back audit status transition (preserves `appliedAt`)

## 6. Known Limitations

1. **No cross-file ACID transaction**: State write (physics_states.json) and audit write (longitudinal_commit_audit.json) use separate file locks. A crash between the two writes leaves state updated but audit stale (or vice versa).
2. **No autonomous background scheduler**: Characters do not run without explicit API calls. The "characters should continue changing even when nobody is watching" vision is not yet implemented.
3. **No relationship system**: Strictly single-character. No cross-character dependencies.
4. **No multi-character society**: Per long-term roadmap, this is V20+.
5. **No world simulation**: Character-internal states only.
6. **Commit deferred in dry-run simulate route**: The `POST /life/simulate` route with `commitPolicy.enabled=true` runs commit on an internal clone but warns "Commit is deferred." Full persistence goes through the commit/preview → commit/apply path.
7. **PREVIEW_TIMESTAMP is hardcoded**: Preview uses a fixed timestamp for deterministic audit IDs. Apply uses real wall-clock time.
8. **Rollback removes generated memories only**: No full snapshot restore; only memory seeds listed in the rollback plan are removed.

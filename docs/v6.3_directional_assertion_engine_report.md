## 1. 新增文件

### `src/core/benchmark/directionalAssertion.ts`（305 行）

**Pure assertion evaluation engine. 不调用 CharacterPhysicsEngine，不读写 CharacterPhysicsState。**

#### Exported functions

| 函数 | 签名 | 用途 |
|------|------|------|
| `evaluateBenchmarkAssertion` | `(params: EvaluateAssertionParams) => BenchmarkAssertionResult` | 评估单个断言 |
| `evaluateBenchmarkAssertions` | `(params: EvaluateAssertionsParams) => EvaluateAssertionsResult` | 批量评估多个断言 |
| `extractNumericValue` | `(obj: Record<string, unknown>, path: string) => number` | 从对象路径提取数值 |

#### Supported tolerance modes

| Mode | 评估逻辑 | 适用场景 |
|------|---------|---------|
| `directional` | `delta >= minDelta` (increase) / `delta <= -minDelta` (decrease) / `\|delta\| < minDelta` (unchanged) | 行为方向断言 |
| `bounded` | `after ∈ [minBound, maxBound]` | 值范围约束 |
| `monotonic` | series is non-decreasing/increasing or non-increasing/decreasing | 多 tick 连续衰减/恢复 |
| `invariant` | `\|delta\| ≤ epsilon` | 结构性不变量 |
| `approximate` | `\|delta\| ≤ epsilon` | 浮点近似检查 |
| `exact` | `\|delta\| ≤ epsilon` (default epsilon 1e-9) | 精确等值 |

#### Expected directions handled

| Direction | 行为 |
|-----------|------|
| `increase` | after > before by at least minDelta |
| `decrease` | after < before by at least minDelta |
| `unchanged` | `\|delta\| < minDelta` |
| `bounded_above` | after ≤ bound |
| `bounded_below` | after ≥ bound |

#### Edge cases handled

| Case | Result |
|------|--------|
| NaN before/after | `passed = false`, explanation mentions NaN |
| Infinity/−Infinity | `passed = false`, explanation mentions infinite |
| unknown tolerance mode | `passed = false`, explanation mentions unknown mode |
| empty series for monotonic | `passed = false`, requires ≥ 2 values |
| missing observation | `passed = false`, treated as NaN |

#### Explanation format

Every assertion result includes a human-readable explanation that describes the expected and observed values:

```
expected trust to decrease, observed 0.5 → 0.4 (Δ-0.1) ✓
expected fear to increase, but observed 0.5 → 0.45 (Δ-0.05, below minDelta 1e-6)
```

## 2. 为什么方向判断优先

```text
V6.3 坚持 V6.0 Design Charter 的核心哲学：

  测量方向，不是精确值。

1. directional tolerance 不要求精确 delta：
   "trust should decrease" 是可测量的。
   "trust should drop by exactly 0.128" 是脆弱的。

2. monotonic tolerance 检查连续方向：
   如果 memory recency 在 7d, 14d, 30d 三个 tick 上
   应该是 monotonically decreasing，不应出现反弹。

3. bounded tolerance 检查安全范围：
   coordinate values 应该在 [0, 1]。
   memory count 应该 >= 0。
   这些是结构性不变量，不是行为断言。

4. exact/invariant 只用于结构性不变量：
   daysElapsed=0 → no change
   final state.boundary === homeostasis.regulatedBoundary (D10)
   这些是系统级不变量，epsilon tolerance 合理。
```

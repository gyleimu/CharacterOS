## 3. 每个变量的计算

| 字段 | 计算方式 |
|------|---------|
| `currentValue` | 直接从 state 读取 |
| `normalizedValue` | `clamp01(currentValue)` |
| `baselineValue` | 每变量预定义（来自 default state） |
| `deviationFromBaseline` | `currentValue - baselineValue` |
| `homeostaticPressure` | `clamp01(abs(deviation) * 1.5)` |
| `stabilityRisk` | `abs(dev) >= 0.35 → high; >= 0.18 → medium; else low` |
| `direction` | `abs(dev) <= 0.05 → near_baseline; dev > 0 → above; dev < 0 → below` |
| `timeScale` | 每变量预定义（fast/medium/slow/structural） |

**预定义 baseline 来源：** `defaultMetaState()`, `createPsychologicalBoundary()`, `defaultRewardState()`, `defaultHomeostasisState()`, `defaultBoredomState()`, `neutralCoordinate().values`

## 5. 设计决策

### 为什么是观察层

```text
InternalStateField 不替代 CharacterPhysicsState。
它是 CharacterPhysicsState 的只读投影。
所有值都从已有 state 计算，不引入新状态源。
```

### 为什么不覆盖所有状态字段

```text
当前覆盖 46 个变量，聚焦于时间演化域。
以下内容有意不覆盖：
- PersonalityCoordinate 全部 9 维（只取 trust/fear/attachment/control）
- DerivedCharacterState（这是 Phase 2 的只读推导，不是时间演化）
- personality (BigFive) — 从 coordinate 推导，冗余
- galaxy / particles / clusters — 结构性数据，不是时间变量
- identity — 静态数据
- biologicalNature — V4 前置原型
```

### 为什么 baseline 是硬编码

```text
V3 没有存储 per-character baseline。
baseline 当前是 defaultState 的值。
未来 V4 可以在 CharacterPhysicsState 中增加 baseline 字段，
让不同角色有不同的 homeostatic target。
```

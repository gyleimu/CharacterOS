## 2. 新增/修改文件

### 新增：`src/core/temporal/adapters/boredomTemporalAdapter.ts`（102 行）

- `BoredomTemporalAdapterInput` — 与 `updateBoredomForTick` 完全一致的输入参数
- `BoredomTemporalAdapterResult` — 包含 V3 trace + phase metadata + TemporalProcessResult
- `executeBoredomTemporalAdapter(input)` — 委托给 `updateBoredomForTick`，返回等价结果

**设计决策：**
- Adapter 不直接写 state — 调用方负责 `state.boredomState = result.trace.after`
- 返回完整的 V3 `BoredomTickTrace`（before/after/delta/reasons 等全部保留）
- 额外返回 `phase` metadata 用于 V3 phases 数组拼接
- 额外返回 `temporalResult` 用于 V4 UnifiedTickTrace 兼容

### 修改：`src/core/time/continuousTick.ts`（Phase 14）

**之前：**
```typescript
const boredom = updateBoredomForTick({
  boredom: state.boredomState,
  meta: state.metaState,
  reward: state.rewardState,
  boundary: state.boundary,
  daysElapsed
});
state.boredomState = boredom.after;
phases.push({
  name: "boredom",
  changedStates: ["boredomState"],
  reasons: boredom.reasons
});
```

**之后：**
```typescript
const boredomResult = executeBoredomTemporalAdapter({
  boredom: state.boredomState,
  meta: state.metaState,
  reward: state.rewardState,
  boundary: state.boundary,
  daysElapsed
});
state.boredomState = boredomResult.trace.after;
phases.push(boredomResult.phase);
```

**等价性：** `boredomResult.trace` === 原 `boredom` 变量（`updateBoredomForTick` 的返回值）。`boredomResult.phase` 包含相同的 `name: "boredom"`, `changedStates: ["boredomState"]`, `reasons: [...]`。

### 修改：`src/core/temporal/temporalProcessAdapterRegistry.ts`

boredom adapter 的 `implementationStatus` 从 `"adapter_shell"` 改为 `"delegated"`。

### 新增：`tests/core/temporal/boredomTemporalAdapter.test.ts`（10 tests）

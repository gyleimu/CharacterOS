| 常量 | 成员数 | 成员 |

#### 核心接口

| 接口 | 用途 |
|------|------|
| `BenchmarkInputEvent` | 确定性 benchmark event（无随机性） |
| `BenchmarkTickSpec` | Tick 规格（label, daysElapsed, memoryDecayRate） |
| `BenchmarkInitialStateSpec` | 初始状态来源（inline state 或 blueprint reference） |
| `BenchmarkScenario` | 场景 = initialState + events + ticks |
| `BenchmarkCase` | 完整 benchmark case = scenario + expectedDirections + metrics + tolerance |
| `BenchmarkExpectedDirection` | 方向断言（metricPath, direction, reason, optional bound/tolerance） |
| `BenchmarkMetricSelector` | 从哪个 source 读哪个 path 的哪个 subprocessKind |
| `BenchmarkTolerancePolicy` | 容差策略（mode + optional epsilon/minDelta/bounds） |
| `BenchmarkAssertionResult` | 断言评估结果（valueBefore/After, delta, passed, explanation） |
| `BenchmarkMetric` | 收集的数值（path, source, valueBefore/After, delta） |
| `BenchmarkResult` | Case 运行结果（verdict, assertionResults, metrics, warnings, explanation, durationMs） |

#### Helper functions

| 函数 | 签名 | 用途 |
|------|------|------|
| `isBenchmarkCategory` | `(v: unknown) => v is BenchmarkCategory` | Type guard |
| `isToleranceMode` | `(v: unknown) => v is ToleranceMode` | Type guard |
| `defaultBenchmarkTolerancePolicy` | `() => BenchmarkTolerancePolicy` | 默认 directional + minDelta |
| `describeTolerancePolicy` | `(p: BenchmarkTolerancePolicy) => string` | 稳定可读描述 |
| `normalizeBenchmarkCase` | `(k: BenchmarkCase) => BenchmarkCase` | 填充默认 tolerance，不修改输入 |

## 2. 与现有 impact.ts 的关系

```text
src/core/benchmark/impact.ts 保持不变。

impact.ts:  event impact scoring utility
  - calculateImpactScore(inputs) → ImpactScore
  - classifyImpact(value) → [ImpactBand, description]
  - 用于 V3 physics engine 的事件处理

benchmarkTypes.ts:  benchmark type system
  - 更高层的 benchmark 框架类型
  - 定义 benchmark case / scenario / assertion / result
  - 不调用 CharacterPhysicsEngine
  - 不影响 impact.ts 的任何使用

两者是互补关系：
  - impact.ts 是运行时工具（V3）
  - benchmarkTypes.ts 是类型定义层（V6）
```

## 4. 默认 Tolerance 设计决策

```text
defaultBenchmarkTolerancePolicy() 返回：
  { mode: "directional", minimumAbsoluteDelta: 1e-6 }

设计理由：
  - directional 优先：不追求精确数值（"trust = 0.347"）
  - minimumDelta 防止浮点噪音：delta < 1e-6 被视为 "unchanged"
  - exact 仅用于结构性不变量（D10 overwrite, coordinate integrity）
  - 每个 case/assertion 可以覆盖默认 tolerance
```

# CharacterOS V2.19 Trace Summary Controls 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.19 Trace Summary Controls
```

这一阶段继续提升 V2 replay 轨道总览的可用性。目标不是扩大内核，而是让已有 replay summary 更容易比较。

## 本阶段新增

### 1. 方向过滤

更新：

```text
src/components/character/PhysicsDashboard.tsx
```

“Replay 轨道总览”新增：

```text
方向过滤
```

可选：

```text
全部轨道
defensive_drift / 防御漂移
recovery_drift / 修复漂移
```

选项来自当前 `replayIndex.directionCounts`，会跟随 API 刷新后的数据变化。

### 2. 排序控件

新增：

```text
排序
```

支持：

```text
主星团质量增量
trust 漂移幅度
fear 漂移幅度
标题
```

默认按主星团质量增量排序，让最强的 replay 轨道优先浮出。

### 3. 空状态

当过滤条件没有匹配 replay 时，显示：

```text
没有符合当前过滤条件的 replay 轨道。
```

### 4. 样式

更新：

```text
src/app/styles.css
```

新增：

```text
trace-summary-controls
```

桌面端并排显示，移动端自然堆叠。

## 当前意义

Replay 轨道总览现在可以回答：

```text
只看防御漂移有哪些？
只看修复漂移有哪些？
哪条轨道的主星团质量增长最大？
哪条轨道对 trust / fear 的改变最大？
```

这让 Dashboard 更接近“人格星云调试台”，而不是单纯的数据展示。

## 1. DTO Inventory

| DTO | Purpose |
|-----|---------|
| `AgentSessionConfig` | Session lifecycle: policy, safety, LLM mode |
| `AgentTurnInput` | Normalized input from external source |
| `AgentEventCandidate` | Extracted event draft with confidence |
| `AgentPolicyDecision` | block/preview/apply/confirm decision |
| `AgentGroundingBundle` | Context bundle (no raw state) |
| `AgentReplyPlan` | Structured reply blueprint (not prose) |
| `AgentWritebackPlan` | Conversation → character event plan |
| `AgentTurnResult` | Complete turn output |
| `AgentSafetyNotice` | Structured safety constraint |

## 2. Safe Defaults

| Setting | Default |
|---------|---------|
| `writebackPolicy` | `require_user_confirmation` |
| `safetyMode` | `strict` |
| `llmMode` | `disabled` |
| `readOnlyDefault` | `true` |
| `noMultiCharacter` | `true` |
| `noDiagnosis` | `true` |
| `consentForWriteback` | `false` |
| `requiresPreview` | `true` |

## 3. Writeback Policy Model

| Policy | Behavior |
|--------|----------|
| `never` | Conversation never affects character |
| `preview_only` | User sees what would happen, no write |
| `auto_apply_safe_events` | Safe events auto-applied (with audit) |
| `require_user_confirmation` | User must confirm "apply" |

## 4. LLM Boundary

- `llmMode: disabled` by default
- `AgentReplyPlan.llmAllowed: false` by default
- `llmBoundaryInstructions` is a string field for future LLM guardrails
- LLM can NEVER mutate core state
- LLM output is always a suggestion, never ground truth

## 5. No Raw State Exposure

- `AgentGroundingBundle.omittedRawState: true`
- All state data comes through V11 Explorer DTOs (qualitative only)
- No `CharacterPhysicsState` in any agent DTO

## 6. Relationship / Multi-Character Prohibition

- `AgentSessionConfig.noMultiCharacter: true` on every session
- No `partnerId`, `relationshipType`, `multiCharacter` fields
- Agent SDK is strictly single-character

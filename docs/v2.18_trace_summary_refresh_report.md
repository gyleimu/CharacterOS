# CharacterOS V2.18 Trace Summary Refresh 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.18 Trace Summary Refresh
```

这一阶段把 V2.17 的 Trace Summary API 接入 Dashboard 运行链路。Replay 轨道总览不再只是服务端初始数据，也可以在前端通过 API 刷新。

## 本阶段新增

### 1. Dashboard Replay Index State

更新：

```text
src/components/character/PhysicsDashboard.tsx
```

新增状态：

```text
replayIndex
replayStatus
```

初始值来自服务端传入的 `traceSummaryIndex`。

### 2. API Refresh

新增：

```text
refreshReplaySummary()
```

请求：

```text
GET /api/trace/replay/summary
```

成功后更新：

```text
replayIndex
replayStatus
```

失败时显示：

```text
刷新失败
```

### 3. Dashboard Button

“Replay 轨道总览”面板新增：

```text
刷新轨道
```

按钮，用于手动重新读取 summary index。

### 4. UI 样式

更新：

```text
src/app/styles.css
```

新增：

```text
panel-title-row
```

用于面板标题、状态和操作按钮的稳定布局。

## 当前意义

Replay 数据链路现在形成了完整闭环：

```text
Core replay scenarios
-> Summary index
-> API
-> Dashboard state
-> User refresh
```

这让 replay 轨道总览从静态展示变成可刷新的产品调试面板。

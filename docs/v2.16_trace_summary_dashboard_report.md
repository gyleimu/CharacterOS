# CharacterOS V2.16 Trace Summary Dashboard 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.16 Trace Summary Dashboard
```

这一阶段把 V2 replay summary index 接入现有 Dashboard。目标不是做 3D 星云，也不是扩大系统边界，而是让 Personality Galaxy 的 replay 实验轨道能被直观看到。

## 本阶段新增

### 1. 服务端生成 Summary Index

更新：

```text
src/app/page.tsx
```

页面服务端现在会生成：

```text
buildTraceReplaySummaryIndex({ daysPerStep: 14 })
```

并传入 Dashboard。

### 2. Replay 轨道总览面板

更新：

```text
src/components/character/PhysicsDashboard.tsx
```

```text

```

显示：

```text
固定 replay 场景数量
defensive_drift / recovery_drift 分布
每条 replay 的主导星团
trust 漂移
fear 漂移
trust 速度增量
主星团质量增量
```

### 3. Dashboard 样式

更新：

```text
src/app/styles.css
```

```text
trace-summary-header
```

布局保持调试面板风格：密集、可扫读、不过度装饰。

## 当前意义

V2 replay 数据链路现在已经从纯文件输出进入 Dashboard：

```text
TraceReplayScenario
-> TraceReplayArtifact
-> TraceReplaySummary
-> TraceReplaySummaryIndex
-> Dashboard Replay 轨道总览
```

这让用户可以在界面上直接看到：

```text
哪些 replay 是防御漂移
哪些 replay 是修复漂移
哪个星团主导了人格牵引
长期事件让 trust / fear 如何变化
```

# CharacterOS Architecture

## Philosophy

CharacterOS is not a chatbot.

CharacterOS is an experiment on experience-driven personality evolution.

The goal is not to write stories.

The goal is to simulate people.

---

# Core Principle

Characters are not defined by traits.

Characters are defined by experiences.

Experiences
↓
Memories
↓
Beliefs
↓
Deficiencies
↓
Desires
↓
Emotions
↓
Decisions
↓
New Experiences

This loop never stops.

---

# Desires

Desires should not be predefined.

Desires emerge from deficiencies.

Examples:

Lack of love
→ desire for intimacy

Lack of safety
→ desire for power

Lack of recognition
→ desire for fame

Deficiencies drive behaviors.

Behaviors create new experiences.

---

# Beliefs

Experiences do not directly shape personality.

Interpretations of experiences shape personality.

The same betrayal can produce:

* kindness
* paranoia
* coldness

Beliefs are the bridge.

---

# Plot Emergence

Traditional writing:

Plot
↓
Characters react

CharacterOS:

World
↓
Characters
↓
Interactions
↓
Plot emerges

Characters should drive the story.

Not serve the plot.

---

# Memory Space

Memory should not be represented as a list.

Memory behaves more like a space.

Important memories stay close to the center.

Trivial memories drift toward the boundary.

Frequently activated memories become stronger.

Forgetting is gradual.

---

# Memory Clusters

Similar memories attract each other.

Examples:

Mother leaving

First breakup

Fear of abandonment

Loneliness

These memories form an abandonment cluster.

Clusters eventually become personality.

---

# Trauma

Trauma behaves like a gravity well.

New experiences are attracted into it.

Partner not replying

↓

Fear of abandonment

↓

Old memories reactivate

Trauma amplifies reactions.

---

# Memory Reinforcement

Every recall slightly increases influence.

Thousands of tiny reinforcements create personality.

Personality changes slowly.

Not suddenly.

---

# Personality Field

Personality is not a table.

Personality is a dynamic field.

Dimensions may include:

Memory

Emotion

Belief

Relationship

Value

Desire

Need

Time

A 30-year-old and a 60-year-old should have fundamentally different personality fields.

---

# Ideal Self

Each character has:

Current Self

Ideal Self

Characters spend their entire lives approaching their Ideal Self.

But never fully reach it.

---

# Memory Galaxy Model

Personality may resemble a galaxy.

Memories are stars.

Clusters are constellations.

Trauma becomes black holes.

The center contains core memories.

Outer layers contain weak memories.

Memories near the edge may stay for years before disappearing.

Forgotten memories may return because of smells, music or people.

---

# Physical Model

Memory node = particle

Importance = mass

Similarity = attraction

Trauma = gravity well

Time = decay

Center = personality core

Boundary = forgetting layer

Clusters = galaxies

Current events propagate like waves.

Activated memories create the current mental state.

The LLM reasons on top of this state.

---

# Context Loading

Context should not contain an entire life.

Context loading is hierarchical.

Layer 1:

Core memories

Always loaded.

Layer 2:

Near-core memories.

Loaded when relevant.

Layer 3:

Middle memories.

Loaded only when strongly related.

Layer 4:

Boundary memories.

Usually ignored.

Strong triggers may reactivate them.

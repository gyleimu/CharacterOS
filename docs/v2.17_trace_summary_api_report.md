# CharacterOS V2.17 Trace Summary API 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.17 Trace Summary API
```

这一阶段把 V2 replay summary index 暴露为只读 API，方便 Dashboard 后续动态刷新，也方便外部实验工具直接读取 replay 轨道总览。

## 本阶段新增

### 1. Trace Summary API

新增：

```text
src/app/api/trace/replay/summary/route.ts
```

接口：

```text
GET /api/trace/replay/summary
```

返回：

```text
{
  index: TraceReplaySummaryIndex
}
```

接口内部会先执行：

```text
buildTraceReplaySummaryIndex({ daysPerStep: 14 })
validateTraceReplaySummaryIndex(index)
```

如果校验失败，返回 500。

### 2. API Contract

更新：

```text
src/appContracts/characterPhysics.ts
```

新增：

```text
GetTraceReplaySummaryResponse
```

### 3. Route Test

新增：

```text
tests/app/traceReplaySummaryRoute.test.ts
```

验证：

```text
GET 返回 200
scenarioCount > 0
summaries.length === scenarioCount
directionCounts 包含 defensive_drift

```

### 4. Vitest Alias

更新：

```text
vitest.config.ts
```

新增：

```text
@ -> src
```

让 API route 在 Vitest 中可以直接使用和 Next 一致的导入路径。

## 当前意义

V2 replay 数据链路现在变成：

```text
TraceReplayScenario
-> TraceReplayArtifact
-> TraceReplaySummary
-> TraceReplaySummaryIndex
-> Dashboard
-> API
```

这让 replay 轨道开始从“内部 demo 数据”变成可被产品层和实验层共同读取的稳定接口。

```typescript
runLongTermAccumulationAudit({
  baselineState: CharacterPhysicsState,

  followUpDecisionScenario: EnvironmentSeed,
  expectedTrend?: {

    personalityDistance?: "growing" | "stable" | "shrinking",
  },
  maxStepOnePersonalityRatio?: number,     // default 0.55
  maxNeutralPersonalityDistance?: number,  // default 0.04

```

### 2.3 Output

- `stepResults[]` — per-event accumulated deltas (personality, trust, fear, boundary, memory, belief)
- `accumulationCurve` — per-step personality distance, trust, fear, neuroticism, boundary stress/integrity curves
- `saturationMetrics` — last-step/total ratios for personality, boundary, trust, fear
- `stepOneJumpRatios` — step-1 contribution ratio (lower = more gradual)
- `accumulationVerdict` — PASS/WARN/FAIL with warnings and reasons

### 2.4 Verdict Rules

| Check | Threshold | Severity |
|-------|-----------|----------|
| Step-1 personality ratio > 0.55 | 55% | WARN |
| No saturation after 4+ steps | >60% last-step/total | WARN |
| Expected monotonic trend violated | >1 violation | WARN |
| Neutral distance > max | 0.04 default | WARN |
| Linear unbounded growth detected | recent-avg > early-avg * 0.9 | WARN |

## 3. How the System Accumulates

### 3.1 Mechanism

The system's accumulation mechanics are built into the galaxy simulation:

1. **Impact Cluster** (`absorbImpactParticle`): Each event particle is absorbed into its category cluster by weighted averaging. Later particles have decreasing weight (oldWeight = mass / totalMass).

2. **Galaxy Force** (`calculateClusterForce`): Force ∝ mass × stability × density / distance². As a cluster accumulates more events, its mass grows, creating stronger gravitational pull on the personality core.

3. **Momentum Drift** (`applyMomentumDrift`): Velocity = oldVelocity × 0.82 + force × learningRate. Momentum smooths single-event changes. Velocity clamped to ±0.08 per step.

4. **Boundary Impact** (`applyBoundaryImpact`): V10.70 added event-category-aware boundary handling — support events reduce stress, negative events increase it.

### 3.2 Natural Properties

- **Personality IS a slow channel**: learningRate (0.02–0.03) × momentum (0.82) × velocity clamping (±0.08) ensures gradual drift
- **Cluster mass grows with repetition**: repeated events in the same category accumulate cluster mass, amplifying future forces
- **Trust is harder to rebuild than to break**: abandonment delta (-0.09 trust) > support delta (+0.06 trust)
- **Boundary stress responds faster**: V10.70 boundary handling provides immediate stress relief for support events

### 4.2 Repeated Support — Sensitive Baseline (Lin Fan)

| Metric | Baseline | After 5 Events | Delta |
|--------|----------|-----------------|-------|
| **Trust** | 0.257 | 0.256 | **-0.001** (stable) |
| **Fear** | 0.823 | 0.825 | **+0.002** (stable) |
| **Personality Distance** | 0 | 0.006 | **+0.006** |

**Accumulation Curve:**
```
Step 1: dist=0.002  stress=1.909
Step 2: dist=0.004  stress=1.623
Step 3: dist=0.006  stress=1.337
Step 4: dist=0.006  stress=1.050
Step 5: dist=0.006  stress=0.764
```

**Verdict: WARN** (trust/fear show minimal movement — personality barely shifts)

**Assessment**: Boundary healing works very well (V10.70 mechanism). Personality saturation is visible — after step 3, personality distance stops growing. However, trust and fear barely move even after 5 support events. This is because:
1. Support coordinate delta (+0.06 trust) is smaller than abandonment (-0.09)
2. Learning rate (0.02) is slow for personality change
3. Existing abandonment cluster (from seeded experiences) creates counter-force

This is psychologically plausible — rebuilding trust after deep attachment wounds takes more than 5 supportive events — but it means the system as currently configured makes trust repair slower than trust erosion. This is a **real finding**, not a test failure.

## 5. Key Findings

### ✅ Confirmed

1. **Personality IS a slow channel**: Step-1 contributes only 4.3% of total accumulated drift for sensitive, 0.8% for secure baseline
2. **Repeated events DO accumulate**: Trust/fear move monotonically across repeated negative events
3. **Sensitive vs secure differentiation**: 8× accumulation difference
4. **Secure still responds**: Secure baseline shows small but measurable trend after 5 major events (not completely immune)
5. **Neutral events stay stable**: 0.023 distance after 5 neutral events (well within bounds)
6. **Boundary healing works**: V10.70 support boundary stress reduction accumulates across events (1.909 → 0.764)
7. **Trust is harder to rebuild than break**: Support events slow/prevent decline but don't quickly rebuild trust

### ⚠️ Limitations (Reported as WARN)

1. **No natural saturation in galaxy forces**: Cluster mass grows linearly with events → forces accelerate rather than saturate. This is a real system limitation.
2. **Trust rebuild from support is very slow**: With learning rate 0.02 and support delta +0.06 (vs abandonment -0.09), trust rises < 0.001 over 5 support events. This may be too slow for narrative purposes.
3. **Cluster competition**: Existing abandonment cluster from seeded experiences competes with support cluster, making trust repair even harder in pre-wounded characters.

## 7. Recommendation: V10.72

建议继续推进 V10.72，重点解决 V10.71 发现的真实限制：

1. **Galaxy force saturation** — 在 `calculateClusterForce` 中对 mass 使用 sqrt 或 log 衰减，使后期事件的边际影响力递减
2. **Trust repair asymmetry tuning** — 评估 support delta 是否需要微调，或引入"安全证据累积阈值"机制
3. **Cluster decay** — 旧事件的影响应随时间衰减，不仅被新事件稀释

这些是增强性改进。V10.71 的核心成果是：**验证了人格作为 slow channel 的假设成立，并建立了可量化的累积校准审计框架。**

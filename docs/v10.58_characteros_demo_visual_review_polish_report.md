## Overview

V10.58 adds three visual review improvements to the static offline demo, making it easier for human reviewers to understand scenario differentiation, internal life dynamics, and memory significance at a glance.

## Changes

### 1. Scenario Strategy Matrix (New)

A strategy-vs-scenario matrix is now rendered between the "固定场景观察" intro card and the detailed comparison table on the Scenarios tab.

**Implementation details:**
- Columns are dynamically discovered from `data.scenarios[].strategyId` / `data.scenarios[].strategy` — no hardcoded strategy names
- The active strategy for each scenario is marked with ● (`.matrix-hit`)
- Non-matching cells show · (`.matrix-miss`) in a muted color
- Strategy column headers use the existing `strategyBadge()` for color consistency
- Rendered by new `renderScenarioMatrix()` function in the demo JS

**User value:** A reviewer can immediately see that 5 scenarios activate 4 distinct strategies, with "确认关系信号" used in 2 scenarios, and the other 3 strategies each used once.

### 2. Life Preview Self-Action Candidate Bars (Redesigned)

The `selfActionCandidates` list has been converted from text-only `.mini-card` elements to visual bar charts.

**Implementation details:**
- New `candidateBar(c)` function replaces `candidate(c)`
- Each candidate renders as a `.candidate-bar` with:
  - Header row: action name (`.candidate-name`) + score percentage (`.candidate-score`)
  - Horizontal bar (`.bar-track`) with width proportional to score
  - Bar color shifts at thresholds: >25% uses warmer purple, ≤15% uses default blue
  - Reasons list (`.candidate-reasons`) preserved below the bar
- The warning "候选动作不会自动执行，只用于观察内部生命倾向" remains prominently displayed above the bars

**User value:** The relative strength of internal drives becomes visually scannable — the reviewer sees which action dominates without reading numbers.

### 3. Today Memory Card Deterministic Hints (New)

Each recent memory card now includes a human-readable interpretation line generated by a deterministic helper.

**Implementation details:**
- New `memoryHint(emotion, importance)` function uses pure conditionals:
  - `importance > 0.8` + `emotion === "fear"` → "高重要性恐惧记忆，容易在当前压力下被重新激活并影响解释。"
  - `importance > 0.8` + `emotion === "relief"` → "高重要性支持性记忆，可能提供修复证据和安全感来源。"
  - `importance > 0.8` + `emotion === "sadness"` → "高重要性悲伤记忆，容易加深当下的无力感。"
  - `importance > 0.5` + `emotion === "fear"` → "中等重要性恐惧记忆，可能在类似场景里被触发。"
  - `importance > 0.5` + `emotion === "relief"` → "中等重要性支持性记忆，偶尔被作为参考调用。"
  - Fallbacks for low importance and other emotions
- Memory header redesigned: emotion name + importance percentage bar (`.memory-header`, `.memory-imp`)
- Hint rendered as `.memory-hint` with a left border accent
- **No LLM, no API, no state mutation** — pure deterministic conditions

**User value:** The reviewer no longer needs to interpret raw emotion + importance numbers; each memory card explains why that memory matters to the character's current state.

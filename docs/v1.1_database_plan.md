# CharacterOS V1.1 Database Plan

V1.0 已完成最小 Character Decision Engine 闭环。

V1.1 的目标是把当前 JSON 文件仓库逐步迁移到 Prisma + SQLite。

## 当前策略

不要一次替换现有运行链路。

先新增：

```text
prisma/schema.prisma
src/db/prismaClient.ts
```

保留当前：

```text
FileCharacterPhysicsRepository
data/physics_states.json
```

等 Prisma repository 测试通过后，再切换 server singleton。

## 数据模型

当前 schema 包含：

```text
Character
EventLog
MemoryNode
ImpactCluster
ImpactParticle
```

后续可继续新增：

```text
BeliefState
NeedDeficiency
DesireState
BehaviorDecision
Relationship
WorldlineSnapshot
```

## 迁移步骤

1. 生成 Prisma client。
2. 创建 SQLite dev database。
3. 实现 PrismaCharacterPhysicsRepository。
4. 写 repository round-trip 测试。
5. 加 JSON -> SQLite 迁移脚本。
6. 切换 Next server singleton 到 Prisma repository。
7. 保留 File repository 作为 fallback/dev fixture。

## 运行命令

```bash
npx prisma generate
npx prisma db push
```

需要 `.env` 中配置：

```text
DATABASE_URL=file:./dev.db
```

## 2. ReplyPlan vs Final Reply

| ReplyPlan | Final Reply |
|-----------|------------|
| Structured blueprint | Natural language message |
| Deterministic | May involve LLM |
| Evidence-grounded facts | May paraphrase |
| Safety notices embedded | Safety constraints applied |
| Intent + tone + outline | Prose with tone |
| **V12.6 implements** | **Deferred to future LLM boundary** |

## 3. Intent Selection

| Condition | Intent |
|-----------|--------|
| Decision = block | `refuse_unsafe` |
| Decision = confirmation_required | `ask_confirmation` |
| Has candidates | `preview_event` |
| Has evidence, no candidates | `explain_state` |
| Has evidence (apply) | `summarize_history` |
| No candidates, no evidence | `clarify_input` |
| Fallback | `neutral_ack` |

## 4. Tone Selection

| Condition | Tone |
|-----------|------|
| Safety = unsafe | `safety_first` |
| Safety mode = strict | `cautious` |
| Journal/story input | `reflective` |
| Plugin/tool input | `technical` |
| Block or confirmation | `concise` |
| Default | `calm` |

## 5. Grounded Facts

Every fact derived from AgentGroundingBundle:
- State headline + emotion + stress
- Top 2 beliefs with strength
- Behavior tendency

## 6. LLM Boundary Instructions (when llmAllowed)

Do NOT produce medical/psychological diagnosis.

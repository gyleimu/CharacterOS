## 1. Executive Summary

V10.36 implements the influence factor drift animation as a pure data system. Each node's drift is modeled as a slow elliptical oscillation derived from its V10.33 drift vector. Frame sampling is deterministic at any arbitrary timeMs. No `requestAnimationFrame`. No `Date.now`. No `Math.random`. No DOM.

## 4. Motion Modes

| Mode | Amplitude | Zoom Gating | When |
|------|-----------|-------------|------|
| `still` | 0 | Any | reducedMotion=true, or mode=still, or L0/L1 with mode=subtle |
| `subtle` | drift.magnitude × 0.15 (max 0.5px) | L2+ | mode=subtle, or mode=visible at L2-L3 |
| `visible` | drift.magnitude (max 2px) | L4 only | mode=visible at L4 |

### Motion mode resolution:

```
reducedMotion?           → still (always)
mode === "still"?        → still
mode === "subtle" + L0/L1? → still (no visible drift at lowest zoom)
mode === "subtle" + L2+? → subtle
mode === "visible" + L0-L3? → subtle (downgraded)
mode === "visible" + L4? → visible
stable nodes (magnitude < 0.1)? → still
```

## 5. Elliptical Drift Model

Each node with a drift vector gets an elliptical orbit:

```
semiMajor = clamp(drift.magnitude × modeMultiplier, 0, maxOffset)
semiMinor = semiMajor × 0.3

offsetX(t) = semiMajor × cos(ωt + φ) × cos(θ) - semiMinor × sin(ωt + φ) × sin(θ)
offsetY(t) = semiMajor × cos(ωt + φ) × sin(θ) + semiMinor × sin(ωt + φ) × cos(θ)

where:
  ω = 2π / periodMs
  φ = deterministicPhase(seed, nodeId)
  θ = driftVector.angle
  periodMs = durationMs × periodMultiplier (0.6–2.0, deterministic per node)
```

- **Semi-major axis** aligns with the drift direction angle
- **Semi-minor axis** is perpendicular (30% of major)
- **Period** varies per node (0.6–2.0 × base duration) so nodes don't oscillate in sync
- **Phase** is deterministic per node so they start at different positions

## 6. Frame Sampling

```typescript
sampleGalaxyDriftFrame(track, timeMs) → GalaxyDriftFrame
```

- `timeMs` is arbitrary — wraps using `timeMs % track.periodMs`
- Same `(track, timeMs)` → identical `GalaxyDriftFrame` every time
- Zero allocations inside the hot path (pure computation)
- No state mutation — just reads track params, returns new frame object

## 7. Integration with V10.34 Renderer

The optional `applyDriftFrameToCommands` helper applies drift offsets to render commands:

```
V10.34 Command List
       │
       ▼
sampleGalaxyDriftSnapshot(animation, currentTimeMs) → Map<nodeId, offset>
       │
       ▼
applyDriftFrameToCommands(commands, frame) → NEW commands with circle/driftVector positions shifted
```

This does NOT modify the original commands. It returns new ones. The V10.34 renderer API is unchanged.

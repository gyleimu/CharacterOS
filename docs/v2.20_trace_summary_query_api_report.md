# CharacterOS V2.20 Trace Summary Query API 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.20 Trace Summary Query API
```

这一阶段把 V2.19 Dashboard 里的 replay 过滤和排序规则下沉到 core 与 API。这样 Dashboard、API 和未来 Benchmark 工具可以共享同一套 summary query 逻辑。

## 本阶段新增

### 1. Core Summary Query

更新：

```text
src/core/trace/traceReplaySummary.ts
```

新增：

```text
TraceReplaySummarySortMode
TraceReplaySummaryIndexQuery
applySummaryQuery(summaries, query)
```

支持：

```text
direction
sort
```

排序模式：

```text
dominantMassDelta
trustDrift
fearDrift
title
```

### 2. API Query Parameters

更新：

```text
src/app/api/trace/replay/summary/route.ts
```

接口现在支持：

```text
GET /api/trace/replay/summary?direction=recovery_drift&sort=trustDrift
```

无效 `sort` 会被忽略，回退到默认排序。

### 3. Dashboard Refresh Query

更新：

```text
src/components/character/PhysicsDashboard.tsx
```

点击“刷新轨道”时，会携带当前控件状态：

```text
direction
sort
```

### 4. Route 测试

更新：

```text
tests/app/traceReplaySummaryRoute.test.ts
```

新增验证：

```text
direction=recovery_drift 只返回修复漂移
sort=trustDrift 按 trust 漂移幅度排序
directionCounts 与过滤后的 summaries 一致
```

## 当前意义

Replay summary 现在不只是“前端自己筛选”，而是具备稳定的查询入口：

```text
Core query
-> API query
-> Dashboard controls
```

这让 replay 轨道总览开始接近真正的数据查询面板。

# CharacterOS V2.5 Trace Replay 阶段报告

## 版本结论

当前项目进入：

```text
CharacterOS V2.5 Trace Replay
```

这一阶段仍然只推进 V2 内核可审计性，没有扩张 V3/V4，也没有做 3D 可视化。

目标：

```text
将多步 trace 生成稳定的本地调试 artifact。
```

## 本阶段新增

### 1. Trace Replay Demo

新增：

```text
src/core/demo/traceReplayDemo.ts
```

新增 npm script：

```text
npm run demo:trace
```

运行后会输出：

```text
outputs/trace_replay_demo.json
```

默认场景：

```text
abandonment
-> abandonment
-> support
-> support
```

用于观察：

```text
创伤星团如何累积负向 trust force
支持星团如何产生反向 trust/fear force
为什么修复不会瞬间抹除创伤惯性
```

### 2. GalaxyStepTrace 增加 Per-cluster Forces

更新：

```text
src/core/trace/galaxyTrace.ts
```

`GalaxyStepTrace` 新增：

```text
forces
```

每个 force 包含：

```text
clusterId
category
distance
magnitude
vector
```

这样 trace 不只显示总力，也能回答：

```text
哪个星团正在拉动人格核心？
每个星团分别贡献了多少 trust / fear 力？
```

### 3. Simulation Demo 输出增强

更新：

```text
src/core/demo/simulationDemo.ts
```

现在输出：

```text
boundaryPhase
driftMultiplier
forceTrust
forceFear
velocityTrust
velocityFear
```

## Artifact 示例意义

在 `abandonment_then_repair` 场景中，trace 会显示：

```text
abandonment cluster: trust < 0, fear > 0
support cluster: trust > 0, fear < 0
total force: 可能仍被 abandonment cluster 主导
```

这符合 Character Physics 的原则：

```text
修复性经历会产生反向力，
但不会瞬间删除长期创伤星团的惯性。
```

# CharacterOS V2.26 Continuous Tick API Report

## Version

```text
CharacterOS V2.26 Continuous Tick API
```

## Goal

V2.26 exposes Continuous Tick outside the pure core layer.

The goal is to make this possible:

```text
time passes
-> character changes internally
-> no LLM call required
-> no external event required
```

## New Service Capability

`CharacterPhysicsService` now includes:

```text
tickCharacter(characterId, options)
```

It runs:

```text
runContinuousTick()
```

then persists the updated state through the existing repository.

## New API

```text
POST /api/characters/[characterId]/physics/tick
```

Request body:

```json
{
  "daysElapsed": 7,
  "memoryDecayRate": 0.02,
  "deepThinkingThreshold": 0.72
}
```

All fields are optional.

Response:

```text
characterId
trace
state
```

The trace includes:

```text
memory decay
boundary recovery
meta state drift
attention profile before / after
procedural routine strength before / after
deep thinking recommendation
```

## DTO

New contract:

```text
ContinuousTickRequest
ContinuousTickResponse
```

New adapter:

```text
toContinuousTickResponse()
```

## Tests

Updated:

```text
tests/services/characterPhysicsService.test.ts
tests/services/characterPhysicsDto.test.ts
```

Coverage:

```text
service tick mutates and persists state
DTO returns JSON-safe tick trace and latest serialized state
```

## Meaning

Before V2.26, Continuous Tick was a core-only function.

Now it is a service/API capability.

This is the first external entry point for:

```text
continuous living without conversation
```

It prepares the project for future background schedulers, offline aging, habit drift and long-term autonomous character state changes.

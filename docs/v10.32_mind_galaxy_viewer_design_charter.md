## 1. Core Visual Metaphor

### 1.1 The Nebula

CharacterOS 的人格不是一张表格。不是一棵树。不是一个列表。

它是星云。

```
Outer nebula (static gas/dust glow)
  └── Inner clusters (brighter star groups)
       └── Individual stars (memories, beliefs)
            └── Slow orbital drift (personality change over time)
```

**The nebula should feel:**
- Vast but not empty — a diffuse glow fills the background
- Ancient — slow, patient, impersonal. Nothing flickers. Nothing jerks.
- Alive — at maximum zoom, you can see influence factors slowly shifting. Barely. Like watching a continent move.
- Private — only one character's mind at a time. No comparisons. No leaderboards. No social features.

**The nebula should NOT feel:**
- Like a video game. No particles that pop in for attention. No "active effects" animations.
- Like a dashboard. No tables. No bar charts. No progress bars. No percentages.
- Like a chat UI. No message bubbles. No typing indicators.
- Like a tool for editing. No draggable sliders. No color pickers. No save buttons.

### 1.2 Influence Factors (Stars within the Nebula)

The 11 influence factors that structure the personality galaxy:

| Factor | Node Type | Visual Form | Mass Source |
|--------|-----------|-------------|-------------|
| Personality Core | `personality_core` | Central bright point, warm white-gold | Coordinate stability |
| Memory | `memory` | Small diffuse stars, varied brightness | `importance × recency` |
| Impact Particle | `impact_particle` | Transient bright dots, fade over time | `impactScore` |
| Impact Cluster | `impact_cluster` | Dense star groups, colored by category | `mass` |
| Belief | `belief` | Steady medium stars, slow color shift | `strength` |
| Need | `need` | Pulsing faint points, orbit near core | Deficiency intensity |
| Desire | `desire` | Directional streaks, drift toward targets | Desire intensity |
| Behavior Bias | `behavior_bias` | Subtle gravitational lensing arcs | Bias strength |
| Temporal Process | `temporal_process` | Faint orbital rings around clusters | Decay/recovery rate |
| Internal State Variable | `internal_state_variable` | Ambient nebula color temperature | Homeostasis pressure |
| Benchmark Signal | `benchmark_signal` | Rare calibration markers, cold blue | Assertion confidence |

### 1.3 The Slow Variable Principle

Personality is not a fast variable.

- `trust` changes at ~0.001 per event
- `fear` changes at similar rates
- Belief strength drifts over tens of events
- Cluster mass accumulates over hundreds of memories

**Therefore:**
- At normal viewing distance, the nebula appears completely static
- Only at maximum zoom (L4) do influence factors show visible drift
- Drift speed is calibrated to be just barely perceptible — like watching the minute hand of a clock
- No sudden jumps. No "animation". Just slow, continuous drift.

## 2. Zoom Level Architecture (L0 → L4)

### L0 — Outer Nebula (farthest)

**Zoom range:** 1.0x – 3.0x (viewport shows entire mind)

**Visible:**
- Diffuse nebula background glow (color temperature from homeostasis/boredom state)
- Central bright core (personality_core position)
- Faint cluster halos (one per impact_cluster, colored by category)
- Very faint edge wisps between clusters and core (gravity lines, opacity ~0.08)

**Not visible:**
- Individual nodes of any type
- Labels
- Drift vectors
- Edge arrows

**Interaction:** Pan only. No hover. No click targets.

**Frame rate target:** Static render. Redraw only on pan/zoom. No continuous animation loop.

### L1 — Cluster Glow (medium-far)

**Zoom range:** 3.0x – 6.0x

**Visible:**
- All L0 elements
- Cluster boundaries become distinct (density gradient visible)
- Cluster-to-core gravity lines strengthen to opacity ~0.25
- Largest memories appear as faint points within clusters
- Belief stars emerge as steady medium-brightness points near clusters
- Temporal process orbital rings become visible (faint, dashed arcs)

**Not visible:**
- Individual labels
- Small/weak memories
- Needs, desires, behavior biases
- Drift vectors

**Interaction:** Pan + hover over clusters shows cluster name tooltip.

**Transition trigger:** Smooth LOD (level-of-detail). Elements fade in, not pop in.

### L2 — Memory / Belief / Need Hints (medium)

**Zoom range:** 6.0x – 12.0x

**Visible:**
- All L1 elements
- Individual memory nodes (small stars, brightness = importance × recency)
- Belief nodes (medium stars, color temperature = belief valence)
- Need nodes (faint pulsing points near personality core)
- Desire directional streaks (subtle arcs from needs toward targets)
- Impact particles (transient bright dots near recent events)
- Edge types that connect: `belongs_to_cluster`, `activates_belief`, `creates_need`, `drives_desire`

**Not visible:**
- Labels for individual nodes (hover only)
- Behavior biases
- Benchmark signals
- Drift vectors
- Full edge set (only relationship edges, not temporal edges)

**Interaction:** Pan + hover over nodes shows: type label, weight/mass value, stableId.

**Transition trigger:** Nodes and edges fade in at opacity thresholds. Memories below weight 0.15 remain hidden.

### L3 — Influence Factor Labels (medium-close)

**Zoom range:** 12.0x – 24.0x

**Visible:**
- All L2 elements
- Labels on major nodes (personality_core, clusters, beliefs, top needs/desires)
- Edge type labels on strong edges (weight ≥ 0.5)
- Behavior bias arcs (subtle gravitational lensing effect around clusters)
- Benchmark signal markers (cold blue calibration dots)
- Internal state variable ambient color becomes readable as a temperature gradient
- Full edge set visible (including `temporal_transition`, `decays_to`, `derived_from`, `observed_by_benchmark`)

**Not visible:**
- Individual memory labels (too many — hover only)
- Drift vectors (next level)

**Interaction:** Pan + hover over any node or edge shows full metadata panel. Click on a node to pin it (see §6).

**Transition trigger:** Labels appear when node screen-space radius exceeds 8px. Labels below 8px are hidden to avoid text soup. Edge labels appear when edge screen-space length exceeds 60px.

### L4 — Drift Vectors and Slow Motion (maximum zoom)

**Zoom range:** 24.0x – 48.0x

**Visible:**
- All L3 elements
- Drift vectors on all moving nodes (small arrows showing direction and magnitude of recent change)
- Velocity trail on personality_core (faint path showing last N positions)
- Cluster mass change indicators (subtle size pulse: growing = slight expansion, shrinking = slight contraction)
- Belief strength delta markers (small + or − near belief nodes)
- Temporal process phase indicators (orbit color = current phase)

**Drift rendering rules:**
- Vector length = `velocity * 100` logical pixels (so typical velocity of 0.001 → 0.1px, only visible at high zoom)
- Vector color = warm (red-orange) for fear/neuroticism increase, cool (blue) for trust/openness increase
- Velocity trail = last 20 positions from historical snapshots
- Trail opacity decays exponentially (oldest = 0.05, newest = 0.8)

**Not visible:**
- Nothing hidden at this level. Full transparency.

**Interaction:** All L3 interactions + hover over drift vector shows: delta value, time window, source event count.

**Transition trigger:** Drift vectors appear when node screen-space radius exceeds 16px AND zoom ≥ 24.0x. This ensures drift is only visible when it's genuinely observable.

## 3. Data Sources

### 3.1 Directly Available (V7/V8/V10 — No New DTOs Needed)

| Data | Source | Fields | Viewer Use |
|------|--------|--------|------------|
| Node list | `MindGraphSnapshot.nodes` | `id, type, label, stableId, weight, mass, risk, confidence, metadata` | L0-L4 node rendering |
| Edge list | `MindGraphSnapshot.edges` | `id, type, sourceNodeId, targetNodeId, directed, weight, confidence, evidence, metadata` | L1-L4 edge rendering |
| Graph summary | `MindGraphSnapshot.summary` | `nodeCount, nodeCountsByType, edgeCount, edgeCountsByType, averageEdgeWeight` | UI overview stats |
| Layout positions | `GraphLayoutSnapshot.layoutNodes` | `nodeId, x, y, radius, fill, stroke, strokeWidth, opacity` | Node placement |
| Layout edges | `GraphLayoutSnapshot.layoutEdges` | `edgeId, sourceNodeId, targetNodeId, x1, y1, x2, y2, stroke, strokeWidth, opacity` | Edge placement |
| Cluster colors | `ImpactCluster.category` | Category → color mapping | Cluster glow color |
| Personality coordinate | `PersonalityCoordinate.values` | `trust, fear, control, openness, conscientiousness, neuroticism, attachment, agreeableness, extroversion` | Core node position, ambient color temperature |
| Boundary state | `PsychologicalBoundary` | `stressLoad, integrity, cracks, phase` | Nebula boundary glow intensity |

### 3.2 Available but Needs DTO Wrapping (V10.33)

| Data | Source | What's Missing | New DTO |
|------|--------|---------------|---------|
| Recent decisions | `BehaviorDecision`, `DifferentiatedDecision` | No graph node representation | `DecisionNode` in GraphSnapshot |
| Decision explanations | `ExplanationTrace` | No edge from decision → belief/need chain | `explains` edge type |
| Life tick state | `ProjectedLifeState` (energy, fatigue, sleep, boredom) | Not in graph model | `LifeStateNode` in GraphSnapshot |
| Dream fragments | `DreamFragment[]` | No graph representation | `dream` node type or metadata on memory nodes |
| Random thoughts | `RandomThought` | Not in graph model | `thought` node type |
| Self-action tendencies | `GeneratedSelfActionCandidate[]` | Not in graph model | `self_action_tendency` node type |

### 3.3 Needs Historical Snapshot Pipeline (V10.33+)

| Capability | Current State | Needed For |
|------------|--------------|------------|
| Historical snapshots | `MindGraphSnapshot` is point-in-time | L4 velocity trails and drift vectors |
| Snapshot diff | Not implemented | Computing drift deltas |
| Snapshot storage | Not persisted (on-demand generation) | Comparing snapshots across time |
| Animation timeline | Not implemented | Interpolating between snapshots for smooth drift |

**Recommendation:** `MindGraphSnapshot` can be generated on-demand from current state. For L4 drift, we need at minimum: current snapshot + one previous snapshot (stored in memory or temporary file). The diff between these two generates the drift vectors. Full historical timeline is V10.35+.

## 4. Technical Recommendations

### 4.1 Rendering Stack

**Primary: Canvas 2D**

```text
HTMLCanvasElement + 2D context
- performant for thousands of soft circles (stars)
- native globalAlpha for nebula glow layers
- transforms for zoom/pan (ctx.scale + ctx.translate)
- requestAnimationFrame for smooth zoom transitions
- no DOM overhead per node
```

Why not SVG for the main view:
- SVG DOM nodes for 500+ stars = performance degradation on zoom/pan
- SVG filter effects (blur, glow) are slower than Canvas shadowBlur
- SVG `transform` on thousands of elements causes layout recalculations

**SVG only for export:**
- Static snapshot export (PNG/SVG download button)
- Print-quality vector output
- Not the interactive viewer

### 4.2 What NOT to Use

| Technology | Reason |
|-----------|--------|
| WebGL / Three.js | Overkill for 2D nebula. Adds complexity, shader maintenance, GPU compatibility issues. V10.32 scope is Canvas 2D only |
| React component tree per node | Thousands of DOM nodes kill performance. Canvas is flat |
| D3.js | Heavy. Simple 2D transforms + manual draw calls are sufficient |
| CSS animations | No DOM nodes to animate |
| Web Workers | Premature. Single-threaded Canvas is fine for < 2000 nodes |
| WebAssembly | No computational bottleneck justifies it |
| any 3D library | Explicitly 2D. No depth axis. No camera orbits |

### 4.3 Rendering Architecture

```text
Graph Data (MindGraphSnapshot + GraphLayoutSnapshot)
        │
        ▼
Layout Engine (already computed in GraphLayoutSnapshot)
        │
        ▼
LOD Controller (zoom level → visible set selection)
        │
        ▼
Draw Queue (sorted by z-index: nebula → edges → clusters → nodes → labels)
        │
        ▼
Canvas 2D Renderer (single pass, no incremental DOM)
        │
        ▼
Interaction Layer (event delegation on canvas, hit-testing against node positions)
```

### 4.4 Performance Targets

| Metric | Target |
|--------|--------|
| Nodes rendered at L0 | ~30 (clusters + core + background) |
| Nodes rendered at L2 | ~200 (adds memories, beliefs, needs) |
| Nodes rendered at L4 | ~500 (all nodes + labels + vectors) |
| Frame time (static) | < 16ms (60fps achievable) |
| Frame time (zoom transition) | < 32ms (30fps acceptable during zoom) |
| Memory | < 50MB for graph data + layout + canvas buffers |
| Initial load time | < 500ms from API response to first render |

## 5. Animation Principles

### 5.1 Deterministic Rendering

- Every frame is computed from the same graph data → same input always produces identical output
- No `Math.random()` in rendering. If randomness is needed for star field distribution, use a deterministic seed from `characterId + snapshotId`
- Animation timeline driven by `elapsedRealTime` (for zoom transitions) and `simulatedTime` (for drift)
- No physics simulation in the viewer. Drift is a linear interpolation between two snapshot positions

### 5.2 Speed Calibration

| Element | Speed | Rationale |
|---------|-------|-----------|
| Nebula background glow | 0 (static) | Ambient, not animated |
| Zoom transitions | 300ms ease-out | Smooth but not sluggish |
| Pan inertia | 200ms decay | Light momentum feel |
| Drift vectors (L4) | 0.5–2.0 px/second | Barely perceptible. Like watching moss grow |
| Cluster mass pulse | 0.2 px/second amplitude | Almost invisible. Only at L4 |
| Belief strength delta markers | Fade in over 500ms | Appear, don't animate |
| Hover glow | 150ms ease-in | Immediate feedback, no delay |

### 5.3 Reduced Motion

```text
@media (prefers-reduced-motion: reduce) {
  - All drift vectors: hidden (static snapshot only)
  - Zoom transitions: instant (0ms)
  - Hover glow: instant
  - Cluster mass pulse: hidden
  - Velocity trails: hidden
}
```

A setting in the viewer UI: "Show drift (slow computer / accessibility)" toggle. Default respects OS preference.

### 5.4 Pause Control

- Drift animation can be paused (freezes all motion, shows static snapshot)
- Pause does not hide drift vectors — it freezes them at current position
- Useful for inspection, screenshots, teaching

## 6. Interaction Principles

### 6.1 Navigation

| Action | Input | Behavior |
|--------|-------|----------|
| Zoom in/out | Mouse wheel | Scale by 1.15× per tick. Clamped to [1.0, 48.0]. Zoom toward cursor position |
| Pan | Mouse drag (left button) | Translate viewport. Inertia decay on release |
| Reset view | Double-click empty space | Animate back to L0 (zoom 1.0, centered on personality_core) |
| Zoom to cluster | Double-click cluster | Animate to L2 centered on that cluster |
| Zoom to node | Double-click node | Animate to L4 centered on that node |

### 6.2 Inspection

| Action | Input | Behavior |
|--------|-------|----------|
| Hover node | Mouse over | 150ms fade-in tooltip: type, label, weight/mass, stableId |
| Hover edge | Mouse over | Show edge type label + weight |
| Hover drift vector | Mouse over (L4 only) | Show: delta value, time window, source events count |
| Click node | Left click | Pin node. Hover panel becomes sticky. Shows full metadata |
| Unpin | Click empty space or Esc | Dismiss pinned panel |

### 6.3 What You CANNOT Do

- No dragging nodes to new positions (layout is computed, not user-editable)
- No deleting nodes or edges
- No creating nodes or edges
- No editing labels, weights, or metadata
- No saving state changes
- No commit/apply/rollback from the viewer
- No exporting modified graph
- No multi-select, no group operations
- No undo/redo (nothing to undo)
- No keyboard shortcuts for mutation

### 6.4 Pinned Factor Panel

When a node is clicked (pinned):

Content varies by node type. Memory shows cluster + belief links. Belief shows need → desire chain. Cluster shows mass trend. Core shows full personality coordinate.

## 8. Phased Roadmap

### V10.32 ✅ Design Charter (this document)

### V10.33 Galaxy View DTO
- Define `GalaxyViewData` in `src/appContracts/galaxyView.ts`
- Aggregate: `MindGraphSnapshot` + `GraphLayoutSnapshot` + current `CharacterPhysicsState` summary + recent life tick projection
- Type-only. No route implementation.
- Include deterministic star field background seed

### V10.34 Static Canvas Renderer
- Implement `GalaxyRenderer` class in `src/core/galaxy/galaxyRenderer.ts`
- Pure function: `renderSnapshot(ctx, snapshot, viewport) → void`
- L0-L2 static rendering (no animation, no drift)
- Node and edge drawing with LOD selection
- Hit-testing for hover
- Tests: deterministic output, LOD thresholds correct, no DOM dependency

### V10.35 Zoom / LOD System
- Zoom controller: wheel → scale transform + LOD recalculation
- Pan controller: drag → translate transform
- Smooth zoom transitions (ease-out 300ms)
- LOD transition (fade-in for new nodes at threshold crossings)
- Double-click to zoom to node/cluster
- Tests: zoom clamped to [1.0, 48.0], LOD thresholds correct at each level

### V10.36 Influence Drift Animation
- Historical snapshot storage (current + previous)
- Drift vector computation (linear diff between snapshots)
- Animation loop (`requestAnimationFrame`) for L4 drift rendering
- Reduced motion support
- Pause/resume control
- Tests: drift vectors deterministic, velocity trails correct length, pause stops all motion

### V10.37 Viewer API Integration
- Implement `GET /product/[characterId]/galaxy-view`
- Returns `GalaxyViewData` (snapshot + layout + state summary)
- No state mutation
- Tests: response shape valid, no sensitive state leak, READ-ONLY

### V10.38 Galaxy Viewer QA
- End-to-end: generate galaxy view data → render static → zoom through all levels → verify LOD → verify drift → verify hover panels
- Accessibility: reduced-motion verified
- Performance: < 16ms frame time at L4 with 500 nodes

## 9. Explicitly Forbidden

| Forbidden | Reason |

| Complex React SPA | Minimal HTML + Canvas + vanilla JS controls |
| Real-time data streaming | Snapshot-based, request/response only |
| User accounts / auth beyond API key | Existing CHARACTEROS_API_KEY sufficient |
| Gamification (points, badges, progress bars) | Observation tool, not a game |

## 10. Final Judgment

**Mind Galaxy Viewer is an observation instrument, not a control panel.**

It should let the user see how personality is shaped by experience — slowly, incrementally, at the scale of hundreds of memories and thousands of hours.

The outer nebula is the promise: "there is structure here." The zoom journey is the reveal: "here is what that structure is made of." The drift vectors at maximum zoom are the answer: "人格不是固定的。它在动。很慢。但它在动。"

This is not a tool for editing characters. It is a way of seeing them.

The viewer does not talk. It does not suggest. It does not rate, rank, optimize, or gamify. It simply shows you the galaxy inside one person's mind, and lets you look at it for as long as you want.

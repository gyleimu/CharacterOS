# CharacterOS V3.7.41 Manual Adjustment Override Dashboard Report

## Summary

This update exposes governance override as a deliberate Dashboard workflow.

Before this update:

```text
API could accept governanceOverride
history could record governanceOverride
Dashboard could display override history
```

But the Dashboard could not initiate an override.

Now, when governance cooldown is active, the Dashboard shows an explicit override reason field and a separate override apply button.

## Architecture Meaning

Manual parameter adjustment now has a complete first operator loop:

```text
proposal
↓
preview
↓
audit
↓
patch
↓
snapshot
↓
governance
↓
manual apply or explicit override
↓
history
```

This keeps CharacterOS editable without making it casually mutable.

The system can now say:

```text
You may adjust.
You should wait.
You may override, but only with an explicit reason.
```

That is closer to the project principle:

Human participation is allowed, but character continuity must be protected.

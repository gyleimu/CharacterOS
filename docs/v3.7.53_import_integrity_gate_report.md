# CharacterOS V3.7.53 Import Integrity Gate Report

## Goal

V3.7.52 added state integrity inspection.

This update connects that inspection to the import pipeline.

An import package should not be trusted just because its JSON shape is valid.

It must also be internally coherent.

## Core Change

Updated:

```text
src/core/export/characterImportPlan.ts
```

`CharacterImportPlan` now includes:

```text
stateIntegrity
```

During plan construction, CharacterOS now:

```text
validates export package shape
summarizes package
deserializes candidate state
inspects state integrity
classifies import risk
```

If state integrity fails, the plan becomes:

```text
status: blocked
risk: high
requiresExplicitConfirmation: false
willReplaceState: false
willReplaceAdjustmentHistory: false
```

This blocks packages with broken references such as:

```text
MemoryNode -> missing ImpactCluster
ImpactCluster -> missing ImpactParticle
BeliefState -> missing MemoryNode
```

## API Change

Updated:

```text
src/app/api/characters/[characterId]/import/validate/route.ts
```

`/import/validate` now treats integrity failure as non-importable.

Response behavior:

```text
valid JSON shape + valid state integrity -> 200
invalid JSON shape -> 422
valid JSON shape + broken state integrity -> 422
```

The response still remains read-only:

```text
mutatesState: false
```

## Apply Gate

No extra bypass was added.

`authorizeCharacterImportApplication` already depends on `buildCharacterImportPlan`.

Therefore:

```text
broken package
-> blocked import plan
-> blocked apply authorization
```

Even the correct confirmation phrase cannot apply a package whose state integrity failed.

## Architecture Meaning

CharacterOS now has an import boundary that checks both:

```text
shape validity
state coherence
```

This is important before adding richer character editing, migration, benchmark packages, or calibration state transfer.

The import pipeline now refuses broken psychological states instead of merely accepting well-formed JSON.

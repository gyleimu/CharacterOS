# CharacterOS V3.7.17 Tick Manual Controls Report

## Summary

This update expands Continuous Tick manual controls in the Dashboard.

The goal is to make artificial inspection and tuning easier without persisting temporary values into character state.

## Added Dashboard Controls

```text
fatigue
sleepDebt
memoryDecayRate
deepThinkingThreshold
```

## Updated Files

```text
src/components/character/PhysicsDashboard.tsx
tests/app/continuousTickRoute.test.ts
```

## Behavior

The controls are sent to:

```text
POST /api/characters/[characterId]/physics/tick
```

They affect:

```text
parameterNetwork trace
effective memory decay rate
effective deep thinking threshold
```

They do not persist to:

```text
CharacterPhysicsState
```

## Dashboard Additions

The Tick summary now displays:

```text
effectiveMemoryDecayRate
effectiveDeepThinkingThreshold
fatigue / sleepDebt
```

## Development Principle

This keeps manual tuning in the observation/calibration layer first.

Temporary controls should become persistent character parameters only after repeated behavior proves they are meaningful.

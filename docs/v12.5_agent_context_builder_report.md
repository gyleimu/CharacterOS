## 1. Module

`src/core/agent/agentContextBuilder.ts` — assembles safe AgentGroundingBundle from V11 surfaces

## 2. What Context Is Included

| Source | Included When |
|--------|--------------|
| CharacterStateSurface | Always (headline, emotion, stress, beliefs) |
| ExplainabilityTimeline | Policy ≠ block |
| RealityAuditPanel | Policy = apply_allowed |
| TimeMachineRefs | If provided (IDs only) |
| EvidenceRefs | Derived from surface + explainability |
| Safety notices | Always (simulation-not-diagnosis) |

## 3. What Is Intentionally Omitted

| Omitted | Reason |
|---------|--------|
| Raw CharacterPhysicsState | Never exposed to external consumers |
| Coordinate values | Qualitative bands only |
| Full memory payloads | Summary counts only |
| Repository data | Internal storage |
| API keys/secrets | Security boundary |
| Raw plugin payload | Summarized upstream |
| Engine internals | velocity, driftMultiplier, etc. |

## 4. Policy-Aware Context

| Decision | Context Level |
|----------|--------------|
| `block` | Minimal (state surface only) |
| `preview_only` | State + explainability |
| `confirmation_required` | State + explainability + audit notice |
| `apply_allowed` | Full (state + explainability + reality audit) |

## 5. Evidence Selection

EvidenceRefs are derived from:
1. State surface headline + emotional state
2. Top 2 beliefs
3. Explainability causal steps (up to 3)
4. Fallback: "历史证据不足" if no history

## 6. Compact Summary for ReplyPlanner

```
[State] headline | [Emotion] label | [Stress] label | [Beliefs] ... | [Policy] decision
```

Structured, not prose. Designed for ReplyPlanner consumption.

# CharacterOS V3.7.6 Replay Calibration Dashboard Report

## Purpose

Expose replay calibration through API and Dashboard.

This pass did not change character physics behavior.

It made the existing calibration harness visible and easier to run.

## New API

```text
GET /api/trace/replay/calibration
```

Response:

```text
ReplayCalibrationSuite
```

If a calibration check fails, the route returns:

```text
500
```

with the failed checks and calibration payload.

## Dashboard

The Dashboard now includes:

```text
Replay 校准
```

It shows:

```text
overall pass / fail
pass count
warn count
fail count
scenario direction
trust drift
fear drift
check severities
```

It also provides:

```text
重新校准
```

which refreshes data from the API.

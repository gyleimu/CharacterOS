## Objective

Add deterministic, template-based, no-LLM explanations to the DifferentiatedDecision chain (V10.12). Every stage of the differentiation pipeline — schema activation, need formation, desire formation, strategy selection, action surface, and legacy comparison — now produces structured `ExplanationTrace` output.

---

## Why Explainability Before More Benchmark

V10.11 proved the engine scores well (Matrix 0.858, Focus 0.676, etc.). V10.12 made it a mainline feature. But scores alone don't tell you **why** the system chose strategy A over strategy B, or why need X was stronger than need Y.

Explainability closes that gap:

- **Debugging**: developers can trace a decision back to its root signals
- **Audit**: every fact cites the source data it's derived from
- **Trust**: "the system chose `verify_before_commitment` because betrayal_schema was activated by 低信任 + 证据感 need" is verifiable

---

## Explanation Chain (6 stages)

### 1. Schema Activation Explanation
- Lists all activated schemas with intensity, source, matched keys, interpretation
- Top schema highlighted as dominant driver
- Secondary schemas noted if multiple

### 2. Need Formation Explanation
- Each need traced to its source schemas
- Intensity rationale from schema activation strength
- Reason from the differentiation engine (e.g., "背叛图式把环境压力转化为安全感需求")

### 3. Desire Formation Explanation
- Each desire traced to source schemas and needs
- Intent captured (e.g., "推动角色核验对方是否可信")
- Intensity from schema strength + need intensity

### 4. Strategy Selection Explanation
- All candidate strategies listed with intensity, direction, coreReason, actionIntent
- Selected strategy explained: why it ranked first
- Runner-up noted with intensity gap

### 5. Action Surface Explanation
- Final action text, direction, reason
- Template penalty explanation (warns if >0.3)
- References check: environment/persona/need/desire referenced?

### 6. Legacy Comparison Explanation
- Legacy `BehaviorDecision.mostLikelyAction` versus `DifferentiatedDecision.actionSurface.action`
- Match/diff detection
- Explanation of why differentiated is more specific (schema/need/desire chain vs. behavior bias)

### ExplanationTrace Output

```
{
  id: deterministic hash,
  scope: "differentiated_decision",
  title: "{persona}在{env}中的分化决策解释",
  summary: one-sentence chain summary,
  reasons: [schema, need, desire, strategy, action, legacy] (7-10 reasons),
  facts: [schemas[], needs[], desires[], strategies[], action, comparison] (15-30 facts),
  warnings: [template penalty, missing references],
  createdAt: deterministic anchor (2026-06-24T12:00:00.000Z)
}
```

---

## Determinism & No-LLM Guarantee

- All fact/ID generation uses `stableHash(seed + suffix)` — no `Date.now()`, no `Math.random()`
- `createdAt` uses a fixed deterministic timestamp
- All reason text is template-generated from structured data
- Zero LLM calls
- Same input → identical `ExplanationTrace` every time

## API Response Changes

`GET /api/characters/[characterId]/decision` now includes:

```json
{
  "...legacy fields unchanged...": "...",
  "differentiatedExplanation": {
    "id": "dd-expl_...",
    "scope": "differentiated_decision",
    "title": "...",
    "summary": "...",
    "reasons": [...],
    "facts": [...],
    "warnings": [...]
  }
}
```

Explanation failure is gracefully handled — the API returns `differentiatedExplanation: null` with a warning, never breaks the route.

## Known Limitations

- Explanation text is template-based — doesn't capture nuanced interactions between schemas
- Top schema/need/desire are highlighted, but secondary interactions (e.g., abandonment + scarcity co-activation) aren't deeply explained
- Legacy comparison only compares action text, not the full decision structure
- `createdAt` uses deterministic anchor, not real timestamp (intentional for test stability)
- Persona/environment names come from runtime state, not benchmark fixtures — may be generic for default characters

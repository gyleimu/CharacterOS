# CharacterOS V3.7.61 Import History Trace Link Report

## 目标

V3.7.60 已经建立了 Import Transition History，但 `/import/apply` 的即时响应和历史列表之间还缺少一个直接链接。

V3.7.61 的目标很小：让每次 import apply 返回的 trace 都能直接指向它写入的历史记录。

## 新增字段

`CharacterImportApplyTrace` 新增：

```text
historyEntryId
historyRecordedAt
```

适用范围：

```text
applied import trace
blocked import trace
deserialization failure trace
```

也就是说，只要请求进入 `/import/apply` 并被记录到 Import Transition History，返回 trace 就会携带对应的历史记录 id 和记录时间。

## 行为

现在一次导入应用请求会形成以下关系：

```text
POST /import/apply
-> CharacterImportApplyTrace.historyEntryId
-> CharacterImportTransitionHistoryEntry.id
-> CharacterImportTransitionHistoryEntry.trace.historyEntryId
```

三者保持一致。

## 为什么重要

在没有前端页面的阶段，CharacterOS 主要依赖 API 和 trace 进行人工调试。

如果调用方执行了一次高风险导入操作，它应该能立即知道：

- 这次操作是否被记录。
- 记录 id 是什么。
- 之后查询历史列表时，哪一条就是刚才的操作。

V3.7.61 让 apply 响应和历史审计之间不再需要靠时间或列表顺序猜测。

# CharacterOS V3.7.5 Replay Calibration Report

## Purpose

Start calibrating the existing core with fixed Lin Fan replay scenarios.

This pass did not add a new psychological subsystem.

The goal was to answer:

```text
Do fixed scenarios drift in the expected psychological direction?
Is the drift visible but still slow?
Do velocity caps and force caps hold during repeated events?
Do cluster masses grow under repeated similar experiences?
```

## New Core Module

```text
src/core/trace/replayCalibration.ts
```

It builds a deterministic calibration suite from existing trace replay scenarios.

Each scenario is checked for:

```text
dominant direction
drift magnitude
velocity cap
dominant cluster mass growth
```

## Observed Behavior

The latest run passed all calibration checks.

Direction distribution:

```text
defensive_drift: 3
recovery_drift: 2
```

Important observation:

```text
short scenarios produce very small drift
long repeated scenarios produce visible but bounded drift
```

This is desirable for the current model because:

```text
personality is a slow variable
single events should not rewrite the character
repeated experiences should accumulate force
```

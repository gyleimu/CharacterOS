### 参数注册表

`src/core/parameters/modelParameterRegistry.ts` 管理四组校准关键参数：

- Temporal：事件密度窗口、重复压力、最低 impact 比例、恢复上限、velocity 半衰期。
- Memory：默认衰减率、长期 recency floor、动态 recency 权重。
- Boundary：正向安全信号、负向压力、overflow 损伤、恢复、repair nudge 和类别压力权重。
- Personality：默认学习率、不同事件类别的学习激活和 momentum。

注册表当前包含 54 个数值参数和 54 条一一对应的治理描述。每个描述记录路径、单位、合法范围、依赖通道、来源和用途。注册参数集全部深度冻结，并使用稳定内容指纹防止静默漂移。

当前注册版本：

| 参数集 | 用途 |
|--------|------|
| `legacy-v14.0.0` | 旧状态兼容和历史回放 |
| `model-calibration-v1.0.0` | 当前默认参数集 |

旧序列化状态没有 `parameterSetVersion` 时会迁移到冻结的 legacy set；新状态默认写入 current set。Engine 与 State 参数版本不一致时 fail closed，不允许混用规则继续计算。

## Golden Trajectory

轨迹矩阵覆盖：

```text
10 event categories
x 4 baseline personalities
x 1 / 5 / 20 / 100 event horizons
= 160 state trajectories

160 trajectories
x relationship / study / social / action
= 640 decision scenario projections
```

四类 baseline：secure/resilient、attachment-sensitive、achievement-oriented、adaptive/exploratory。验收采用方向和合理区间，不拟合唯一答案：

- 单次重大事件必须可见但不能瞬间改写人格。
- 重复事件可以累积，但 coordinate 始终有限且保持 `[0,1]`。
- support/success 与 abandonment/betrayal 在目标通道上保持相反或可区分方向。
- fatigue/general 不得被错误解释为 trust 人格变化。
- 相关场景的策略分布应响应，top strategy 可以保持稳定。
- 每个事件类别分别达到最低响应率，避免总体平均值掩盖局部失效。
- 五次事件后的策略分布距离受上限保护，避免短期过度响应。
- 每个输入仍保留独立 MemoryNode 和审计证据。

## 本轮发现并修复的核心问题

### 1. 弱通道量化死区

原实现每次都把 personality velocity 和 coordinate 截断到四位小数。单步小于 `0.00005` 的变化会永久变成零，即使重复 100 次也无法累积。这会让 uncertainty 等弱而持续的通道失真。

修复后内部积分使用八位精度，输出报告仍可以四位小数展示。这样保留了 slow channel 的累积语义，没有通过放大学习率制造表面变化。

### 2. 长期 repair 在低压力后停止

原 repair nudge 完全依赖当前 boundary stress。support 将 stress 降到接近零后，安全信号随之消失，后续支持事件无法继续修复 trust/fear。结果是系统能减压，却不能形成稳定的正向再学习。

修复后分离两个含义：

- stress relief 仍随当前压力趋近于零；
- repair evidence 由有界的正向 event impact 保留，因此低压力下仍能缓慢更新 trust/fear/openness。

20 次 abandonment 后再执行 20 次 support 的结果：

| 指标 | 数值 |
|------|------|
| Trust damage | 0.0399 |
| Trust repair | 0.0191 |
| Scar retention | 0.5209 |

这形成了“损伤可修复、修复不瞬间抹除历史”的显式不对称。

### 3. Galaxy 重算绕过参数集

`simulatePersonalityGalaxyStep()` 曾在内部用默认 memory 参数重新计算 cluster mass，导致 sensitivity variant 虽然传入了新的 recency floor，真正的人格力场仍使用默认值。

修复后 memory parameters 沿 Engine -> Galaxy -> Cluster Metrics 显式传递。参数变体现在确实改变被审计的核心计算，不再只是报告层变量。

### 4. 疲劳期望的概念边界

疲劳会影响短期能量、耐心和执行，不应被强行验收为长期 conscientiousness 人格下降。因此 Golden Trajectory 对 fatigue 的人格要求是 trust/conscientiousness 基本稳定，行为层影响仍应由 life/energy context 负责。该调整避免把 transient state 和 personality trait 混为一谈。

逐类别决策覆盖审查显示 fatigue 是当前最弱通道：相关 action 场景响应率为 50%，刚好达到类别下限。原因不是人格积分不足，而是 `CharacterPhysicsState` 尚未持久化 Life Simulation 的 energy/fatigue transient state。当前只能通过 boundary 间接影响决策。该缺口不通过放大人格漂移掩盖，交给 Durable State schema 正式解决。

## 属性与 Metamorphic 验收

16 个固定 seed 各生成 30 个合法事件，逐一验证：

- 完整序列 byte-identical deterministic replay。
- State Integrity 无错误。
- Memory/Particle ID 唯一。
- 全部 coordinate 有限且位于 `[0,1]`。

Metamorphic Test 验证：

- 无关地点措辞不改变结构化物理结果。
- tag 顺序不改变结果。
- abandonment 与 support 产生可区分的 trust 方向。
- 集中与分散时间只改变剂量，不反转类别方向。
- 同一事件序列在 secure 与 sensitive baseline 上保留人格分化。

## 敏感性分析

以下关键参数分别执行 `-10% / baseline / +10%` 单变量扰动：

- repeat density pressure
- personality velocity half-life
- memory recency floor
- memory decay rate
- positive safety impact
- negative boundary stress base
- standard personality learning base

7/7 探针保持预期单调性，0 个方向反转，归一化输出变化均低于过敏阈值。敏感性变体使用独立参数版本和指纹，不修改当前默认参数集。

## 仍然成立的限制

1. 轨迹区间是工程合理性约束，不是现实人群标定结果。
2. Event category coordinate templates 已受轨迹门保护，但尚未通过真实长期观察数据拟合。
3. 生成属性测试是确定性抽样，不是所有合法序列的穷举证明。
4. 乱序历史事件目前只阻止时钟倒退，尚不能自动在正确位置重放后续事件。
5. fatigue 的持久 transient energy state 仍主要位于 Life Simulation，不应借人格坐标替代。
6. `parameterSetVersion` 固定已注册数值，但不能独自冻结算法代码；跨版本重放还必须保存 engine semantics version。

## 2. D10 关键设计

```text
Phase 3 boundary recovery 是一个 intermediate step。
Phase 4 homeostasis 通过 regulatedBoundary 覆盖 Phase 3 的结果。

V5.4 的 boundary_recovery subProcess trace 记录了：
  - recoverBoundary() 的输入/输出
  - NOT 最终的 tick boundary

最终 state.boundary = homeostasisResult.trace.regulatedBoundary
（D10-1 关系在 V4.12 已验证，V5.4 保持不变）

测试验证：
  ✓ Phase 3 subProcess metrics 反映 recoverBoundary intermediate
  ✓ D10: final boundary === regulatedBoundary（覆盖发生）
  ✓ D10: subProcess 不干扰 overwrite semantics
```

## 3. 这是 Instrumentation，不是 Delegation

```text
V5.4 与前两个 subprocess 遵循相同原则：

  - recoverBoundary() 调用仍在 runContinuousTick Phase 3 中 inline 执行
  - buildBoundaryRecoverySubProcessTrace() 是纯 trace builder，只读不写
  - 不改变任何 mutation 的位置、顺序或输入/输出
  - homeostasis overwrite semantics (D10) 完全不变
```

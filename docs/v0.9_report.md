# CharacterOS V0.9 阶段报告

## 当前版本判断

当前项目可以标记为：

```text
CharacterOS V0.9
```

V0.9 的核心意义是：CharacterOS 已经从「事件影响人格」推进到「事件影响人格，并进一步推导人物当前心理动力和行为倾向」。

当前主链路：

```text
Natural Language Event
-> LLM / Rule Parser
-> ParsedExperienceEvent
-> ImpactParticle
-> MemoryNode
-> ImpactCluster
-> PersonalityDrift
-> BeliefState
-> NeedDeficiency
-> DesireState
-> BehaviorBias
```

这已经接近 Character Physics Engine V1 的最小闭环。

## 本阶段完成内容

### 1. Belief System V1

新增：

```text
src/core/belief/beliefState.ts
```

能力：

- 从 MemoryNode 的 `beliefEffect` 派生当前信念。
- 合并相同信念。
- 计算 `strength`。
- 记录 `evidenceCount` 和来源记忆。

示例：

```text
重要的人可能会突然离开
strength: 0.606
evidenceCount: 2
```

### 2. Need Deficiency V1

新增：

```text
src/core/need/needDeficiency.ts
```

当前支持：

- 安全感缺失
- 信任缺失
- 依恋确认缺失
- 控制感缺失

输入来源：

```text
PersonalityCoordinate
BeliefState
ImpactCluster
```

### 3. Desire System V1

新增：

```text
src/core/desire/desireState.ts
```

当前从缺失派生欲望，例如：

```text
想确认对方是否仍然安全、仍然在关系中。
想获得解释和一致性证据，而不是只听安慰。
想确认自己没有再次被抛下。
想通过克制、询问或拉开距离重新获得控制感。
```

### 4. Behavior Bias V1

新增：

```text
src/core/behavior/behaviorBias.ts
```

当前输出行为倾向：

- 压住情绪，先追问原因。
- 表现得克制、冷淡，避免暴露依赖。
- 立刻爆发质问。
- 完全无所谓地忽略这件事。

行为倾向不是最终剧情，而是人物在当前状态下的行为概率偏置。

### 5. Derived Character State

新增：

```text
src/core/state/derivedCharacterState.ts
```

统一派生：

```text
beliefs
needs
desires
behaviorBiases
```

并已接入：

```text
SerializedCharacterPhysicsState.derived
```

因此 API 返回角色物理状态时，会同时返回当前心理动力状态。

## 当前能力总结

现在 CharacterOS 已经可以完成：

```text
输入事件
-> 自动解析事件
-> 生成影响粒子
-> 写入记忆节点
-> 聚合影响星团
-> 人格核心慢速漂移
-> 派生当前信念
-> 派生当前缺失
-> 派生当前欲望
-> 派生行为倾向
```

这已经回到了项目最初目标：

```text
不要写故事。
模拟人物。
```

## 为什么仍不是 V1.0

V0.9 仍然是最小闭环，不是最终 V1。

主要缺口：

1. Belief Update 仍然是统计派生，不是真正的长期贝叶斯更新。
2. Need / Desire 目前是规则模型，还没有独立状态、衰减、强化。
3. BehaviorBias 只是倾向，不是完整 Decision Engine。
4. 记忆、信念、欲望还没有独立持久化表结构。
5. 还没有角色编辑器。
6. 还没有正式数据库。
7. 还没有 LLM 行为解释层。

## 下一步建议：V1.0 最小正式版

V1.0 应该聚焦一件事：

```text
Behavior Decision Engine
```

把当前 `BehaviorBias` 升级为正式决策输出：

```text
他会怎么想
他会产生什么情绪
他内心冲突是什么
他不会做什么
他最可能做什么
为什么这符合他的人格
```

建议 V1.0 链路：

```text
DerivedCharacterState
-> DecisionContext
-> BehaviorDecision
-> LLM Explanation / Rule Explanation
```

V1.0 的完成标准：

输入一个事件后，系统能输出：

```text
1. 激活记忆
2. 影响星团
3. 当前信念
4. 当前缺失
5. 当前欲望
6. 行为倾向
7. 最可能抉择
8. 心理一致性解释
```

这会让 CharacterOS 从：

```text
Character Physics Dashboard
```

进入：

```text
Character Decision Engine
```

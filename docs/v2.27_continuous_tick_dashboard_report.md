# CharacterOS V2.27 Continuous Tick Dashboard Report

## Version

```text
CharacterOS V2.27 Continuous Tick Dashboard
```

## Goal

V2.27 makes Continuous Tick visible and operable from the Dashboard.

Before this version, tick could be called through core/service/API.

Now it can be triggered directly in the UI.

## Dashboard Additions

`PhysicsDashboard` now contains:

```text
连续生活 Tick
```

Controls:

```text
推进 1 天
推进 7 天
推进 30 天
```

The panel calls:

```text
POST /api/characters/[characterId]/physics/tick
```

and updates the current serialized physics state.

## Design Rule

Tick is displayed separately from event processing.

This keeps the distinction clear:

```text
Event processing = something happened to the character
Continuous Tick = time passed inside the character
```

## Verification

```text
npm run build

```

Both pass.

## Meaning

CharacterOS now has a visible loop for:

```text
no new event
no dialogue
no LLM
time still changes the character
```

This is a necessary step toward continuous living characters.

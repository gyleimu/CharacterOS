# CharacterOS V3.7.57 Import Audit Summary Report

## Goal

The import plan now contains detailed validation, integrity, digest and governance information.

That is useful for machines, but a human API workflow also needs a compact decision layer.

This update adds an import audit summary.

## Core Change

Updated:

```text
src/core/export/characterImportPlan.ts
```

`CharacterImportPlan` now includes:

```text
auditSummary
```

The summary contains:

```text
decision
canApply
requiresReview
nextAction
blockers
warnings
checks
```

## Decisions

Possible decisions:

```text
can_apply
review_required
rejected
```

Mapping:

```text
ready        -> can_apply
needs_review -> review_required
blocked      -> rejected
```

## Checks

Audit checks include:

```text
package_shape
state_integrity
integrity_snapshot
package_digest
character_match
governance
```

Each check returns:

```text
pass
warn
fail
```

## Why This Matters

Without a frontend, the API response itself must be readable enough for manual operation.

The full plan remains available for detailed inspection.

The summary answers:

```text
Can this package be applied?
Does it need review?
Why?
What should the operator do next?
```

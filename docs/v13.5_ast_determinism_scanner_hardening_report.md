## Why Upgrade from Regex to AST

V13.3's static scanner used regex/path-based matching with these limitations:

1. **Regex is fragile** ‚Äî `Date.now` in a comment or string literal looks identical to code
2. **Path-based classification** ‚Äî operational module builder functions could be missed
3. **No context awareness** ‚Äî couldn't distinguish `createMemoryNode` (operational) from `buildEventStudioDraft` (sensitive)
4. **Template expressions invisible** ‚Äî `` `draft_${Date.now()}` `` was undetectable without AST

V13.5 replaces regex scanning with TypeScript compiler API AST traversal, providing:
- Correct handling of comments and string literals (AST naturally excludes them)
- Function-level context awareness
- Template expression detection
- Export-aware sensitivity analysis
- Structured, typed findings with line/column precision

- Functions starting with `build*`, `create*`, `plan*`, `preview*`, `apply*`, `generate*`, `make*`

### What V13.5 Caught That V13.3 Could Miss

The AST scanner's sensitivity improvements:

1. **Template expressions** ‚Äî `` `prefix_${Date.now()}` `` and `` `audit_${Math.random()...}` `` are now visible to the scanner

## Known Limitations (Post-V13.5)

1. **AST scanner only covers `.ts`/`.tsx` files** ‚Äî `.js` files not in scope (project is TypeScript-only)
2. **`setTimeout`/`setInterval` for ID generation** ‚Äî not detected (AST doesn't reason about usage intent)
3. **Cross-module ID propagation** ‚Äî if a non-deterministic ID is generated in an allowed module and passed to a sensitive builder, the scanner won't detect the propagation
4. **Scanner self-exclusion** ‚Äî the scanner excludes itself, so non-deterministic patterns in scanner code go unchecked (acceptable ‚Äî the scanner is audit tooling)

## Conclusion

**V13.5 AST-based Determinism Scanner** upgrades the static analysis from fragile regex matching to context-aware AST traversal. The scanner correctly distinguishes operational timestamps from default-path violations, uses 21 explicit allowlist rules with documented justifications, and produces structured findings with line/column precision.

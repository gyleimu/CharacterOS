# CharacterOS V3.7.38 Parameter Adjustment Cooldown Report

## Summary

This update adds active cooldown tracking for manual parameter adjustment governance.

Governance no longer only says:

```text
allow
cooldown
pause
```

It now also reports:

```text
cooldownUntil
cooldownActive
```

## Updated Core

```text
src/core/parameters/parameterAdjustmentGovernance.ts
```

## Cooldown Rules

Cooldown is calculated from:

```text
summary.latestAt + cooldownDays
```

Current durations:

```text
medium risk without frequent targets -> 3 days
medium risk with frequent targets    -> 7 days
high risk                            -> 14 days
```

## Updated Dashboard

```text
src/components/character/ParameterAdjustmentHistoryPanel.tsx
src/components/character/PhysicsDashboard.tsx
```

Dashboard now shows:

```text
cooldown active / ended
cooldownUntil
```

The manual apply button is disabled when:

```text
governance.cooldownActive === true
```

Rollback remains available when a snapshot exists.

## Architecture Meaning

Manual calibration now has time awareness.

CharacterOS can distinguish:

```text
this character should not be edited right now
```

from:

```text
this character had risk before, but the cooling window has passed
```

That is closer to the project's deeper principle:

Change needs time, and governance should respect time.

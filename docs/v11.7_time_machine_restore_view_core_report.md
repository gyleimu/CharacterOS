## 2. Restore View vs Rollback

| Restore View | Rollback |
|-------------|----------|
| Read-only historical display | Write mutation to live state |
| `noMutation: true` | Requires confirmation |
| `isHistoricalView: true` | Changes current state |
| Safety banner warns it's not current | Audit trail records rollback |
| Many views can be open | Only one active state |

V11.7: restore views only. Rollback still deferred.

## 3. How Current vs Historical Is Detected

```typescript
isCurrentSnapshot =
  snapshot.snapshotId === timeline.currentSnapshotId ||
  snapshot.snapshotId === currentSnapshot.snapshotId
```

The timeline's latest snapshot is "current". Any other snapshot is "historical".

## 4. Qualitative Diff

When a current snapshot is provided for comparison:
- Compares personality bands (low/moderate/high) across 5 dimensions
- Reports direction: increased/decreased/changed/unchanged
- Human-readable summary per dimension
- No raw numbers

```
trust: 信任偏低 → 信任中等  [increased]
fear: 高度恐惧 → 恐惧偏高  [decreased]
```

## 5. Safety Banner

Every restore view includes:

```
📜 历史视图 — 这是过去某一时刻的状态快照
这不是当前角色状态
这是模拟系统的历史记录，不是医学或心理诊断
查看历史视图不会写回或修改角色当前状态
```

## 6. What Happens If Galaxy/Audit Refs Are Unavailable

- `mindGalaxyRef === null` if snapshot's galaxyRef is unknown
- Warning added: "此快照的 Mind Galaxy 引用不可用"
- Same for audit ref: "此快照的 Reality Audit 引用不可用"

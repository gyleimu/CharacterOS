## 2. Policy Matrix

| Policy | No Consent | Consent | Confirmation + PASS Preview |
|--------|-----------|---------|---------------------------|
| `never` | preview_only | preview_only | preview_only |
| `preview_only` | preview_only | preview_only | preview_only |
| `require_user_confirmation` | preview_only | confirmation_required | **apply_allowed** |
| `auto_apply_safe_events` | preview_only | confirmation_required* | **apply_allowed** |

*If all safety conditions met (low intensity, high confidence, no warnings, PASS preview, trusted mode)

## 3. Default Safe Behavior

- Never auto-apply without consent
- Reality Audit FAIL → block
- Reality Audit WARN → confirmation required
- Diagnosis language → block
- Low confidence → preview/confirmation required
- Story/plugin/tool modes → not auto-applied

## 4. Blocking vs Warning Flags

| Blocking | Warning |
|----------|---------|
| `possible_diagnosis_claim` | `low_confidence` |
| `unsafe_writeback_without_consent` | `fictional_or_story_input` |
| `high_risk_self_harm_or_medical` | `possible_multi_character_relationship` |
| `raw_state_request` | `sensitive_personal_data` |
| `policy_override_missing` | `plugin_or_tool_input` |
| — | `requires_user_confirmation` |

## 5. Auto-Apply Constraints

`auto_apply_safe_events` only applies when ALL hold:
- Intensity < 0.7
- Confidence > 0.5
- No warning flags (except consent-related)
- Preview PASS
- Input mode is chat/journal (or explicitly trusted)

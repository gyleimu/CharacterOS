## 4. Regression / Improvement Rules

### REGRESSED (high severity)
- qualityVerdict degrades to FAIL
- releaseReady flips to false
- failure count increases
- supportBoundarySafe or neutralStable flips to false

### REGRESSED (medium/low severity)
- qualityVerdict degrades from PASS to WARN
- warning count increases beyond threshold (default: +2)
- benchmark pass rate drops beyond tolerance

### IMPROVED
- qualityVerdict improves (FAIL→WARN, WARN→PASS)
- warning count decreases
- failure count decreases
- benchmark pass rate increases

### STABLE
- All metrics within tolerance of baseline
- No regression or improvement flags

### NO_BASELINE
- No previous baseline exists
- Current run saved as initial baseline

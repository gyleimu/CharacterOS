# CharacterOS V3.7.23 Tick Option Normalization Report

## Summary

This update adds core-level normalization for Continuous Tick options.

The Dashboard and API can send temporary controls into Tick, so the core must guard against unsafe input.

## Updated Core

```text
src/core/time/continuousTick.ts
```

## Normalized Options

```text
daysElapsed          0 ~ 3650
memoryDecayRate      0 ~ 0.2
deepThinkingThreshold 0 ~ 1
fatigue              0 ~ 1
sleepDebt            0 ~ 1
```

## Updated Tests

## Architecture Meaning

Manual controls are useful, but they should not be able to destabilize the physics core.

This keeps experimentation safe while the parameter system is still evolving.

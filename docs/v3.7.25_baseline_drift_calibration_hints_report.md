## Hint Levels

```text
info   short-lived fluctuation or no candidate
watch  possible baseline drift, keep observing
adjust strong candidate, review manually before changing data
```

## Design Constraint

The calibration hint system does not write values back into the character.

It only helps a human operator decide whether a candidate is:

```text
too early
worth watching
strong enough for manual review
```

## Architecture Meaning

CharacterOS is now closer to an operator-assisted calibration workflow:

```text
observe trace
detect candidate drift
explain confidence
allow human review
avoid automatic personality rewrites
```

This matches the current development principle:

Build systems that make parameter evolution visible before making them automatic.

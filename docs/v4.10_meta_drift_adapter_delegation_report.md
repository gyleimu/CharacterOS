## 3. Meta Drift 的特殊性

与 boredom/belief_evolution 不同，meta_drift 的 `metaState` 变量被 **4 个位置** 引用：

| 位置 | 原代码 | 新代码 |
|------|--------|--------|
| Phase 2 计算 | `const metaState = updateMetaStateForTick(...)` | `const metaDriftResult = executeMetaDriftTemporalAdapter(...)` |
| Phase 3 写入 | `state.metaState = metaState.after` | `state.metaState = metaDriftResult.trace.after` |
| Phase 5 读取 | `metaBefore: metaState.before` | `metaBefore: metaDriftResult.trace.before` |
| Return trace | `metaState,` | `metaState: metaDriftResult.trace,` |

这是三次 delegation 中引用最分散的。说明 adapter 模式可以处理多引用场景。
